V14 的问题已经不是“评委看不懂这是一个什么软件”，而是：**核心价值已经讲出来了，但部分真正有分量的实现仍停留在代码里，观众只能看到功能结果，看不见这些结果为什么可信。**

| 实现深度可感知度 | 7.2 / 10 | 画像可纠正、用户原话边界、多 Agent 真实编排、质量门禁和失败降级仍然偏隐形 |

## 二、核心机制审查

### V14 中实际提出的主张

1. 通用 Agent 回答学生已经问出来的问题，AXIOM 发现学生还不知道该问的问题。
2. Dashboard 记录的不是积分，而是正在生长的知识、连接、卡片和复核状态。
3. 一个预测问题可以在正确答案下面暴露隐藏的因果误解。
4. 六维画像不是标签，而是会改变下一问、讲法、节奏和验收方式的教学参数。
5. 已经掌握的内容不重复教，相关旧知用于迁移，真正缺失的才进入追问和生成。
6. Galaxy 的不同视角分别回答关系、任务、掌握和证据问题。
7. 文献卡、闪念卡、永久卡构成知识生命周期；卡片合格不等于能力掌握。
8. 图谱、画像、路径和掌握证据共同决定连接推送与资源推送。
9. 用户确认以后，系统才生成六类可预览资源或写入真实关系。
10. 陌生迁移通过以后，能力状态和学习路径才改变。
11. 原始表达、卡片、画像、评估和路径能够跨会话恢复并反向追溯。
12. 同一套证据闭环可以扩展到学生、教师和完整课程。
13. Agent Harness、RAG、任务系统、测试和文档使它成为可运行系统。

### 真正只能保留的一个核心

> AXIOM 把学生的真实表现变成证据，再让证据决定接下来问什么、教什么、不再教什么，直到陌生迁移证明真的掌握。

这个核心是成立的。V14 的绝大多数机制都在服务它，因此当前不是“核心不明确”，而是**个别实现价值还没有被翻译成观众能一眼识别的结果**。

### 3. 已经实现，但不应进入正片主叙事

这些内容能够证明工程工作量，却不会直接帮助评委理解小林为什么学得更好：

- Clean Architecture 的目录与依赖方向。

- 向量维度、working set、contentHash、重试次数和队列参数。

- ToolContract、Guardrail、确认令牌和审计日志的类名清单。
- 31 个数据模型、289 条规格、592 项断言的逐项展开。

这些内容的正确位置是：**PPT 画面底部的可信度证据条、评审文档索引或答辩追问页，而不是新增口播。**

## 四、传播效果诊断

### 核心问题一句话

V14 的传播问题不是“没内容”，而是**部分 A/B 页重复解释同一个结论，同时最能证明实现深度的控制信号没有被拍到**。

### 五维度诊断

1. **沉默解除：有效**  
   “当答案变得无限便宜，真正稀缺的是问出能暴露真实缺口的问题”说出了教师与学生共同的不安：答案越来越多，但“看过”依然不等于“会了”。这是整套文稿最值得保留的句子。

2. **满足动机：有效**  
   对学生，满足“别重复教我会的，也别让我自己先诊断盲区”；对教师，满足“不要只给分数，让我看见误解与干预依据”；对评委，满足“我要看到赛题功能真的连成了一个结果”。三种动机都被覆盖，但教师和课程价值直到第七、八幕才显现。

3. **立场框架：有效**  
   文稿始终站在学习结果一边，而不是站在模型生成量一边。“卡片合格不等于能力 mastered”“资源不是越多越好”都强化了这个立场。

4. **传播入口：弱到有效之间**  
   开头观点很强，但 01A 随后连续解释 Dashboard 指标、三类卡片和任务状态，容易让观众在故事真正开始前进入“听产品说明书”的状态。第一个会转述它的人能记住“好问题”，却未必能立刻复述 AXIOM 的完整工作方式。

5. **信念结构：有效**  
   全片持续确认一个常识：“学生说懂了、AI 讲过了、笔记写漂亮了，都不能单独证明掌握。”随后用陌生迁移和路径变化给出替代标准，认知闭环完整。

## 五、四个具体传播风险

### 风险一：01A 同时承担四个任务

它同时负责核心钩子、软件定义、Dashboard 指标和卡片盒解释。内容都需要，但必须按顺序渐进出现：

1. 先说这是什么软件；
2. 再说通用 Agent 的问题；
3. 再落到小林的 Dashboard；
4. 最后只高亮与后文有关的三组数字。

现在的口播从钩子直接进入 NODES、EDGES 和卡片名，产品定义主要依赖屏幕文字。对只听音频的人，第一分钟的确定感还可以更强。

### 风险二：部分 B 页在重复 A 页，而不是增加证据

02A/02B、03A/03B、06A/06B、07A/07B 的因果关系是对的，但 B 页口播仍会再次复述 A 页结论。正确职责应固定为：

- A 页只回答“为什么这项设计有价值”；
- B 页只指出“刚才画面中哪一个状态发生了变化，它证明了什么”。

不是简单删短，而是把重复结论换成具体证据名称。

### 风险三：03 组与 05B 各自承担了两个证明任务

- 03 组既要证明语义复用，又要证明 Galaxy 多视角。
- 05B 既要证明推送有依据、执行有确认，又要证明六类资源真实交付。

03 组可以保留，因为 Galaxy 是把语义判断变得可见；但只切三种视角，不再解释十种模式。05B 必须在同一视频内加两个固定章节标记：

> 0—16 秒：为什么推  
> 16—36 秒：确认后交付

这样评委知道前半段在证明“推荐精准”，后半段在证明“资源能交付”，不会把六个快速切换的预览看成一段无意义操作。

### 风险四：08A 的工程可信度仍是名词条

“自研 Harness、Clean Architecture、Postgres / Qdrant / LightRAG、自动化测试”能说明技术栈，却不能让评委直接感知做了多少。结尾更适合展示三个可核对结果：

> 确定性黄金案例门禁 13/13  
> 六类资源独立 Job 与真实预览  
> 来源、评估、路径和失败记录可反向追溯

技术名可以保留为小字，但不能再承担主要证明责任。

## 六、传播边界：三层表达法

### 第一层：必须进入口播的产品结果

- AXIOM 是一套面向高校的 AI 掌握学习系统。
- 通用 Agent 等学生先问；AXIOM 从表现中发现学生还不知道该问的缺口。
- 画像证据会改变下一问。
- 已会内容被跳过，相关旧知用于迁移，真实缺口才触发下一步。
- 只有学生自己的表达能成为“我的理解”；AI 可以整理，不能替他证明学会。
- 卡片质量审核与能力掌握评估是两道不同的门。
- 推送有依据，执行要确认。
- 陌生迁移改变能力和路径。
- 状态能继续，判断能追溯，并最终沉淀为课程资产。

### 第二层：只需上屏或录进视频，不增加口播

- 画像的来源、可信度、待验证/已确认/已推翻状态和用户纠正按钮。
- 前台导师、后台画像、审核、资源、路径的同一工作流状态。
- 系统候选关系、用户确认、真实图谱边的三段状态。
- 推送的证据百分比、验收标准、确认与忽略按钮。
- 六类 Job 的独立进度、质量状态、Manifest 和 HTML/MP4 分层交付。
- Qdrant 快速可用、LightRAG 后台增强的状态标签。
- 13/13 黄金案例门禁与关键对象数量。

### 第三层：留在评审文档和答辩追问

- 具体框架、类名、数据库模型和目录结构。
- 推送八路查找器、十种布局、RAG 同步和重建算法的完整说明。
- 安全规则、确认令牌、队列重试、降级策略和全部测试分层。
- 开源依赖、AI Coding 披露、部署与环境配置。

判断标准只有一个：**能直接改变小林经历和评委结论的，进主叙事；能证明主张是真的，进画面；只能证明工程工作量的，进文档。**

## 七、最小修改清单

以下修改足以把“实现深度可感知度”从当前偏弱状态拉起来，不需要新增页面。

### 1. 01A：先说清软件，再解释 Dashboard

建议用下面两句替换当前口播开头，不增加总时长：

> AXIOM Space 是一套面向高校的 AI 掌握学习系统。通用 Agent 等学生先提出问题；AXIOM 从他的回答里发现他还不知道该问的缺口，再让证据决定下一步。

随后再进入小林和 Dashboard 数字。

### 2. 02B：补一个无口播的画像证据条

> 来源：小林原始回答｜可信度：78%｜状态：待验证 -> 已确认｜用户可纠正

这四个字段能把“画像报告”升级为“可核查的教学决策”。

### 3. 04A：把用户所有权说透

建议加入或替换一句：

> 只有小林自己的表达能成为“我的理解”；AI 可以帮助整理，但不能替他证明学会。

这是代码里已经实现、同时最符合核心立场的一条价值，值得进入主口播。

### 4. 05B：固定为两个章节，并拍到控制信号

前半段只拍证据、可信度、验收标准、确认/忽略；后半段再拍六个 Job、质量状态和预览。候选关系确认后变成真实边，也放在前半段完成。

### 5. 08A：把泛化技术名换成可核对成果

保留技术名为小字，把主证据条改成：

> 13/13 黄金案例门禁｜六类独立资源 Job｜全过程可追溯

## 八、最终判断

V14 已经能够让完全不了解项目的人，从小林的一次错误出发，理解 AXIOM 是如何发现缺口、改变教学、复用旧知、沉淀知识、生成资源、验证掌握并调整路径的。这个 0 到 1 的主故事已经成立。

目前真正损失传播效果的，不是遗漏了更多功能，而是两件事：

1. 有些 B 页重复结论，挤占了展示证据的时间；
2. 代码里最有价值的“可纠正、不可冒充、需确认、可追溯”还没有被做成观众一眼能看到的状态。

因此下一版不应继续扩页，而应执行一条更严格的表达原则：

> **口播只讲价值和因果，画面负责证明控制与结果，文档负责承接工程深度。**

改完后的整片骨架应当是：

> Dashboard 显示小林已经积累的学习状态 -> 一个好问题暴露隐藏缺口 -> 证据改变画像与下一问 -> 已会内容被跳过、旧知参与迁移 -> 小林自己的表达经过质量门进入知识网络 -> 图谱与画像提出有依据且需确认的下一步 -> 陌生迁移证明掌握并改写路径 -> 同一证据链持续服务学生、教师和完整课程。
