# AXIOM Space A3 演示视频脚本 09：新手从零使用场景剧本

## 1. 这个剧本讲什么

这版脚本是给一个第一次使用 AXIOM Space 的新手看的。

他不知道这个软件怎么用，所以视频必须按真实使用顺序讲：

```text
先创建仓库
→ 再让系统了解自己
→ 再导入资料
→ 再生成学习路径
→ 再进入第一张卡学习
→ 再生成资源
→ 再评估掌握
→ 最后看图谱和画像沉淀
```

这不是功能清单。  
这是一条新手从零开始完成第一次学习闭环的路线。

## 2. 用户是谁

用户是一名计算机专业专科学生。

他现在正在学《数据结构》，手里有老师发的课件和教材，但学到“图与最短路径”时卡住了。

他的真实问题不是没有资料，而是：

```text
我看过资料，也问过 AI。
但我还是分不清路径、边权、最短路径和 Dijkstra。
我以为最短路径就是经过边数最少。
```

所以这条视频要教会新手：

```text
第一次打开 AXIOM Space，应该怎么一步一步使用它来学懂一个知识点。
```

## 3. 总体顺序

```text
1. 新建学科仓库：数据结构
2. 系统主动问用户当前要学什么
3. 系统把用户困难变成画像
4. 系统建议导入真实资料
5. 用户导入真实长资料
6. 系统把资料变成 Markdown 文献节点
7. 系统生成第一条学习路径
8. 用户打开第一张理解卡
9. AI 引导用户用自己的话解释
10. 用户解释被写进卡片并自动保存
11. 系统显示画像变化如何影响后续输出
12. 系统生成针对性学习资源
13. 系统展示多智能体角色和各自产物
14. 系统推送下一步连接和任务
15. 用户完成当前任务并做掌握评估
16. 错误内容被拦截
17. 正确内容升级为永久知识
18. 用户查看知识图谱变化
19. 用户查看认知画像变化
20. 用户看到第一次学习闭环完成
```

### 场景 09：AI 不直接给答案，而是要求用户解释

AI 提问：

```text
先不要背定义。请你用自己的话解释：

```

```text

如果一条边权重是 100，而三条边每条权重是 10，

所以最短路径比较的是权重总和，不是边数。
```

为什么接这一步：

```text
因为看过解释不等于学会。
系统要让用户自己说出来，才知道他是否真的理解。
```

```text
为什么带权图里的最短路径不是边数最少？
请给出一个反例。
```

用户回答：

```text
A 到 B 直接边权是 100。
A 到 C 到 D 到 B 三条边分别是 10、10、10。
三条边总代价是 30，所以三条边这条路线更短。
```

系统输出：

```text
掌握判断：通过。

通过原因：
1. 能区分边数和权重总和。
2. 能给出具体反例。
3. 能用自己的话解释。

下一步：学习 Dijkstra 的选择过程。
```

为什么接这一步：

```text
因为用户已经学习完当前卡片。
系统要判断他是真的掌握，还是只是看过。
```

这一幕要让新手明白：

```text
任务不是点一下就完成。
你需要输出解释，系统才会判断是否通过。
```

A3 踩点：

```text
学习效果评估。
多维度、精准评估。
动态调整学习计划。
```

### 场景 16：错误内容不能升级
