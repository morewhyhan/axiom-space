# AXIOM Space A3 演示视频脚本 第五版：证据得分版

## 1. 成果价值

AXIOM Space 的成果价值不在于“多做了几个 AI 功能”，而在于解决了高等教育个性化学习里的三个核心问题：

```text
1. 学生不知道自己该怎么学。
2. 学习资源很多，但不精准、不成路径。
3. 学生看过资料，却无法证明自己真的掌握。
```

所以视频要证明的不是“系统有很多页面”，而是这一条闭环：

```text
学生原来卡住了
→ 系统主动识别卡点
→ 基于真实资料生成路径
→ 针对卡点生成资源
→ 逼学生用自己的话输出
→ 系统评估是否掌握
→ 合格知识沉淀进图谱和画像
```

一句话总结：

> AXIOM Space 把 AI 从“给答案的工具”，推进成“能建画像、排路径、推资源、促输出、做评估、沉淀知识”的可验证学习系统。

## 2. 第五版表达策略

这一版要比第四版更硬：

```text
少说：我们支持很多功能。
多说：我们解决了哪个学习断点。

少说：我们生成了很多资源。
多说：资源为什么是针对这个学生的。

少说：系统很智能。
多说：画面上能看到哪些证据。

少说：AI 可以辅导。
多说：学生是否真的说清楚了，系统如何判断。
```

录制时每一段必须回答三个问题：

```text
解决了什么？
画面证据是什么？
对应哪个 A3 得分点？
```

## 3. 演示设定

学生：

计算机专业专科学生，正在学习《数据结构》。

仓库：

```text
数据结构
```

本次学习主题：

```text
图与最短路径
```

核心误区：

```text
学生把“边数最少”误认为“最短路径”。
但在带权图里，最短路径比较的是路径上所有边权的总和。
```

这一误区非常适合演示：

```text
有真实教材依据。
有清楚的错误样子。
有可解释的反例。
能生成图解、练习、代码和动画。
能通过输出评估学生是否掌握。
```

## 4. 总体视频结构

```text
1. 痛点：资料多，但学生不知道哪里没懂。
2. 建仓库：新建《数据结构》长期知识库。
3. 主动诊断：系统主动问，识别学生误区。
4. 导入资料：用真实长资料作为生成依据。
5. 生成路径：把资料拆成明确学习步骤。
6. 打磨卡片：用费曼学习法逼学生说清楚。
7. 资源生成：围绕误区生成至少 5 类资源。
8. 精准推送：推送连接和下一组任务。
9. 掌握评估：用输出证明学会，而不是看过。
10. 防幻觉：不合格内容不能升级。
11. 沉淀结果：图谱、画像、知识卡同步更新。
```

### 第七段：费曼输出，证明学生真的开始理解

页面操作：

Agent 提问：

```text
先不要背定义。请你用自己的话解释：
为什么一条边不一定比三条边更短？
```

用户回答：

```text
因为每条边有不同权重。
如果一条边权重是 100，而三条边每条权重是 10，

所以最短路径比较的是权重总和，不是边数。
```

```markdown
边数少不一定总代价低，边数多也可能总代价更低。
```

旁白：

这里不是 AI 讲得更完整，而是学生自己说清楚了。AXIOM Space 用费曼学习法逼出学生的表达，并把它保存为学习证据。
