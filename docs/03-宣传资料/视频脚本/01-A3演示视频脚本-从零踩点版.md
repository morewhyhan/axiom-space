# A3 演示视频脚本：从零踩点版

## 0. 脚本目标

本视频只做一件事：

> 从一个新账号、新仓库开始，完整展示 AXIOM Space 如何通过对话了解学生、导入课程资料、生成知识图谱、围绕一张卡片进行 AI 辅导和自动写卡、用审核机制阻止低质量内容进入永久知识，并最终让画像、路径、资源推送和长期档案形成闭环。

这不是功能导览。每一步都必须对应赛题得分点，并且让评委在页面上看到证据。

建议成片控制在 7 分钟内。等待生成过程可以剪辑压缩，但关键输入、关键输出、失败证据和状态变化不能剪掉。

---

## 1. 录制总原则

### 1.1 每一幕都必须回答

1. 当前在哪个页面。
2. 用户点了什么、输入了什么。
3. 页面出现了什么可见变化。
4. 这个变化证明了哪个赛题踩点。
5. 如果评委质疑，页面上有什么反证。

### 1.2 不能只靠旁白声称

以下说法必须有页面证据：

- 系统了解了学生画像。
- 系统有多智能体协作。
- 系统生成了个性化资源。
- 系统做了学习效果评估。
- 系统拒绝了低质量永久卡。
- 系统根据评估更新了画像、路径和推送。

### 1.3 本次演示主案例

学生：小林，高职专科二年级，计算机应用技术专业。

课程：《软件设计模式》。

局部打磨节点：`Visitor 双重分派`。

注意：视频要明确让评委看到整门课资料和图谱，但深入演示只选择一个节点。不能让人误以为系统只会学 Visitor。

节点数量口径：

- 从零新建仓库只验证功能闭环、课程结构和页面证据是否自然，不要求 300+ 节点。
- 300+ 节点属于后半段“成熟学期档案”展示，用来证明长期使用后的产品形态。
- 因此从零导入只要覆盖完整课程模块、能生成路径/图谱/卡片/推送，并且看起来不突兀即可。

### 5:10-5:40 自动写卡：证明对话进入知识库

页面：AI 工作台，右侧卡片预览。

操作：

1. 不手动编辑右侧卡片。
2. 等待 Agent B 自动记录。
3. 观察右侧卡片内容刷新。
4. 点击卡片的保存状态或历史记录。

画面必须看到：

- 右侧卡片从空框架变成有内容。
- 有“自动记录中/已写入卡片/已保存”提示。
- 内容包含用户自己的理解，而不是标准百科定义。
- 能看到来源消息或写入记录。

卡片内容应变成类似：

```text
## 我当前的理解
Visitor 的双重分派不是神秘语法，而是分两步把两个对象的真实类型都用上。

第一步 element.accept(visitor) 靠元素对象的运行时类型进入具体元素类。
第二步在 PdfNode.accept 里面调用 visitor.visit(this)，此时 this 的静态类型就是 PdfNode，所以 Java 重载选择能选到 visit(PdfNode)。

## 例子
Document element = new PdfNode();
直接 visitor.visit(element) 时，编译器只看到 Document。
进入 PdfNode.accept 后，this 在方法体里是 PdfNode。

## 仍待验证
我还需要用一个陌生 AST 节点解释同样机制。
```

## 5. 录制前检查清单

### 5.1 数据检查

- 新账号可以注册登录。
- 新仓库可以创建。
- 初始对话可以生成 6 维画像。
- 资料导入后能生成整门课节点。
- 图谱能切换环形、邻域、证据等布局。
- 图谱能点击节点进入工作台。
- Visitor 卡片初始是空框架或半成品，不是完整答案。
- Agent B 能自动写卡。
- 永久卡审核能失败。
- 补充后永久卡审核能通过。
- 路径会根据审核结果变化。
- 推送箱有连接、资源、任务组三类对象。
- 成熟档案有 300+ 节点。

### 5.2 页面检查

- 认知洞察顶部只保留胶囊、画像面板、注入画像提示词按钮。
- 路径规划资源推送是箱子式列表，不是两个切换按钮。
- 工作台三栏清楚：左侧上下文/对话入口，中间 AI 对话，右侧卡片预览。
- 图谱录制时不要长时间展示星系/星团视图，优先使用环形、邻域、证据。
- 审核面板能显示逐项评语。
- 资源预览不是空内容。

### 5.3 旁白检查

旁白只解释页面正在证明什么，不替页面编结论。

可以说：

```text
系统提出了可验证的学习机制假设。
系统根据画像改变了解释顺序和审核标准。
这张卡片通过了逐项审核，所以进入永久知识。
```

不要说：

```text
系统完全看透学生。
系统比学生更懂自己。
系统已经证明长期学习成绩显著提升。
```
