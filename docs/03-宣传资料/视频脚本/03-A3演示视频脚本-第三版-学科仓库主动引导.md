# AXIOM Space A3 演示视频脚本 第三版：学科仓库与主动引导

## 1. 这一版的核心修正

这版视频从一个真正合理的仓库开始：

仓库不是一次任务，也不是一张单独卡片，而是某个学科或领域的长期知识空间。本次演示新建的仓库叫：

```text
数据结构
```

它存放的是数据结构这门课里的长期知识：线性表、树、图、路径、最短路径、算法适用边界、代码实操、练习题和学习证据。

本次演示只从这个仓库里选择一个具体学习主题：

```text
图与最短路径
```

第二个核心修正是：用户不应该自己新建对话、自己写自我介绍。新手进入空仓库后，系统应该主动开启一个普通对话会话，向用户提必要问题，完成学习画像初始化。

第三个核心修正是：导入资料必须像真实学习场景。用户导入的是一份很长的课程文件，或者直接粘贴一大段教材/讲义全文；系统把它转成 Markdown 文献节点，再生成路径、卡片和图谱。

## 2. 演示设定

主角：

一名计算机专业专科学生，第一次使用 AXIOM Space。他正在学习《数据结构》课程。

新建仓库：

```text
数据结构
```

仓库含义：

这是学生长期学习数据结构的知识库，不是一次聊天，也不是一次任务。它会持续保存课程资料、文献节点、理解卡、永久知识卡、资源卡、任务组、学习路径和认知画像证据。

本次学习主题：

```text
图与最短路径
```

真实痛点：

学生见过“图、顶点、边、路径、边权、Dijkstra”这些词，但说不清它们之间的关系。尤其容易把“边数最少”和“权重总和最小”混为一谈。

## 3. 视频主线

1. 新建一个学科仓库：`数据结构`。
2. 系统主动发起普通对话，询问学习目标、基础和卡点。
3. 用户说想先学 `图与最短路径`。
4. 用户导入一份真实长资料，或粘贴一大段课程讲义。
5. 系统把资料保存为 Markdown 文献节点。
6. 系统从资料中抽取二级概念节点、灵感草稿和学习路径。
7. 用户进入第一张理解卡学习。
8. AI 用费曼学习法让用户说出自己的理解。
9. 系统根据卡点推送连接、资源和任务组。
10. 系统生成多类型个性化资源。
11. 用户完成掌握评估。
12. 内容通过质量校验后沉淀为永久知识，进入图谱和画像。

### 第四段：导入一份真实的长资料

页面操作：

```text

```

导入资料展示两种真实方式。

第一种：选择文件。

用户点击 `选择文件`，选择类似文件：

```text

```

或者：

```text
数据结构_图与最短路径_课堂笔记.docx
```

第二种：粘贴长文本。

用户也可以把教材或讲义中一整节内容粘贴进去，例如：

```text
第六章 图
6.1 图的基本概念
图由顶点集合和边集合组成……

6.2 路径与连通性
在图中，路径是顶点和边交替组成的序列……

6.3 带权图与路径长度
如果每条边都有一个权值，那么路径长度通常指路径上各边权值之和……

6.4 最短路径问题
最短路径问题是在带权图中寻找两个顶点之间权值总和最小的路径……

6.5 Dijkstra 算法
Dijkstra 算法适用于边权非负的单源最短路径问题……

……
```

录屏时要表现这是长资料，不是一两句话。可以快速滚动文本框，让观众看到资料包含章节、定义、例子和算法说明。


### 第七段：AI 用费曼学习法追问

页面操作：

Agent 主动提问：

```text
先不要背定义。
请你用自己的话解释：为什么一条边不一定比三条边更“短”？
```

用户回答：

```text
因为图里的边可能有不同代价。
如果一条边的权重是 100，而三条边每条权重是 10，
那么三条边这条路线总权重是 30，反而更短。
```

Agent 继续追问：

```text
很好。请把这句话写进右侧卡片：
最短路径比较的是路径总代价，不是经过边的数量。
```

用户把自己的话写入卡片。

旁白：

这里的重点不是 AI 讲得多，而是学生能不能说出来。系统通过费曼学习法让学生输出自己的理解，并把这段理解保存到卡片中。

## 6. 第三版录制原则

1. 仓库名称要是学科或领域，例如 `数据结构`，不要用一次任务当仓库名。
2. 系统必须主动引导用户，不要让用户自己新建对话并自我介绍。
3. 导入资料要展示真实长文件或长文本，不要只贴几句示例。
4. 每一步都要展示产物：画像、文献节点、二级节点、路径、卡片、资源、评估、图谱。
5. A3 关键词要跟着真实动作出现，不要变成单独念口号。
