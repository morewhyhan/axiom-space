# AXIOM Space A3 演示视频脚本 第四版：简洁得分导向

## 1. 写作原则

这版脚本只围绕一件事：

> 一个计算机专业学生，面对《数据结构》里“图与最短路径”这个知识点，原本资料很多但学不清。AXIOM Space 主动理解他、整理资料、规划路径、生成资源、评估掌握，最后把学习结果沉淀成长期知识。

录制时少讲：

```text
我们做了很多功能。
```

多讲：

```text
学生原本解决不了什么问题。
AXIOM Space 怎么解决。
强在哪里。
为什么符合 A3 赛题得分点。
```

注意：这些不是直接念出来的口号，而是整条视频的表达方向。

## 2. 演示设定

学生：

计算机专业专科学生，正在学《数据结构》。

新建仓库：

```text
数据结构
```

本次学习主题：

```text
图与最短路径
```

真实问题：

学生有课件、教材和 AI，但还是分不清“路径、边权、最短路径、Dijkstra”之间的关系。尤其不知道最短路径到底是在比较边数，还是比较权重总和。

这就是视频要解决的问题。

## 3. 视频主线

```text
新建学科仓库
→ 系统主动了解学生
→ 导入真实长资料
→ 生成学习路径
→ 打磨第一张理解卡
→ 生成个性化资源
→ 做掌握评估
→ 沉淀到图谱和画像
```

## 4. 分镜脚本

### 第一段：先展示问题，不先展示功能

页面操作：

打开 AXIOM Space，进入知识库创建页。创建新仓库：

```text
数据结构
```

旁白：

很多学生学数据结构时，并不是没有资料。课件、教材、网课、AI 回答都有，但真正的问题是：资料太多，顺序不清，自己也说不清到底哪里没懂。

今天我们从一个空的《数据结构》仓库开始，演示 AXIOM Space 如何帮助学生把一个知识点真正学明白。

画面重点：

展示空仓库。中心节点是 `数据结构`，表示这是一个长期学科知识库，不是一次临时聊天。

内部得分意图：

先让评委看到用户痛点：**学习资源繁杂无序、难以精准匹配自身需求**。

**【A3 踩点：学习资源繁杂无序、难以精准匹配自身需求】**

**【A3 踩点：以某一具体专业课程为切入点】**

### 第二段：系统主动问，而不是让用户自己会用

页面操作：

进入空仓库后，系统主动弹出一个普通对话：

```text
欢迎进入「数据结构」仓库。
我先了解你的学习情况，再帮你生成学习路径。

你现在最想解决数据结构里的哪个知识点？
你已经知道什么？
你最容易卡在哪里？
```

用户回答：

```text
我想学图与最短路径。
我知道图有点和边，但分不清路径、边权、最短路径和 Dijkstra。
我看定义能看懂，一做题就乱。
```

旁白：

这里没有让学生填写复杂表单，也没有要求他自己写完整提示词。系统主动发起对话，用自然语言识别学生的目标、基础和卡点。

画面重点：

右下角出现画像记录提示，或短暂切到认知洞察展示画像维度：

```text
学习目标
当前基础
易错点
解释偏好
节奏负荷
掌握校验
```

内部得分意图：

强调系统不是被动聊天框，而是主动完成 **对话式学习画像自主构建**。

**【A3 踩点：对话式学习画像自主构建】**

**【A3 踩点：摒弃传统繁琐表单】**

**【A3 踩点：不少于6个维度】**

### 第三段：导入真实资料，而不是凭空生成

页面操作：

系统提示：

```text
建议先导入你的课程讲义，我会基于真实资料生成路径和卡片。
```

进入 `路径规划`，点击 `新任务`，选择 `导入资料`。

主题填写：

```text
图与最短路径
```

选择一个真实长文件：

```text
数据结构课程讲义-第6章-图.pdf
```

或者粘贴一整段很长的教材内容，录屏时快速滚动，让观众看到它包含章节、定义、例题和算法说明。

点击：

```text
导入资料并生成路径
```

旁白：

AXIOM Space 不是让大模型凭空讲课，而是先接收学生真实使用的课程资料。系统会把长文件或长文本保存成 Markdown 文献节点，再从里面抽取概念、卡片和学习路径。

画面重点：

展示生成进度：

```text
解析资料内容
抽取灵感草稿
生成学习路径
同步知识图谱
```

内部得分意图：

体现“有依据的生成”，不是裸 AI 文本，为后面的防幻觉和 RAG 铺垫。

**【A3 踩点：依托各类高等教育资源】**

**【A3 踩点：初始知识库/文档集】**

**【A3 踩点：生成进度追踪】**

### 第四段：资料变成清晰学习路径

页面操作：

导入完成后，Learn 页面出现学习路径：

```text
1. 图的点和边
2. 路径与连通性
3. 边权与路径代价
4. 最短路径到底短在哪里
5. Dijkstra 的基本直觉
6. Dijkstra 的适用边界
```

旁白：

普通 AI 往往只给一段解释，但学生真正需要的是顺序。AXIOM Space 会把长资料拆成可以一步步推进的学习路径，让学生知道先学什么、后学什么、每一步要解决什么问题。

画面重点：

左侧路径列表，中间学习路线，右侧当前任务详情。

内部得分意图：

让评委看到系统解决的是“不会规划、不会筛资源”的问题。

**【A3 踩点：个性化学习路径规划和资源推送】**

**【A3 踩点：明确学习步骤和顺序】**

### 第五段：围绕一张理解卡真正学会

页面操作：

点击任务：

```text
最短路径到底短在哪里
```

进入 AI 工作台。右侧出现理解卡：

```markdown
## 当前疑问

最短路径是在比较边数，还是比较权重总和？

## 来自资料的依据

在带权图中，路径长度通常指路径上各边权值之和。
```

Agent 提问：

```text
先不要背定义。请你用自己的话解释：
为什么一条边不一定比三条边更短？
```

用户回答：

```text
因为每条边有不同权重。
如果一条边权重是 100，而三条边每条权重是 10，
三条边的总代价是 30，反而更短。
```

旁白：

这里的关键不是 AI 又讲了一遍，而是学生必须用自己的话说出来。系统围绕当前卡片追问、纠错、记录，把一次对话变成可沉淀的理解。

画面重点：

用户把自己的解释写进卡片，右下角出现自动保存或观察记录提示。

内部得分意图：

突出强点：不是“给答案”，而是“逼出掌握证据”。

**【A3 踩点：智能辅导】**

**【A3 踩点：实时跟踪学生的学习行为】**

**【A3 踩点：画像的随学随新】**

### 第六段：资源生成要针对卡点

页面操作：

用户输入：

```text
请基于这张卡和刚导入的讲义，生成适合我的学习资源。
```

系统生成资源，展示进度：

```text
读取当前卡片
检索导入资料
分析用户卡点
生成资源
校验内容
```

资源结果：

```text
讲解文档：路径代价和最短路径
思维导图：图、路径、边权、最短路径的关系
练习题：判断哪条路径总代价更小
代码实操：用 Python 表示带权图
视频/动画脚本：Dijkstra 的选择过程
```

旁白：

这些资源不是统一模板，而是针对学生刚暴露的问题生成的。他混淆边数和权重，系统就围绕这个误区生成文档、图解、练习、代码和动画资源。

画面重点：

多种资源以卡片形式展示。

内部得分意图：

明确“强在哪里”：资源不是多，而是准；不是堆材料，而是补短板。

**【A3 踩点：多智能体协同的资源生成】**

**【A3 踩点：至少5种类型的个性化资源生成】**

**【A3 踩点：多模态学习资料】**

### 第七段：推送下一步，而不是让学生自己找

页面操作：

回到 `路径规划`，展示 `推送箱`。

连接推送：

```text
建议连接：边权 -> 最短路径
理由：学生已理解路径总代价，应建立显式概念关系。
```

资源与任务：

```text
建议任务组：Dijkstra 为什么不能处理负权边
理由：当前路径缺少算法适用边界。
```

旁白：

当学生完成一个理解点后，系统会判断下一步应该补什么。它可能推送一个概念连接，也可能推送一组新任务。这样学生不需要自己在海量资料里找下一步。

画面重点：

两个盒子：`连接推送`、`资源与任务`。

内部得分意图：

把“资源推送”讲成学习闭环里的下一步决策，而不是推荐列表。

**【A3 踩点：基于画像实现学习资源的精准推送】**

**【A3 踩点：动态调整学习资源推送策略和学习计划】**

### 第八段：评估掌握，防止“看过等于会了”

页面操作：

点击当前任务 `完成 / 评估掌握`。

系统提问：

```text
不用教材原话，请解释：
为什么带权图里的最短路径不是边数最少？
```

用户回答：

```text
因为最短路径比较的是路径上所有边权的总和。
边少不一定总代价小，边多也可能总代价更低。
```

系统评估：

```text
掌握判断：通过
证据：能区分边数和权重总和，能给出反例。
下一步：学习 Dijkstra 如何选择当前代价最小的节点。
```

旁白：

AXIOM Space 不把“看过资源”当成学会，而是要求学生输出解释，并根据输出判断是否掌握。通过后，路径才进入下一步。

画面重点：

显示掌握判断、证据、下一步。

内部得分意图：

说明凭什么有效：有输出、有证据、有评估、有路径调整。

**【A3 踩点：学习效果评估】**

**【A3 踩点：多维度、精准评估】**

### 第九段：质量校验，保证知识能沉淀

页面操作：

点击 `升级为永久知识`。

如果内容不合格，弹窗驳回：

```text
暂不能升级。

问题：
1. 清晰性不足：没有说明路径总代价。
2. 准确性不足：仍把最短路径说成边数最少。
3. 必要性不足：缺少例子。
```

修改后再次升级，显示：

```text
已升级为永久知识
```

旁白：

进入长期知识库的内容必须经过校验。只有清晰、准确、必要的内容，才会成为永久知识。这样可以减少错误内容被长期保存。

画面重点：

驳回弹窗、修改、升级成功。

内部得分意图：

把“防幻觉”落到可见机制：不是一句承诺，而是有驳回、有理由、有边界。

**【A3 踩点：防幻觉与内容安全过滤机制】**

**【A3 踩点：确保生成的学术内容无事实性错误】**

## 5. 录制时的判断标准

正确的效果：

```text
观众看完后记住的是：
这个系统解决了新手不知道怎么学的问题。
它会主动了解学生。
它能把真实资料变成路径和资源。
它要求学生输出，并判断是否掌握。
它把学习结果沉淀成长期知识。
```

错误的效果：

```text
观众只记住：
这个系统有很多页面、很多按钮、很多功能。
```

所以录制时，每个画面都要回答一个问题：

```text
这个画面证明 AXIOM Space 解决了什么学习问题？
```
