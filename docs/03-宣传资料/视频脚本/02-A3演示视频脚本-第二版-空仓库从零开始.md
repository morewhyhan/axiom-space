# AXIOM Space A3 演示视频脚本 第二版：空仓库从零开始

## 1. 这一版的核心

这条视频不先展示一堆功能，而是演示一个新手第一次使用 AXIOM Space：

从一个空知识库开始，学生只需要说明“我要学什么、我哪里不会”，系统就会一步步帮他建立学习画像、导入资料、生成路径、创建卡片、推送资源、辅导理解、评估掌握，并把结果沉淀到知识图谱里。

视频要让观众明白：

AXIOM Space 不是普通 AI 聊天框，也不是资料生成器，而是一个会把“学习目标”逐步变成“知识结构、学习路径、资源和掌握证据”的 AI 掌握学习系统。

## 2. 演示设定

主角：

一名计算机专业专科学生，第一次使用 AXIOM Space。他不是要做项目，而是要学懂一个具体知识主题。

本次学习主题：

`数据结构：图与最短路径`

新建知识库名称：

`图与最短路径入门`

这个知识库一开始是空的。中心白色节点就是一级节点，代表这个知识库的学习目的：

> 我想从零理解图、路径、边权、最短路径和 Dijkstra 算法之间的关系。

后续所有二级节点、文献节点、理解卡、资源卡、任务组和连接，都是围绕这个一级节点生长出来的。

## 3. 演示主线

整条视频按新手真实操作顺序走：

1. 新建空知识库。
2. 让 AI 先了解学生，而不是直接讲答案。
3. 导入一份课程资料，或粘贴教材片段。
4. 系统把资料变成文献节点、理解卡和学习路径。
5. 学生从第一张卡片开始学习。
6. AI 用费曼学习法逼学生说出自己的理解。
7. 系统生成资源、推送连接和任务组。
8. 学生完成掌握评估。
9. 知识沉淀为永久卡，进入图谱和画像。

### 第三段：先让 AI 了解学生

页面操作：

进入 `AI 工作台`，新建普通对话。用户输入：

```text
我是计算机专业的专科学生，正在学数据结构。
我想学懂“图与最短路径”，但我现在只知道图有点和边，
分不清路径、边权、最短路径、Dijkstra 之间的关系。
你先不要直接讲完整答案，先判断我应该从哪里开始学。
```

Agent 回复时先问必要问题：

```text
我先确认两个点：
1. 你能不能用自己的话说说“路径”是什么？
2. 你觉得“最短路径”是边数最少，还是权重总和最小？
```

用户回答：

```text
我觉得路径就是从一个点走到另一个点。
最短路径应该就是经过的边最少吧？
```

旁白：

这里系统没有要求学生填写表单，而是通过对话识别他的基础和误区。学生的关键卡点是：把“边数最少”和“权重总和最小”混在了一起。

### 第七段：用费曼学习法逼出真实理解

页面操作：

Agent 不直接给最终答案，而是要求学生用自己的话解释：

```text
你先像教同学一样解释：
为什么“边数最少”不一定等于“权重总和最小”？
```

学生回答：

```text
如果每条边的代价不一样，走两条边可能比走一条边更便宜。
所以最短路径不是看经过几条边，而是看总权重最小。
```

Agent 继续追问：

```text
很好。那如果一条边权重是 100，另一条路线有三条边，每条权重是 10，
哪条才是最短路径？请你写进右侧卡片。
```

学生把自己的话写入卡片。

旁白：

系统不会只让学生“看懂”，而是要求他用自己的话讲出来。这个过程使用费曼学习法：能不能解释清楚，才是有没有真正理解的证据。

画面产物：

卡片中出现学生自己的解释。对话过程中的关键理解会被记录，卡片自动保存。

**【A3 原文踩点：针对性学习引导】**

**【A3 原文踩点：实时跟踪学生的学习行为】**

**【A3 原文踩点：画像的随学随新】**

### 第八段：资源推送不是乱推，而是补缺口

页面操作：

回到 `路径规划`，展示 `推送箱`。

推送箱分为两类：

1. `连接推送`
2. `资源与任务`

连接推送示例：

```text
建议连接：边权 -> 最短路径
理由：学生刚刚修正了“边数最少”的误解，需要把边权和路径代价建立显式关系。
置信度：86%
```

资源与任务示例：

```text
建议任务组：Dijkstra 为什么不能处理负权边
理由：当前路径缺少算法适用边界，会影响后续理解。
```

旁白：

推送箱不是简单推荐资料，而是根据当前知识库、学生画像和学习证据，判断哪里缺连接、哪里缺卡片、哪里需要补任务。

用户此刻的理解：

系统不是让我自己找下一步，而是告诉我现在最该补哪一块。

画面产物：

节点连接推送、资源卡片推送、任务组推送。

### 第十段：展示防幻觉和质量边界

页面操作：

打开资源详情，展示引用来源、校验状态、RAG 相关卡片。尝试升级当前卡片为永久知识。

### 第十一段：做一次学习效果评估

页面操作：

在学习路径里点击当前任务 `完成 / 评估掌握`。

系统要求学生最后解释：

```text
请不用教材原话，用自己的话回答：
为什么带权图里的最短路径不是边数最少？
```

学生回答：

```text
因为每条边都有代价。最短路径比较的是一条路线所有边权加起来的总成本。
如果一条边代价很高，三条边的总成本反而可能更低。
```

系统评估：

```text
掌握判断：通过
证据：能区分边数和权重总和，能给出反例。
下一步：学习 Dijkstra 如何一步步选择当前代价最小的节点。
```

旁白：

系统不把“看完资源”当成学会，而是根据学生输出、练习反馈和卡片内容做学习效果评估。评估通过后，学习路径自动推进到下一步。

## 6. 这一版录制时的关键原则

1. 永远从新手视角讲：我现在该点哪里，为什么要这样做。
2. 不要一上来展示所有页面，先让观众看到空仓库怎么长出来。
3. A3 关键词要跟着真实动作出现，不要生硬念清单。
4. 每个页面都要有产物：画像证据、文献节点、路径、理解卡、资源、评估、图谱。
5. 每一步都要说明用户价值：少走弯路、知道下一步、讲得出来、能被评估、能沉淀。
