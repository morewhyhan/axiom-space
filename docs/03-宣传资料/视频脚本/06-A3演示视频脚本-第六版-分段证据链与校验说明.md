# AXIOM Space A3 演示视频脚本 第六版：分段证据链与校验说明

## 1. 文档定位

这不是旁白稿，也不是直接展示给观众看的页面文案。

这份文档用于指导录制、产品调整和开发对齐。它回答每一段视频背后的四个问题：

```text
1. 用户做了什么输入？
2. 系统中间怎么处理？
3. 系统输出了什么？
4. 怎么证明这个输出是正确、可信、可得分的？
```

这份文档基于第四版的简洁主线和第五版的证据得分逻辑。第五版负责“怎么讲”，第六版负责“每个讲法背后必须有什么证据”。

核心原则：

```text
所有表达都必须能在页面上被呈现、被证明。
不能只说系统很强，必须让评委看见它强在哪里。
不能只说符合 A3，必须让评委看见每个得分点的证据链。
```

## 2. 成果价值的硬表达

AXIOM Space 的成果价值不是“做了很多 AI 功能”，而是解决了学习过程里的三个断点：

```text
断点一：学生不知道自己该怎么学。
断点二：资源很多，但不精准、不成路径。
断点三：学生看过资料，却无法证明自己真正掌握。
```

因此，演示视频必须证明：

```text
学生原来卡住了。
系统识别出了具体卡点。
系统基于真实资料生成路径。
系统围绕卡点生成资源。
学生被要求用自己的话输出。
系统用标准判断是否掌握。
合格内容沉淀进长期知识图谱和画像。
```

一句话价值：

> AXIOM Space 把 AI 从“给答案的工具”，推进成“能建画像、排路径、推资源、促输出、做评估、沉淀知识”的可验证学习系统。

## 3. 四个说服维度

这四个问题不能生硬地直接念出来，而要自然埋进每个画面和证据里。

### 3.1 用户为什么有这个需求？

因为真实学生不是缺资料，而是：

```text
资料太多，不知道先学什么。
AI 回答很多，但不一定适合自己。
自己以为懂了，其实一解释就露馅。
课程学习是长期过程，需要持续积累。
```

页面证据应该体现：

```text
空仓库
真实长资料
学生误区
学习路径
掌握评估
画像变化
```

### 3.2 用户为什么信任我们？

不要靠自夸建立信任，要靠产品态度建立信任：

```text
不让 AI 凭空编，先导入真实资料。
不直接说学生懂了，要让学生输出。
不把错误内容沉淀为知识，要做质量校验。
不只服务一次聊天，要长期更新图谱和画像。
```

页面证据应该体现：

```text
资料引用
RAG/文献节点
清晰、准确、必要校验
驳回理由
评估标准
长期知识图谱
```

### 3.3 用户为什么要用这个软件？

因为它不是单点工具，而是闭环系统：

```text
普通聊天工具只回答问题。
普通资料工具只整理资料。
普通笔记工具只保存笔记。

AXIOM Space 把诊断、路径、资源、辅导、评估、沉淀连成闭环。
```

页面证据应该体现：

```text
主动画像
路径规划
个性化资源
推送箱
掌握评估
知识图谱
认知洞察
```

### 3.4 用户为什么现在就要用？

因为他现在就卡在一个具体学习问题上：

```text
马上要学图与最短路径。
手里已经有课程资料。
但现在不知道怎么学、怎么补、怎么判断掌握。
```

页面证据应该体现：

```text
当前课程：数据结构
当前主题：图与最短路径
当前误区：边数最少 ≠ 最短路径
当前下一步：Dijkstra 的选择过程和适用边界
```

## 4. 全局证据规则

每个得分点都要形成四段证据链：

```text
用户输入
→ 系统处理
→ 系统输出
→ 校验方式
```

每个页面画面都要避免空泛表达：

```text
错误表达：系统生成了个性化资源。
正确表达：系统基于“学生混淆边数和权重总和”的误区，生成文档、导图、练习、代码和动画资源。

错误表达：系统会评估学习效果。
正确表达：系统要求学生解释“为什么带权图中的最短路径不是边数最少”，并根据是否能区分边数和权重总和、是否能给出反例来判断掌握。

错误表达：系统防幻觉。
正确表达：卡片升级前检查清晰、准确、必要；如果仍把最短路径说成边数最少，则弹窗驳回并说明原因。
```

## 5. 分段证据链

### 01. 先把问题说硬

本段目标：

```text
证明视频不是功能展示，而是从真实学习痛点出发。
```

用户操作 / 输入：

```text
打开 AXIOM Space。
准备创建新知识库。
学习背景设定为：计算机专业学生正在学数据结构。
当前问题设定为：图与最短路径。
```

系统处理：

```text
进入知识库创建流程。
准备建立一个新的学科级知识空间。
```

系统输出：

```text
新建知识库入口。
可以输入仓库名称。
页面没有预设内容。
```

正确校验：

```text
观众能看见这是从零开始。
主题明确是数据结构，不是泛泛学习。
问题明确是图与最短路径，不是泛泛 AI 辅导。
```

错误样子：

```text
一开始就进入已有复杂仓库，评委看不出从零开始。
只说“AI 学习助手”，没有具体课程和具体误区。
直接展示功能列表，用户痛点不成立。
```

自然说服点：

```text
用户为什么需要：因为资料多但方向不清。
为什么现在用：因为他当前就卡在图与最短路径。
```

A3 对应证据：

```text
学习资源繁杂无序、难以精准匹配自身需求。
以某一具体专业课程为切入点。
```

### 02. 新建学科仓库

本段目标：

```text
证明 AXIOM Space 是长期知识空间，不是一次性聊天。
```

用户操作 / 输入：

```text
创建知识库：数据结构。
```

系统处理：

```text
创建 vault。
初始化根节点。
初始化空图谱、空路径、空画像状态。
```

系统输出：

```text
空的《数据结构》仓库。
中心一级节点：数据结构。
卡片数量：0。
学习路径：0。
画像记录：待初始化。
```

正确校验：

```text
仓库名是学科或领域：数据结构。
不是“图与最短路径入门”这种一次任务。
中心节点代表长期学科空间。
```

错误样子：

```text
仓库名是一次任务，显得系统只能服务单次学习。
仓库创建后直接出现大量假数据，无法证明从零生长。
没有中心节点或仓库目标，图谱逻辑不清。
```

自然说服点：

```text
用户为什么信任：产品不是一次聊天，而是长期沉淀。
用户为什么用这个软件：它能承载课程长期学习过程。
```

A3 对应证据：

```text
个性化资源的自动化生成与建设。
全方位辅助学生开展自主学习。
```

### 03. 系统主动诊断

本段目标：

```text
证明系统不是被动聊天框，而是主动建立学习画像。
```

用户操作 / 输入：

```text
用户进入空仓库。
系统主动弹出普通对话。
用户回答当前学习情况：
我想学图与最短路径。
我知道图有点和边，也听过 Dijkstra。
我觉得最短路径应该是经过的边最少。
```

系统处理：

```text
抽取学习目标：图与最短路径。
抽取当前基础：知道点和边，听过 Dijkstra。
识别核心误区：把边数最少误认为最短路径。
推断解释偏好：需要图解、反例、代码。
初始化画像维度。
```

系统输出：

```text
普通引导会话：数据结构学习画像初始化。
核心误区提示：边数最少 ≠ 路径代价最小。
画像维度：
学习目标
当前基础
易错点
解释偏好
节奏负荷
掌握校验
```

正确校验：

```text
对话由系统主动发起，不是用户手动新建并自我介绍。
系统问的是必要问题，不是调查问卷。
画像不少于 6 个维度。
每个维度都来自用户回答或后续行为，不是空标签。
```

错误样子：

```text
用户必须自己找入口、新建对话、写自我介绍。
系统只回复一段解释，没有识别误区。
画像维度是空的，或者全是泛泛标签。
提问过多，像填表，打断学习。
```

自然说服点：

```text
用户为什么需要：学生经常不知道自己哪里没懂。
用户为什么信任：系统先诊断再行动，不急着给答案。
用户为什么用：普通 AI 等你问，AXIOM 主动发现卡点。
```

A3 对应证据：

```text
对话式学习画像自主构建。
摒弃传统繁琐表单。
不少于6个维度。
画像的随学随新。
```

### 04. 导入真实长资料

本段目标：

```text
证明系统生成有依据，不是大模型凭空讲课。
```

用户操作 / 输入：

```text
进入路径规划。
选择导入资料。
主题填写：图与最短路径。
上传文件：数据结构课程讲义-第6章-图.pdf。
或者粘贴长教材内容。
```

长文本应包含：

```text
图的基本概念
路径与连通性
带权图
路径长度
最短路径问题
Dijkstra 算法
算法适用条件
例题
```

系统处理：

```text
读取文件或文本。
把资料转换为 Markdown。
保存为文献资料节点。
抽取关键概念。
抽取灵感草稿。
生成学习路径。
同步知识图谱。
```

系统输出：

```text
Markdown 文献节点。
资料解析进度。
概念节点草案。
学习路径草案。
知识图谱更新提示。
```

正确校验：

```text
导入的是长文件或长文本，不是一句提示词。
页面能看见文件名或长文本滚动。
文献节点中能看到 Markdown 内容或附件嵌入。
后续路径和卡片能引用这份资料。
```

错误样子：

```text
只粘贴一句话，无法证明依托真实资料。
导入后没有文献节点，资料像丢进黑箱。
路径和资源没有来源，像裸 AI 生成。
上传失败但页面没有明确提示。
```

自然说服点：

```text
用户为什么信任：系统基于他的真实课程资料。
用户为什么现在用：他手里已经有讲义，马上能转成学习路径。
```

A3 对应证据：

```text
依托各类高等教育资源。
初始知识库/文档集。
生成进度追踪。
防幻觉的前置依据。
```

### 05. 生成学习路径

本段目标：

```text
证明系统解决了“资料很多但不知道先学什么”的问题。
```

用户操作 / 输入：

```text
点击导入资料并生成路径。
等待系统完成解析。
```

系统处理：

```text
根据资料结构拆分知识点。
结合画像误区确定学习顺序。
把概念组织成可执行路径。
给每一步绑定资料来源和理解卡。
```

系统输出：

```text
学习路径：
1. 图的点和边
2. 路径与连通性
3. 带权图与路径代价
4. 最短路径到底“短”在哪里
5. Dijkstra 的选择过程
6. Dijkstra 的适用边界

每一步显示：
为什么学这一步
对应资料来源
当前状态
进入理解卡
下一步
```

正确校验：

```text
路径顺序符合从概念到算法的学习逻辑。
当前误区“边数 vs 权重总和”被放在关键步骤。
每一步都能进入具体卡片或任务。
路径不是孤立列表，而是能继续推进。
```

错误样子：

```text
路径只是教材目录复制，没有个性化。
顺序混乱，先讲 Dijkstra 再讲边权。
没有任务状态和下一步。
没有资料来源。
```

自然说服点：

```text
用户为什么需要：他不是缺解释，而是缺学习顺序。
用户为什么用：系统把资料变成可执行路径。
```

A3 对应证据：

```text
个性化学习路径规划和资源推送。
明确学习步骤和顺序。
结合学习进度、知识掌握情况及学习偏好的深度分析。
```

### 06. 打开对准误区的理解卡

本段目标：

```text
证明学习不是泛泛对话，而是围绕具体误区修正。
```

用户操作 / 输入：

```text
点击任务：最短路径到底“短”在哪里。
进入 AI 工作台。
```

系统处理：

```text
创建或打开绑定任务的理解卡。
把用户误区写入卡片。
把资料依据写入卡片。
把当前待解决问题写入卡片。
```

系统输出：

```markdown
# 最短路径不是边数最少

## 学生当前误区
我以为最短路径就是经过边数最少。

## 来自资料的依据
在带权图中，路径长度通常指路径上各边权值之和。

## 当前要解决的问题
为什么一条边不一定比三条边更短？
```

正确校验：

```text
卡片标题直接对准误区。
卡片包含资料依据。
卡片不是空白笔记，也不是泛泛摘要。
卡片会话和右侧编辑内容围绕同一问题。
```

错误样子：

```text
卡片标题太泛，如“图论基础”。
卡片没有误区字段，评委看不出针对性。
没有资料依据，无法证明不是凭空生成。
卡片和对话主题不一致。
```

自然说服点：

```text
用户为什么信任：系统承认并记录误区，不假装用户已经懂。
用户为什么用：学习过程被落到一张可修改、可保存的卡片上。
```

A3 对应证据：

```text
智能辅导。
Markdown 渲染。
个性化学习引导。
```

### 07. 费曼输出与自动记录

本段目标：

```text
证明系统不是替学生学习，而是促使学生真正表达理解。
```

用户操作 / 输入：

```text
用户根据 Agent 提问，用自己的话解释：
为什么一条边不一定比三条边更短？
```

用户示例回答：

```text
因为每条边有不同权重。
如果一条边权重是 100，而三条边每条权重是 10，
那三条边总代价是 30，比一条边更短。
所以最短路径比较的是权重总和，不是边数。
```

系统处理：

```text
识别用户是否说出了关键概念：权重、总代价、边数不等于最短。
提取用户自己的解释。
写入理解卡。
生成观察记录。
更新画像中的易错点和掌握证据。
自动保存卡片。
```

系统输出：

```markdown
## 我的理解

最短路径比较的是路径总代价。
边数少不一定总代价低，边数多也可能总代价更低。
```

同时显示：

```text
自动保存成功。
观察记录已更新。
画像证据已新增。
```

正确校验：

```text
必须有用户自己的话。
必须写入卡片。
必须自动保存或明确保存状态。
必须生成观察记录或画像变化。
```

错误样子：

```text
AI 讲了一大段，用户没有输出。
用户输出没有进入卡片。
卡片需要手动保存，焦点移走后丢内容。
画像没有变化，看不出随学随新。
```

自然说服点：

```text
用户为什么信任：系统不骗他说“看过就是懂”。
用户为什么用：它能把一次解释变成长期学习证据。
```

A3 对应证据：

```text
智能辅导。
实时跟踪学生学习行为。
画像的随学随新。
学习效果评估的前置证据。
```

### 08. 针对卡点生成资源

本段目标：

```text
证明资源生成的价值在于精准，而不是数量。
```

用户操作 / 输入：

```text
用户请求：
请基于这张卡和刚导入的讲义，生成适合我补这个误区的学习资源。
```

系统处理：

```text
读取当前卡片。
检索导入资料。
读取画像误区。
确定资源目标：用反例、图解、练习、代码巩固路径总代价。
调用资源生成链路。
校验清晰、准确、必要。
```

系统输出：

```text
生成依据：
当前卡片：最短路径不是边数最少
学生误区：混淆边数和权重总和
资料来源：数据结构课程讲义-第6章-图
推荐目标：巩固路径总代价

资源：
1. 讲解文档：边权、路径代价和最短路径
2. 思维导图：图、路径、边权、最短路径、Dijkstra 的关系
3. 练习题：判断多条路径哪条总代价最小
4. 代码实操：用 Python 表示带权图并计算路径代价
5. 视频/动画脚本：Dijkstra 如何逐步选择当前代价最小的节点
```

正确校验：

```text
至少 5 类资源。
每类资源都能解释为什么对应当前误区。
资源引用当前卡片或导入资料。
资源以卡片形式展示。
生成过程有进度。
```

错误样子：

```text
只生成一段讲解文本。
资源很多但和“边数 vs 权重”无关。
没有资料引用，像通用 AI 结果。
没有进度，生成时长像卡死。
资源不能打开或预览。
```

自然说服点：

```text
用户为什么需要：资源不精准比没有资源更浪费时间。
用户为什么用：系统围绕自己的短板生成资源。
```

A3 对应证据：

```text
多智能体协同的资源生成。
至少5种类型的个性化资源生成。
多模态学习资料。
资源推送精准化。
```

### 10. 推送箱说明下一步为什么是它

本段目标：

```text
证明系统能动态决定下一步，而不是固定课程表。
```

用户操作 / 输入：

```text
用户完成当前卡片学习。
返回路径规划。
打开推送箱。
```

系统处理：

```text
读取当前卡片内容。
读取评估结果。
读取导入资料中的算法适用条件。
判断需要新增连接和任务组。
生成推荐理由和证据来源。
```

系统输出：

连接推送：

```text
建议连接：边权 -> 最短路径
理由：学生已经能说出路径总代价，应建立显式概念关系。
证据：卡片中出现“最短路径比较的是权重总和”。
```

资源与任务推送：

```text
建议任务组：Dijkstra 的适用边界
理由：学生已理解非负权重下的路径代价，下一步需要知道为什么负权边会影响 Dijkstra。
证据：导入资料中出现“Dijkstra 适用于非负权图”。
```

正确校验：

```text
推送分为连接推送、资源与任务。
每条推送都有理由。
每条推送都有证据来源。
用户可以执行、忽略或稍后处理。
```

错误样子：

```text
只有推荐标题，没有推荐理由。
推荐和当前学习状态无关。
推送不能执行。
推送过多，打断学习。
```

自然说服点：

```text
用户为什么需要：学完一步后不知道下一步补什么。
用户为什么用：系统给出有证据的下一步建议。
```

A3 对应证据：

```text
基于画像实现学习资源的精准推送。
动态调整学习资源推送策略和学习计划。
个性化学习路径规划。
```

### 11. 掌握评估

本段目标：

```text
证明系统不会把“看过资料”误认为“已经掌握”。
```

用户操作 / 输入：

```text
点击完成 / 评估掌握。
回答系统问题：
为什么带权图里的最短路径不是边数最少？
请给出一个反例。
```

用户合格回答：

```text
因为最短路径比较的是路径上所有边权的总和。
比如 A 到 B 直接边权是 100，
但 A 到 C 到 D 到 B 三条边分别是 10、10、10，
三条边的总代价是 30，所以三条边这条路线更短。
```

系统处理：

```text
检查是否区分边数和权重总和。
检查是否给出反例。
检查是否用自己的话表达。
检查是否与资料依据一致。
生成评估结论。
更新学习路径下一步。
```

系统输出：

```text
掌握判断：通过

通过原因：
1. 能区分边数和权重总和。
2. 能给出具体反例。
3. 能用自己的话解释，不依赖教材原句。

下一步：
学习 Dijkstra 如何选择当前代价最小的节点。
```

正确校验：

```text
有学生输出。
有明确评估标准。
有通过或不通过结论。
有下一步路径变化。
结论能被用户回答支撑。
```

错误样子：

```text
点一下完成就算掌握。
评估只显示分数，没有理由。
用户答错也通过。
没有下一步调整。
```

自然说服点：

```text
用户为什么信任：系统用证据判断，不靠感觉。
用户为什么现在用：马上知道自己是真懂还是假懂。
```

A3 对应证据：

```text
学习效果评估。
实时跟踪学习行为。
多维度、精准评估。
动态调整学习计划。
```

### 12. 质量校验与防幻觉

本段目标：

```text
证明错误内容不会直接进入长期知识库。
```

用户操作 / 输入：

```text
点击升级为永久知识。
```

系统处理：

```text
检查卡片是否清晰。
检查卡片是否准确。
检查卡片是否必要。
对照导入资料和学生评估记录。
决定通过或驳回。
```

系统输出：

不合格时：

```text
暂不能升级为永久知识。

问题：
1. 清晰性不足：没有解释“路径总代价”。
2. 准确性不足：仍出现“最短路径就是边数最少”的表述。
3. 必要性不足：缺少反例支撑。

建议修改：
补充带权图反例，并明确“最短”指权重总和最小。
```

合格时：

```text
已升级为永久知识。
依据：导入讲义 + 学生解释 + 评估通过记录。
```

正确校验：

```text
升级前有质量判断。
驳回必须说明具体原因。
通过必须说明依据。
质量标准能对应清晰、准确、必要。
```

错误样子：

```text
所有卡片都能一键升级。
只说“不符合要求”，不说哪里错。
错误概念进入永久知识。
没有资料依据。
```

自然说服点：

```text
用户为什么信任：系统不把错的东西长期保存。
用户为什么用：它让知识沉淀有边界。
```

A3 对应证据：

```text
防幻觉与内容安全过滤机制。
确保生成的学术内容无事实性错误。
```

## 8. 最终判断标准

正确的视频效果：

```text
评委看完后会说：
这个系统不是功能堆叠，而是有学习闭环。
它知道学生哪里错。
它基于真实资料生成路径。
它围绕误区生成资源。
它要求学生输出，并判断是否掌握。
它把结果沉淀为长期知识。
```

错误的视频效果：

```text
评委看完后只记得：
页面很多。
按钮很多。
AI 能生成很多东西。
```

最终必须回答三个问题：

```text
解决了什么？
学生不知道怎么学、资源不精准、掌握不可验证。

强在哪里？
主动画像、资料依据、路径规划、精准资源、输出评估、知识沉淀。

凭什么得分？
A3 要求的画像、资源生成、路径推送、智能辅导、效果评估、防幻觉，都有页面证据。
```
