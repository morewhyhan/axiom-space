# AXIOM Space A3 演示视频脚本 第七版：整合证据版

## 1. 文档定位

这一版把第四版、第五版、第六版重新整合成一个最终工作稿：

```text
第四版的作用：让表达更简洁，少讲功能，多讲价值。
第五版的作用：让表达更硬，所有亮点都要有页面证据。
第六版的作用：把每一段拆成输入、处理、输出、校验和错误样子。
第七版的作用：把三者合并，并补上“用户为什么信任我/我们”的人设与专业理念。
```

这份文档既可以指导视频录制，也可以反向指导页面调整。  
如果实际页面和这份脚本冲突，优先让页面向这个证据链靠拢。

## 2. 最终成果价值

AXIOM Space 的价值不是“做了很多 AI 功能”，而是解决了学习过程里的三个断点：

```text
1. 学生不知道自己该怎么学。
2. 学习资源很多，但不精准、不成路径。
3. 学生看过资料，却无法证明自己真的掌握。
```

所以视频要证明的闭环是：

```text
学生原来卡住了
→ 系统主动识别卡点
→ 基于真实资料生成路径
→ 针对卡点生成资源
→ 逼学生用自己的话输出
→ 系统评估是否掌握
→ 合格知识沉淀进图谱和画像
```

最终硬表达：

> AXIOM Space 把 AI 从“给答案的工具”，推进成“能建画像、排路径、推资源、促输出、做评估、沉淀知识”的可验证学习系统。

## 3. 用户为什么信任我/我们

这里不是讲产品功能，而是讲“我是什么样的人，我相信学习应该怎么做”。

不要生硬地说：

```text
我很专业，所以你应该相信我。
```

而要通过产品设计让评委感受到：

```text
这个人懂学生真正的学习困难。
这个人懂 AI 的能力，也懂 AI 的风险。
这个人不是为了炫技，而是在认真解决学习问题。
这个人重视证据、边界、长期积累和真实掌握。
```

### 3.1 我懂学生的真实困难

我不把学生的问题简化成“缺资料”。  
真实情况是：资料很多，AI 回答很多，但学生不知道自己哪里没懂。

页面证据：

```text
学生误区：把边数最少当成最短路径。
系统主动诊断，而不是直接讲答案。
路径从误区出发，而不是复制教材目录。
```

### 3.2 我懂 AI 不能凭空生成

我不相信“AI 多生成一点”就等于学习资源更好。  
我更相信：生成必须有依据、有引用、有校验。

页面证据：

```text
导入真实长资料。
生成 Markdown 文献节点。
资源生成显示资料来源。
卡片升级前检查清晰、准确、必要。
```

### 3.3 我相信学习必须让学生输出

我不把“看过解释”当成“已经掌握”。  
学生必须能用自己的话解释，系统才有资格判断他是否理解。

页面证据：

```text
费曼追问。
学生自己的解释进入卡片。
掌握评估要求反例。
评估结论说明通过原因。
```

### 3.4 我重视长期知识，而不是一次聊天

我不希望 AI 对话用完就消失。  
一次学习应该沉淀成卡片、资源、图谱、画像和下一步计划。

页面证据：

```text
学科仓库：数据结构。
知识图谱前后变化。
永久知识卡。
画像随学随新。
下一步学习建议。
```

### 3.5 我敢给系统设置边界

真正专业的系统不是永远说“可以”，而是在内容不合格时敢拒绝。

页面证据：

```text
升级永久知识时，如果仍把最短路径说成边数最少，系统驳回。
驳回理由明确到清晰性、准确性、必要性。
修改后才允许沉淀。
```

这一部分的核心：

```text
用户信任我们，不是因为我们说自己专业。
而是因为系统的每个设计，都体现出专业判断：
先诊断，再生成；
先依据，再结论；
先输出，再评估；
先校验，再沉淀。
```

这四个维度要自然藏在视频里，不要变成口号。

### 4.1 用户为什么有这个需求？

因为学生真实面对的是：

```text
课程资料多。
AI 回答多。
但学习顺序不清。
自己误区不自知。
看过不等于会。
```

视频对应画面：

```text
数据结构空仓库。
图与最短路径误区。
真实长资料导入。
学习路径生成。
掌握评估。
```

### 4.2 用户为什么信任我/我们？

因为这套系统背后体现的是一种专业学习观：

```text
不凭空生成。
不盲目鼓励。
不把错误沉淀为知识。
不把看过当掌握。
不做一次性聊天，而做长期学习系统。
```

视频对应画面：

```text
真实资料引用。
费曼输出。
质量驳回。
评估标准。
图谱和画像沉淀。
```

### 4.3 用户为什么用这个软件？

因为普通工具只能解决单点问题：

```text
聊天工具能回答，但不一定诊断。
资料工具能整理，但不一定规划。
笔记工具能保存，但不一定评估。
题库工具能练习，但不一定沉淀画像。
```

AXIOM Space 做的是闭环：

```text
诊断 → 路径 → 资源 → 辅导 → 输出 → 评估 → 沉淀
```

视频对应画面：

```text
主动画像。
路径规划。
资源生成。
推送箱。
掌握评估。
知识图谱。
认知洞察。
```

### 4.4 用户为什么现在就要用？

因为它不是远期愿景，而是能从当前课程资料立刻开始：

```text
用户现在正在学数据结构。
用户现在手里有讲义。
用户现在就卡在图与最短路径。
用户现在就需要知道下一步学什么。
```

视频对应画面：

```text
数据结构课程讲义。
当前主题：图与最短路径。
当前误区：边数最少 ≠ 最短路径。
下一步：Dijkstra 的选择过程和适用边界。
```

## 5. 视频主线

```text
1. 先把问题说硬：学生资料很多，但不知道哪里没懂。
2. 新建学科仓库：数据结构。
3. 系统主动诊断：识别“边数最少 ≠ 最短路径”。
4. 导入真实长资料：课程讲义进入 Markdown 文献节点。
5. 生成学习路径：从概念到算法，步骤清楚。
6. 打磨理解卡：围绕误区做费曼输出。
7. 生成个性化资源：资源围绕卡点，不是堆材料。
8. 展示多智能体协同：角色分工有产物。
9. 推送下一步：连接和任务都有理由、有证据。
10. 做掌握评估：有输出、有标准、有结论。
11. 质量校验：不合格内容不能升级为永久知识。
12. 展示沉淀结果：图谱、画像、下一步全部更新。
```

## 6. 正式脚本与证据链

### 01. 先把问题说硬

旁白方向：

```text
很多学生学《数据结构》时，最大的问题不是没有资料。
课件、教材、网课、AI 回答都有，但资料越多，反而越不知道先学什么、哪里没懂、自己到底会没会。

这次我们只演示一个具体问题：
一个学生学“图与最短路径”，他以为最短路径就是边数最少。
AXIOM Space 要做的，是把这个隐藏误区识别出来，并带他完成一次可验证的掌握学习闭环。
```

用户操作 / 输入：

```text
打开 AXIOM Space。
准备创建知识库。
当前课程：数据结构。
当前问题：图与最短路径。
```

系统处理：

```text
进入知识库创建流程。
等待用户创建学科级仓库。
```

系统输出：

```text
知识库创建入口。
空白起点。
```

正确校验：

```text
观众能看见从零开始。
课程和问题都具体。
痛点不是泛泛“学习困难”，而是“资料多但误区不自知”。
```

错误样子：

```text
开头直接展示复杂功能。
没有具体课程。
没有具体误区。
评委不知道用户为什么需要这个软件。
```

A3 证据：

```text
学习资源繁杂无序、难以精准匹配自身需求。
以某一具体专业课程为切入点。
```

### 02. 新建学科仓库

旁白方向：

```text
这里新建的不是一次临时对话，而是《数据结构》这门课的长期知识库。
后续课程资料、理解卡、资源、评估结果和知识图谱都会沉淀在这里。
```

用户操作 / 输入：

```text
创建知识库：数据结构。
```

系统处理：

```text
创建 vault。
初始化根节点。
初始化空图谱、空路径、空画像。
```

系统输出：

```text
空的《数据结构》仓库。
中心一级节点：数据结构。
卡片数量：0。
学习路径：0。
画像记录：待初始化。
```

正确校验：

```text
仓库名是学科或领域。
中心节点代表长期学习空间。
能看出从零开始。
```

错误样子：

```text
仓库名是“图与最短路径入门”这种一次任务。
创建后直接出现大量假数据。
没有中心节点，仓库目标不清。
```

信任暗线：

```text
我不是做一次性聊天，而是做长期学习系统。
```

A3 证据：

```text
个性化资源的自动化生成与建设。
全方位辅助学生开展自主学习。
```

### 03. 系统主动诊断

旁白方向：

```text
普通 AI 等学生提问，AXIOM Space 会先主动诊断。
它不让学生填写复杂表单，而是通过几句自然语言，识别学习目标、已有基础和关键误区。
```

用户操作 / 输入：

系统主动弹出普通对话：

```text
你现在最想解决数据结构里的哪个知识点？
你已经知道什么？
你最容易卡在哪里？
你更适合文字、图解、代码，还是练习题？
```

用户回答：

```text
我想学图与最短路径。
我知道图有点和边，也听过 Dijkstra。
但我分不清路径、边权和最短路径。
我觉得最短路径应该就是经过的边最少。
```

系统处理：

```text
抽取学习目标：图与最短路径。
抽取当前基础：知道点和边，听过 Dijkstra。
识别核心误区：把边数最少误认为最短路径。
推断解释偏好：图解、反例、代码。
初始化画像维度。
```

系统输出：

```text
已识别核心误区：
学生把“边数最少”误认为“路径代价最小”。

画像维度：
学习目标
当前基础
易错点
解释偏好
节奏负荷
掌握校验
```

正确校验：

```text
对话由系统主动发起。
问题数量少，只问必要问题。
画像不少于 6 个维度。
核心误区明确显示。
```

错误样子：

```text
用户必须自己新建对话并自我介绍。
系统只给解释，没有诊断误区。
画像只有空标签。
提问太多，像填表。
```

信任暗线：

```text
我懂学生经常不知道自己哪里没懂。
所以系统先诊断，不急着生成答案。
```

A3 证据：

```text
对话式学习画像自主构建。
摒弃传统繁琐表单。
不少于6个维度。
画像的随学随新。
```

### 04. 导入真实长资料

旁白方向：

```text
系统不是让大模型凭空讲课，而是先接收学生真实使用的课程资料。
资料会被保存为 Markdown 文献节点，后面的路径、卡片和资源都要能回到这份资料。
```

用户操作 / 输入：

```text
进入路径规划。
选择导入资料。
主题填写：图与最短路径。
上传文件：数据结构课程讲义-第6章-图.pdf。
或者粘贴长教材内容。
```

资料内容应该包含：

```text
图的基本概念
路径与连通性
带权图
路径长度
最短路径问题
Dijkstra 算法
算法适用条件
例题
```

系统处理：

```text
读取文件或文本。
转换为 Markdown。
保存为文献资料节点。
抽取关键概念。
抽取灵感草稿。
生成学习路径。
同步知识图谱。
```

系统输出：

```text
原始文件名。
Markdown 文献节点。
资料解析进度。
引用来源。
```

正确校验：

```text
导入的是长文件或长文本。
能看到文件名或文本滚动。
能看到 Markdown 文献节点。
后续路径和卡片能引用这份资料。
```

错误样子：

```text
只粘贴一句话。
导入后没有文献节点。
后续资源没有来源。
上传失败没有提示。
```

信任暗线：

```text
我懂 AI 的风险。
所以系统不凭空生成，先建立资料依据。
```

A3 证据：

```text
依托各类高等教育资源。
初始知识库/文档集。
生成进度追踪。
防幻觉的前置依据。
```

### 05. 生成学习路径

旁白方向：

```text
学生真正缺的不是又一段解释，而是学习顺序。
AXIOM Space 把一份长资料拆成可执行路径，告诉学生先学什么、为什么先学、下一步是什么。
```

用户操作 / 输入：

```text
点击导入资料并生成路径。
等待系统完成解析。
```

系统处理：

```text
根据资料结构拆分知识点。
结合画像误区确定学习顺序。
把概念组织成可执行路径。
给每一步绑定资料来源和理解卡。
```

系统输出：

```text
学习路径：
1. 图的点和边
2. 路径与连通性
3. 带权图与路径代价
4. 最短路径到底“短”在哪里
5. Dijkstra 的选择过程
6. Dijkstra 的适用边界

每一步显示：
为什么学这一步
对应资料来源
当前状态
进入理解卡
下一步
```

正确校验：

```text
顺序从基础概念到算法。
核心误区被放在关键步骤。
每一步能进入具体卡片或任务。
路径可继续推进。
```

错误样子：

```text
路径只是教材目录复制。
先讲 Dijkstra，再讲边权，顺序混乱。
没有任务状态。
没有资料来源。
```

信任暗线：

```text
我懂学习不是堆资料，而是安排认知顺序。
```

A3 证据：

```text
个性化学习路径规划和资源推送。
明确学习步骤和顺序。
结合学习进度、知识掌握情况及学习偏好的深度分析。
```

### 06. 打开对准误区的理解卡

旁白方向：

```text
这一张卡直接对准学生的误区。
它不是普通笔记，而是一次学习诊断后的修正对象：学生错在哪里、资料依据是什么、现在要解释清楚什么。
```

用户操作 / 输入：

```text
点击任务：最短路径到底“短”在哪里。
进入 AI 工作台。
```

系统处理：

```text
创建或打开绑定任务的理解卡。
写入用户误区。
写入资料依据。
写入当前待解决问题。
```

系统输出：

```markdown
# 最短路径不是边数最少

## 学生当前误区
我以为最短路径就是经过边数最少。

## 来自资料的依据
在带权图中，路径长度通常指路径上各边权值之和。

## 当前要解决的问题
为什么一条边不一定比三条边更短？
```

正确校验：

```text
卡片标题直接对准误区。
卡片包含资料依据。
卡片不是空白笔记。
卡片会话和编辑内容围绕同一问题。
```

错误样子：

```text
卡片标题太泛。
没有误区字段。
没有资料依据。
卡片和对话主题不一致。
```

信任暗线：

```text
我不假装学生已经懂了。
我把误区摆到页面上，让学习真正开始。
```

A3 证据：

```text
智能辅导。
Markdown 渲染。
个性化学习引导。
```

### 07. 费曼输出与自动记录

旁白方向：

```text
这里不是 AI 讲得更完整，而是学生自己说清楚了。
AXIOM Space 用费曼学习法逼出学生的表达，并把它保存为学习证据。
```

用户操作 / 输入：

Agent 提问：

```text
先不要背定义。请你用自己的话解释：
为什么一条边不一定比三条边更短？
```

用户回答：

```text
因为每条边有不同权重。
如果一条边权重是 100，而三条边每条权重是 10，
那三条边总代价是 30，比一条边更短。
所以最短路径比较的是权重总和，不是边数。
```

系统处理：

```text
识别关键概念：权重、总代价、边数不等于最短。
提取用户自己的解释。
写入理解卡。
生成观察记录。
更新画像证据。
自动保存卡片。
```

系统输出：

```markdown
## 我的理解

最短路径比较的是路径总代价。
边数少不一定总代价低，边数多也可能总代价更低。
```

同时显示：

```text
自动保存成功。
观察记录已更新。
画像证据已新增。
```

正确校验：

```text
必须有用户自己的话。
必须写入卡片。
必须自动保存。
必须生成观察记录或画像变化。
```

错误样子：

```text
AI 长篇讲解，用户没有输出。
```

### 11. 掌握评估

旁白方向：

```text
AXIOM Space 不把“看过资料”当成学会。
它要求学生输出解释，并用明确标准判断是否掌握。
只有评估通过，路径才进入下一步。
```

用户操作 / 输入：

```text
点击完成 / 评估掌握。
回答问题：
为什么带权图里的最短路径不是边数最少？
请给出一个反例。
```

用户合格回答：

```text
因为最短路径比较的是路径上所有边权的总和。
比如 A 到 B 直接边权是 100，
但 A 到 C 到 D 到 B 三条边分别是 10、10、10，
三条边的总代价是 30，所以三条边这条路线更短。
```

系统处理：

```text
检查是否区分边数和权重总和。
检查是否给出反例。
检查是否用自己的话表达。
检查是否与资料依据一致。
生成评估结论。
更新学习路径下一步。
```

系统输出：

```text
掌握判断：通过

通过原因：
1. 能区分边数和权重总和。
2. 能给出具体反例。
3. 能用自己的话解释，不依赖教材原句。

下一步：
学习 Dijkstra 如何选择当前代价最小的节点。
```

正确校验：

```text
有学生输出。
有明确评估标准。
有通过或不通过结论。
有下一步路径变化。
结论能被用户回答支撑。
```

错误样子：

```text
点一下完成就算掌握。
评估只有分数，没有理由。
用户答错也通过。
没有下一步调整。
```

信任暗线：

```text
我不靠感觉判断学习效果。
我要求证据、标准和结论同时出现。
```

A3 证据：

```text
学习效果评估。
实时跟踪学习行为。
多维度、精准评估。
动态调整学习计划。
```

### 14. 结尾收束为价值

## 9. 最终判断标准

正确的视频效果：

```text
评委看完后会说：
这个系统不是功能堆叠，而是有学习闭环。
它知道学生哪里错。
它基于真实资料生成路径。
它围绕误区生成资源。
它要求学生输出，并判断是否掌握。
它把结果沉淀为长期知识。
```

错误的视频效果：

```text
评委看完后只记得：
页面很多。
按钮很多。
AI 能生成很多东西。
```

最终必须回答三个问题：

```text
解决了什么？
学生不知道怎么学、资源不精准、掌握不可验证。

强在哪里？
主动画像、资料依据、路径规划、精准资源、输出评估、知识沉淀。

凭什么得分？
A3 要求的画像、资源生成、路径推送、智能辅导、效果评估、防幻觉，都有页面证据。
```
