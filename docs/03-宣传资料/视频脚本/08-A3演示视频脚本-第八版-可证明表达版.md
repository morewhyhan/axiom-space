# AXIOM Space A3 演示视频脚本 第八版：可证明表达版

## 1. 文档定位

这版不是继续扩写脚本，而是给第七版做一次语言收紧。

核心要求：

```text
每句话都必须能回答：
它在视频里怎么被使用？
页面上能不能看见？
评委能不能判断它是真的？
```

如果一句话不能被页面证明，它就不能作为强表达出现。  
它最多只能作为背景理解、旁白弱表达，或者直接删掉。

这版采用一个简单规则：

```text
不要说“我们很强”。
要让页面证明“我们解决了什么”。
```

## 2. 语言规范

### 2.1 一句话必须落到可观察动作

不合格：

```text
系统能实现个性化学习。
```

合格：

```text
系统识别学生把“边数最少”误认为“最短路径”，
并生成围绕这个误区的学习路径和资源。
```

原因：

```text
“个性化学习”太大。
“识别误区、生成路径、生成资源”可以在页面上看见。
```

### 2.2 抽象词必须接证据

这些词不能裸用：

```text
专业
精准
智能
长期
可验证
真实掌握
因材施教
防幻觉
多智能体
```

使用方式：

```text
说“精准”时，必须展示资源为什么对应当前误区。
说“可验证”时，必须展示用户输出、评估标准、评估结论。
说“防幻觉”时，必须展示资料依据、校验结果、驳回原因。
说“多智能体”时，必须展示不同 Agent 的输入、输出和产物。
```

### 2.3 后台逻辑不是页面证据

这些表述本身不够：

```text
系统分析了用户画像。
系统判断了学习状态。
系统检查了准确性。
系统动态调整了路径。
```

必须变成页面证据：

```text
画像维度显示：学习目标、当前基础、易错点、解释偏好、节奏负荷、掌握校验。
评估面板显示：通过原因、失败原因、下一步建议。
推送箱显示：推荐理由、证据来源、执行入口。
质量弹窗显示：清晰性、准确性、必要性的问题。
```

### 2.4 不说无法验证的“人格判断”

不合格：

```text
我是一个专业的人。
我懂学生。
我做得很认真。
```

合格：

```text
系统没有直接讲答案，而是先识别学生误区。
系统没有凭空生成，而是先导入真实讲义。
系统没有把看过当掌握，而是要求学生给出反例。
系统没有让错误卡片升级，而是指出具体错误。
```

这才是在页面上能证明的“专业”。

## 3. 最终价值表达

视频最终要让评委记住这一句话：

> AXIOM Space 不是一个会多生成资料的 AI 工具，而是一个能把“学习卡点”变成“路径、资源、输出、评估和知识沉淀”的学习系统。

这个价值必须由页面证明：

```text
卡点：学生把边数最少误认为最短路径。
路径：系统生成从图、路径、边权到 Dijkstra 的步骤。
资源：系统生成文档、导图、练习、代码、动画。
输出：学生用自己的话解释路径总代价。
评估：系统判断他是否能区分边数和权重总和。
沉淀：知识进入图谱，画像记录发生变化。
```

不要再强调：

```text
我们功能很多。
我们页面很多。
我们 AI 很强。
```

要强调：

```text
学生原来哪里错。
系统怎么发现。
系统给了什么下一步。
学生怎么证明自己懂了。
系统怎么防止错误进入长期知识库。
```

## 4. 强表达与弱表达

### 4.1 可以强说的内容

只有页面可见时才能强说：

```text
系统主动发起学习画像对话。
系统识别学生的具体误区。
系统导入真实长资料并保存为文献节点。
系统生成学习路径。
系统围绕当前卡片生成资源。
系统要求学生用自己的话解释。
系统给出掌握评估结论。
系统在卡片升级前做质量校验。
系统把结果沉淀到知识图谱和认知画像。
```

### 4.2 只能弱说的内容

如果页面没有明确展示，只能谨慎表达：

```text
多智能体协同。
防幻觉机制。
动态调整学习计划。
精准推送。
画像随学随新。
```

这些必须补页面证据后才能强说。

### 4.3 不应该说的内容

```text
完全解决所有学习问题。
百分百防幻觉。
自动保证掌握。
比所有学习软件都强。
用户一定会因此信任我们。
```

这些不可证明，也容易扣分。

## 5. 需要页面证明的核心链路

这一条链路必须完整：

```text
用户输入：
我想学图与最短路径，我以为最短路径就是边数最少。

系统处理：
识别误区：边数最少 ≠ 权重总和最小。

系统输出：
画像、路径、理解卡、资源、评估、图谱。

校验：
学生能否用自己的话解释，并给出反例。

错误拦截：
如果卡片仍把最短路径写成边数最少，则不能升级为永久知识。
```

只要这条链路完整，视频就有说服力。

### 5.1 画像闭环必须这样证明

不能只说：

```text
画像更新后，系统会更懂用户。
```

必须让页面显示这条因果链：

```text
用户输出：
学生用自己的话说明“三条低权重边可能比一条高权重边更短”。

触发证据：
卡片自动写入“最短路径比较的是路径总代价”。

画像变化：
易错点从“混淆边数和权重总和”
变成“需要继续理解 Dijkstra 的选择过程和适用边界”。

资源变化：
后续资源不再只讲边数和权重，
而是增加 Dijkstra 适用边界、非负权重、反例练习。

推送变化：
推送箱推荐“Dijkstra 的适用边界”，
并在推荐理由里引用画像字段“剩余缺口”。
```

正确标准：

```text
画像前后变化能看见。
变化原因能追溯到用户自己的输出。
后续资源和推送都引用了变化后的画像字段。
评委能判断：不是系统随便推荐，而是用户状态改变后推荐改变。
```

错误样子：

```text
只显示“画像已更新”。
画像变化后，资源内容没有变化。
推送箱只有标题，没有引用画像字段。
推荐理由和用户刚刚的学习表现无关。
```

## 6. 分段脚本与可证明标准

### 01. 痛点开场

可说：

```text
学生不是没有资料，而是不知道哪里没懂。
这次我们演示一个具体误区：他以为最短路径就是边数最少。
```

页面证据：

```text
课程：数据结构。
主题：图与最短路径。
学生输入或系统记录：最短路径是不是边数最少？
```

不要说：

```text
所有学生都存在这个问题。
系统能解决所有学习困难。
```

正确标准：

```text
评委能在 15 秒内知道：
用户是谁、学什么、卡在哪里。
```

错误样子：

```text
开头只讲愿景。
没有具体学生。
没有具体误区。
```

### 02. 新建学科仓库

可说：

```text
这里创建的是《数据结构》长期知识库，不是一次临时聊天。
```

页面证据：

```text
仓库名称：数据结构。
空仓库状态。
中心节点：数据结构。
卡片数量、路径数量初始为空。
```

不要说：

```text
这里已经形成完整知识体系。
```

因为刚创建时它还没有形成体系。

正确标准：

```text
能看出这是从零开始。
仓库名是学科级，不是一次任务名。
```

错误样子：

```text
仓库叫“图与最短路径入门”。
页面已经有大量节点，不能证明从零生长。
```

### 03. 系统主动诊断

可说：

```text
系统主动发起对话，先问学习目标、已有基础和当前卡点。
```

页面证据：

```text
普通引导会话自动出现。
系统问题不超过 3-4 个。
用户回答里出现误区：最短路径是边数最少。
系统显示识别结果。
```

可证明输出：

```text
学习目标：图与最短路径。
当前基础：知道点和边。
易错点：混淆边数和权重总和。
解释偏好：图解、反例、代码。
节奏负荷：小步推进。
掌握校验：能用自己的话解释并给出反例。
```

不要说：

```text
系统完全理解了学生。
```

要说：

```text
系统识别出本次学习所需的几个关键维度。
```

正确标准：

```text
画像维度能追溯到用户回答。
核心误区明确显示。
```

错误样子：

```text
系统只回复“好的，我来讲解”。
用户要自己创建会话。
画像是空标签。
```

### 04. 导入真实资料

可说：

```text
系统先接收真实课程资料，再生成路径和资源。
```

页面证据：

```text
上传文件名：数据结构课程讲义-第6章-图.pdf。
或粘贴长文本。
文本中能看到章节、定义、例题、Dijkstra。
生成 Markdown 文献节点。
```

不要说：

```text
系统所有回答都绝对正确。
```

要说：

```text
后续生成有资料依据，可以追溯到导入文献。
```

正确标准：

```text
资料不是一句 prompt。
导入后有文献节点。
路径或卡片能显示资料来源。
```

错误样子：

```text
只粘贴一句“帮我学最短路径”。
没有文献节点。
后续资源没有引用。
```

### 05. 生成学习路径

可说：

```text
系统把长资料拆成可执行学习路径，让学生知道先学什么、后学什么。
```

页面证据：

```text
学习路径包含：
图的点和边
路径与连通性
带权图与路径代价
最短路径到底短在哪里
Dijkstra 的选择过程
Dijkstra 的适用边界
```

必须补充的证据：

```text
每一步为什么存在。
每一步来自哪段资料。
当前任务是什么。
下一步是什么。
```

不要说：

```text
系统自动生成最科学路径。
```

要说：

```text
系统生成一条符合当前误区和资料结构的学习路径。
```

正确标准：

```text
路径顺序从基础概念到算法。
误区被放在关键步骤。
步骤可执行。
```

错误样子：

```text
路径只是教材目录。
先讲算法再讲边权。
没有任务状态。
```

### 06. 打开理解卡

可说：

```text
系统把学生误区落到一张可编辑、可追问、可沉淀的理解卡上。
```

页面证据：

```markdown
# 最短路径不是边数最少

## 学生当前误区
我以为最短路径就是经过边数最少。

## 来自资料的依据
在带权图中，路径长度通常指路径上各边权值之和。

## 当前要解决的问题
为什么一条边不一定比三条边更短？
```

不要说：

```text
系统已经帮学生掌握了这个知识点。
```

此时只是开始修正。

正确标准：

```text
卡片包含误区、资料依据、待解决问题。
```

错误样子：

```text
卡片只有标题。
卡片只是资料摘要。
没有学生误区。
```

### 07. 费曼输出

可说：

```text
系统不把 AI 解释当作掌握，而是要求学生自己解释。
```

页面证据：

```text
Agent 提问：
为什么一条边不一定比三条边更短？

用户回答：
因为每条边有不同权重。
如果一条边权重是 100，而三条边每条权重是 10，
三条边总代价是 30，所以三条边更短。
```

系统输出：

```text
学生自己的解释进入卡片。
自动保存提示。
观察记录或画像证据更新。
```

不要说：

```text
用户已经完全掌握。
```

要说：

```text
用户已经给出一个可用于评估的理解证据。
```

正确标准：

```text
有用户自己的话。
有反例。
写入卡片。
有保存或记录状态。
```

错误样子：

```text
只有 AI 讲解。
用户没有输出。
用户输出没有保存。
```

### 08. 生成个性化资源

可说：

```text
系统不是生成一堆通用材料，而是围绕“边数和权重总和混淆”这个误区生成资源。
```

页面证据：

```text
生成依据：
当前卡片：最短路径不是边数最少。
学生误区：混淆边数和权重总和。
画像变化：已能解释路径总代价。
剩余缺口：Dijkstra 的选择过程和适用边界。
资料来源：数据结构课程讲义-第6章-图。
```

资源输出：

```text
讲解文档
思维导图
练习题
代码实操
视频/动画脚本
```

不要说：

```text
系统生成了最好的学习资源。
```

要说：

```text
系统生成了五类围绕当前误区的学习资源。
```

正确标准：

```text
至少五类。
每类资源和误区有关。
至少一类资源回应画像里的剩余缺口。
能打开或预览。
有生成进度。
```

错误样子：

```text
只有一段文本。
资源和误区无关。
不能预览。
没有来源。
```

### 09. 多智能体协同

这一段必须谨慎。

只有页面能展示角色和产物时，才能强说：

```text
这是多智能体协同。
```

页面证据最低要求：

```text
诊断 Agent：输出学生误区。
文献 Agent：输出资料引用。
路径 Agent：输出学习步骤。
资源 Agent：输出五类资源。
评估 Agent：输出校验结果。
观察 Agent：输出画像记录。
```

如果页面上看不到这些，只能弱说：

```text
在后台，系统按诊断、文献、路径、资源、评估、观察几个角色组织生成流程。
```

不要说：

```text
我们完整实现了多智能体协同。
```

除非每个角色都有可见产物。

正确标准：

```text
每个 Agent 有输入、状态、输出。
最终产物能对应到资源、路径、画像或评估。
```

错误样子：

```text
只有 Agent 名字。
没有产物。
没有状态。
像概念包装。
```

### 10. 推送下一步

可说：

```text
系统不是随便推荐下一项，而是说明为什么推这个连接、为什么推这个任务。
```

页面证据：

连接推送：

```text
建议连接：边权 -> 最短路径
理由：学生已经能说出路径总代价。
证据：卡片中出现“最短路径比较的是权重总和”。
```

任务推送：

```text
建议任务组：Dijkstra 的适用边界
理由：下一步需要理解为什么负权边会影响 Dijkstra。
画像字段：剩余缺口 = Dijkstra 的适用边界。
证据：导入资料中出现“Dijkstra 适用于非负权图”。
```

不要说：

```text
系统精准预测了所有后续学习内容。
```

要说：

```text
系统基于当前卡片和资料证据，给出下一步建议。
```

正确标准：

```text
有推荐理由。
有证据来源。
有画像字段或卡片证据。
可执行或可忽略。
不会打断当前学习。
```

错误样子：

```text
只有推荐标题。
没有理由。
推荐和当前学习无关。
```

### 11. 掌握评估

可说：

```text
系统不把“看过资料”当作掌握，而是要求学生输出，并按标准判断。
```

页面证据：

```text
问题：
为什么带权图里的最短路径不是边数最少？
请给出一个反例。

学生回答：
A 到 B 直接边权是 100；
A 到 C 到 D 到 B 三条边分别是 10、10、10；
三条边总代价是 30，所以更短。
```

评估输出：

```text
掌握判断：通过。
通过原因：
1. 能区分边数和权重总和。
2. 能给出具体反例。
3. 能用自己的话解释。
下一步：学习 Dijkstra 的选择过程。
```

不要说：

```text
系统保证学生已经完全掌握。
```

要说：

```text
系统基于这次回答判断当前任务通过。
```

正确标准：

```text
有题目。
有学生回答。
有判断标准。
有通过/不通过。
有下一步。
```

错误样子：

```text
点击完成就通过。
答错也通过。
只有分数，没有原因。
```

### 12. 防幻觉和质量边界

可说：

```text
进入长期知识库前，系统会检查内容是否清晰、准确、必要。
```

页面证据：

错误卡片示例：

```text
最短路径就是经过边数最少的路径。
```

系统驳回：

```text
暂不能升级为永久知识。
准确性不足：仍把最短路径说成边数最少。
清晰性不足：没有解释路径总代价。
必要性不足：缺少反例支撑。
```

通过依据：

```text
导入讲义。
学生解释。
评估通过记录。
```

不要说：

```text
系统百分百防幻觉。
```

要说：

```text
系统在知识沉淀前设置质量边界，错误内容会被驳回。
```

正确标准：

```text
有错误示例。
有驳回原因。
有修改建议。
修改后才能升级。
```

错误样子：

```text
任何内容都能升级。
只有“失败”提示，没有原因。
没有资料依据。
```

### 13. 前后变化

可说：

```text
一次学习后，空仓库长出了资料、路径、卡片、资源、评估证据和图谱关系。
```

页面证据：

知识图谱：

```text
中心节点：数据结构
二级节点：图、路径、边权、最短路径、Dijkstra
文献节点：课程讲义
理解卡：最短路径不是边数最少
永久知识：带权图中的最短路径定义
资源卡：文档、导图、练习、代码、动画
连接：边权 -> 最短路径
```

认知洞察：

```text
原始状态：混淆边数和权重总和。
当前状态：能解释路径总代价并给出反例。
下一步：学习 Dijkstra 的选择过程和适用边界。
```

不要说：

```text
系统已经完整掌握学生全部学习情况。
```

要说：

```text
系统记录了这一次学习中的关键变化。
```

正确标准：

```text
能看到前后变化。
节点类型清楚。
画像变化具体。
下一步和本次学习相关。
```

错误样子：

```text
图谱没有变化。
画像只是空泛评价。
下一步建议无关。
```

### 14. 结尾

可说：

```text
AXIOM Space 解决的不是“多生成资料”，而是把一次学习变成可诊断、可推进、可评估、可沉淀的过程。
```

证据回放：

```text
学生误区
主动诊断
真实资料
学习路径
理解卡
五类资源
评估结论
质量驳回
图谱和画像变化
```

不要说：

```text
这是最先进的学习系统。
```

要说：

```text
这条演示链路证明：学生从一个具体误区出发，经过资料、路径、资源、输出和评估，形成了可继续积累的知识结果。
```

正确标准：

```text
最后回到学习问题。
不要回到功能清单。
```

## 7. 第七版中需要改写的句子

| 第七版表达 | 问题 | 第八版表达 |
|---|---|---|
| 这个人懂学生真正的学习困难 | 这是人格判断，不能直接证明 | 系统先识别“边数最少 ≠ 最短路径”的具体误区，而不是直接讲答案 |
| 这个人懂 AI 的能力，也懂 AI 的风险 | 太抽象 | 系统先导入真实讲义，再生成路径和资源；卡片升级前有质量校验 |
| 可验证学习系统 | 抽象词 | 有用户输出、评估标准、通过原因和下一步 |
| 精准资源 | 抽象词 | 五类资源都围绕“边数和权重总和混淆”这个误区 |
| 防幻觉机制 | 抽象词 | 错误卡片不能升级，弹窗说明准确性、清晰性、必要性问题 |
| 多智能体协同 | 如果没有角色产物，会被认为是口号 | 展示诊断、文献、路径、资源、评估、观察 Agent 的输入和输出 |
| 长期知识系统 | 抽象词 | 学科仓库、永久知识卡、图谱节点、画像变化和下一步建议 |
| 真实掌握 | 抽象词 | 学生能用自己的话解释，并给出带权图反例 |

## 8. 页面证据分级

### A 级证据：可以强说

```text
页面直接显示，评委能看见。
```

例如：

```text
文件名、长文本、学习路径、卡片内容、资源卡片、评估结论、驳回弹窗、图谱节点。
```

### B 级证据：可以弱说

```text
后台有逻辑，但页面只部分显示。
```

例如：

```text
画像更新、RAG 检索、多智能体分工、动态路径调整。
```

表达方式：

```text
不要说“系统完整实现”。
可以说“系统在这里按……组织处理，并把结果体现在……页面上”。
```

### C 级证据：不要说

```text
页面看不到，后台也不能稳定证明。
```

例如：

```text
百分百防幻觉。
完全掌握学生。
最优学习路径。
全面超过传统工具。
```

## 9. 最低页面对照清单

为了让第八版成立，页面至少要能展示：

```text
1. 创建《数据结构》仓库。
2. 空仓库状态。
3. 系统主动画像对话。
4. 用户误区：最短路径是边数最少。
5. 系统识别误区。
6. 导入真实长资料。
7. Markdown 文献节点。
8. 学习路径步骤。
9. 理解卡：误区、资料依据、待解决问题。
10. 用户自己的解释进入卡片。
11. 自动保存或观察记录。
12. 五类资源卡片。
13. 画像前后变化和触发证据。
14. 资源生成依据引用变化后的画像字段。
15. 多智能体角色产物；如果没有，就弱化多智能体表达。
16. 推送箱推荐理由、证据和画像字段。
17. 掌握评估问题、回答、标准、结论、下一步。
18. 升级永久知识的质量校验和驳回理由。
19. 图谱前后变化。
20. 认知洞察页显示画像沉淀和下一步学习状态。
```

如果某一项页面没有，就不要在视频里强说它。

## 10. 最终判断标准

第八版的目标不是让文档更长，而是让表达更可验证。

正确效果：

```text
评委看到每个主张都有画面证据。
评委知道学生哪里错。
评委知道系统怎么发现。
评委知道资源为什么对他有用。
评委知道学生怎么证明自己懂了。
评委知道错误内容为什么不能沉淀。
```

错误效果：

```text
评委听到很多概念：
个性化、多智能体、防幻觉、精准推送、长期画像。
但页面上看不到对应证据。
```

最终原则：

```text
能证明的，说硬。
只能解释的，说轻。
不能证明的，不说。
```
