# AXIOM Space A3 黄金案例 V19｜一镜到底母视频录制导演稿

> 本版用途：先录一条完整产品操作母视频，再让 HTML 按视频时间轴叠加字幕、局部放大、左右构图与重点动效。  
> 核心原则：视频负责证明产品真的做到了；HTML 负责降低理解门槛。  
> 录制浏览器：Zen Browser。  
> 录屏工具：`D:\_app_soft\_any\Cap\cap-cli.exe`。  
> 演示账号：`demo@axiom.space / demo123456`。  
> 数据状态：A3 黄金案例已通过本地验收，包含干净仓库与长期档案。

## 1. 全片唯一主张

AXIOM Space 是给高校学生使用的 AI 掌握学习系统。

它不是把知识“输入”给学生就结束，而是让学生用自己的话输出；AI 用证据判断是否掌握；掌握证据再改变画像、资源、路径和下一次学习。

全片只证明这一件事：

```text
进入仓库
  -> 建立画像
  -> 导入资料
  -> 资料变成知识节点
  -> 灵感卡进入工作台
  -> 苏格拉底式追问
  -> 学生费曼输出
  -> Agent B 记录证据
  -> 审核失败
  -> 继续打磨
  -> 审核通过
  -> 灵感卡升级为永久卡
  -> 系统按证据推资源和路径
  -> 图谱显示长期掌握状态
```

---

## 2. 录制基准

- 录制一条连续母视频，不在中途切换到 PPT 静态页。
- 鼠标动作要慢，关键结果停留 2 到 4 秒。
- 每个画面只证明一个信息，不要在一个镜头里乱点很多功能。
- 遇到等待加载，不乱晃鼠标；停在能看懂的界面状态上。
- 不录注册、设置、后台、终端和调试页面。
- 不把“生成一个视频任务”说成创新点；它只是基础能力。
- 必须展示三种卡片含义：文献卡是来源，灵感卡是尚未掌握，永久卡是已经掌握。
- 必须展示苏格拉底式提问、费曼输出、Agent B 记录、审核失败与再次通过。
- 必须展示向量库/语义索引带来的旧知迁移，而不是只说“长期仓库”。

## 3. 一镜到底录制顺序

### 00:00-00:20 进入仓库

画面动作：打开 AXIOM，进入“设计模式黄金案例”仓库。

评委必须看见：

- 这是一个高校课程学习场景；
- 产品不是普通聊天框，而是一个学习仓库；
- 仓库内有资料、卡片、画像、图谱、路径这些长期对象。

后续 HTML 叠加：

- 大标题：`AI 给出答案，不等于学生已经学会`
- 小标签：`AXIOM Space｜高校学生的 AI 掌握学习系统`
- 动效：标题 3 秒后缩到左上角，让界面成为主画面。

### 00:20-00:55 初次画像对话式构建

画面动作：进入画像/对话入口，展示系统通过对话了解学生的目标、基础、卡点和学习偏好。

评委必须看见：

- 个性化不是手填标签；
- 画像来自学生的真实表达；
- 画像是可更新的假设，不是永久贴标签。

后续 HTML 叠加：

- 主字幕：`个性化不是填表，是从真实对话开始`
- 逐步字幕：`目标`、`基础`、`卡点`、`适合的解释方式`

### 00:55-01:15 查看画像变化

画面动作：切到认知洞察/画像变化页面，展示画像维度、证据来源、变化历史。

评委必须看见：

- AI 的判断有证据；
- 学生表现会更新画像；
- 画像不是知识点清单，而是学习方式与卡点的判断。

后续 HTML 叠加：

- 主字幕：`AI 不是猜学生，它引用证据修正判断`
- 局部放大：画像变化、证据来源、更新时间。

### 01:15-01:45 导入学习资料

画面动作：展示导入课程资料或已导入资料入口，说明课程材料进入仓库。

评委必须看见：

- 学生不是从空白聊天开始；
- 课程资料成为后续学习的来源；
- 资料不是只保存文件，而是会进入卡片和图谱。

后续 HTML 叠加：

- 主字幕：`课程资料进入仓库，才有可追溯的学习来源`
- 动效：给资料列表或导入结果加高亮框。

### 01:45-02:25 Dashboard 与三色卡片盒

画面动作：进入 Dashboard，展示知识图谱和左侧三种卡片统计。

评委必须看见：

- 文献卡、灵感卡、永久卡三类并存；
- 灵感卡代表尚未掌握；
- 永久卡代表已经掌握；
- 资料已经变成可以继续学习的节点。

后续 HTML 叠加：

- 主字幕：`同一份资料，被拆成可学习、可审核、可积累的知识节点`
- 三色标签：`文献卡=来源`、`灵感卡=尚未掌握`、`永久卡=已经掌握`
- 动效：三色标签贴近真实统计区域，不另做脱离界面的流程图。

### 02:25-02:45 从图谱点击灵感卡进入工作台

画面动作：在图谱上选择 Visitor 相关灵感卡，进入工作台。

评委必须看见：

- 同一知识点从 Dashboard 到工作台不断裂；
- 工作台打开的是一张尚未掌握的灵感卡；
- 后续对话围绕这张卡发生。

后续 HTML 叠加：

- 主字幕：`尚未掌握的知识点，会被带进学习工作台`
- 动效：节点高亮、连线到工作台标题。

### 02:45-03:35 苏格拉底式提问、费曼输出与 Agent B 记录

画面动作：展示 Agent A 追问，学生用自己的话解释 Visitor 双重分派；右侧或后台证据区显示记录/更新。

评委必须看见：

- AI 不是直接给答案，而是用预测、为什么、反例、边界追问；
- 学生必须用自己的话输出；
- Agent B 记录原话、更新画像和灵感卡，不替学生伪造理解。

后续 HTML 叠加：

- 主字幕：`通用 AI 多数帮你输入，AXIOM 强制你输出`
- 角色标签：`Agent A：苏格拉底式追问`、`学生：费曼式输出`、`Agent B：记录证据`
- 动效：对话区局部放大，右侧证据写回区域短暂高亮。

### 03:35-04:20 审核失败、继续打磨

画面动作：触发永久卡审核，展示第一次未通过；继续对话补足边界、反例或新情境。

评委必须看见：

- AI 不能直接宣布掌握；
- 没讲清就不能升级为永久卡；
- 审核失败不是失败，而是下一轮学习的依据。

后续 HTML 叠加：

- 主字幕：`没讲清，就不能算掌握`
- 小字幕：`失败原因 -> 继续追问 -> 补足边界`
- 动效：审核失败状态用红色短闪，不做恐吓式大红屏。

### 04:20-04:55 再次审核通过，灵感卡升级为永久卡

画面动作：展示修正后的回答、新情境应用、再次审核通过；卡片状态从灵感卡升级为永久卡。

评委必须看见：

- 通过标准来自学生输出、边界处理和新情境应用；
- 灵感卡升级为永久卡；
- 永久卡代表这项知识在本流程中已经掌握。

后续 HTML 叠加：

- 主字幕：`通过审核后，知识才从“看过”变成“掌握过”`
- 动效：青色灵感卡到紫色永久卡的状态变化，用真实 UI 高亮，不另画假卡片。

### 04:55-05:35 学生主动请求资源

画面动作：学生主动要求某类资源，例如视频、题库、代码练习或导图，系统生成并展示进度或结果。

评委必须看见：

- 学生知道自己想要什么时，可以主动生成；
- 资源类型服务学习目的；
- 生成结果能打开预览。

后续 HTML 叠加：

- 主字幕：`学生知道需要什么，就主动生成什么`
- 资源标签：`讲解文档`、`导图`、`题库`、`代码`、`图解`、`动画`

### 05:35-06:15 系统主动推荐资源

画面动作：展示系统推送资源建议，包含理由、证据、预期结果以及接受/忽略入口。

评委必须看见：

- 当学生不知道缺什么时，系统会根据画像、卡片、评估和向量检索给出建议；
- 系统不擅自执行，学生可以接受或忽略；
- 推荐理由能追溯到证据。

后续 HTML 叠加：

- 主字幕：`学生不知道缺什么时，系统也能给出有证据的下一步`

### 06:15-06:45 六类资源真实预览

画面动作：依次打开六类资源预览，速度快但每类都要让评委看见真实内容。

评委必须看见：

- 六类资源不是图标；
- 每类资源都有真实内容和用途；
- 资源来自当前学习缺口，而不是通用模板。

后续 HTML 叠加：

- 主字幕：`不是生成文件数量，而是按缺口生成可用资源`
- 资源用途短标签：`读懂`、`看结构`、`练迁移`、`跑代码`、`看过程`、`做复测`

### 06:45-07:25 知识图谱多视角

画面动作：在同一知识图谱上切换结构、关系、来源、卡片、掌握、路径等视角。

评委必须看见：

- 同一个知识网络可以从不同问题进入；
- 能看来源、连接、掌握状态和下一步路径；
- 图谱不是装饰，而是学习决策界面。

后续 HTML 叠加：

- 主字幕：`同一张图谱，回答四个问题：从哪来、怎么连、会了什么、下一步去哪`
- 动效：每切一个模式，只出现一个对应问题。

### 07:25-07:55 长期价值收束

画面动作：回到长期档案或图谱总览，停在稳定明亮画面。

评委必须看见：

- 学习不是一次对话，而是长期积累；
- 每次输出和审核都会成为下一次学习的起点；
- 旧知识可以通过向量检索被找回，用来迁移到新问题。

后续 HTML 叠加：

- 主标题：`让掌握变得可见、可验证、可积累`
- 愿景句：`用 AI 降低高质量学习的门槛，让每个学生都能拥有持续懂自己的学习系统。`
- 最终页必须是浅色，不使用黑底。

## 4. 后续 HTML 设计方式

HTML 不再把每一幕做成孤立 PPT，而是做成一条视频增强时间轴：

```text
一个 video 元素
  -> timeupdate 监听当前播放秒数
  -> 根据时间段切换字幕、标题、标签、局部放大和布局
  -> 必要时把视频缩到右侧，左侧显示当前价值判断
  -> 下一段再回到全屏视频
```

可用布局：

- `full-video`：视频全屏，叠加一行主字幕；
- `focus-zoom`：视频保持全屏，对关键区域加放大镜和高亮框；
- `split-right-video`：视频缩到右侧，左侧显示一个传播学判断；
- `three-labels`：用于三色卡片盒和六类资源；
- `final-light`：最后 8 到 10 秒浅色价值定格。

HTML 上屏文字标准：

- 每一时刻只出现一个主判断；
- 大字必须能 3 秒读完；
- 小字只解释正在看的控件或状态；
- 术语只做标签，不压过中文结论；
- 所有字幕都能脱离配音看懂；
- 配音也能脱离字幕讲清产品；
- 视频画面也能脱离字幕看出真实操作。

## 5. 验收标准

录完后必须逐项检查：

- 是否没有黑屏、空白页、跳出产品、登录卡顿；
- 是否完整覆盖固定叙事顺序；
- 是否看得出 AXIOM 是什么软件；
- 是否看得出它与通用 AI 的差异；
- 是否看得出灵感卡与永久卡的真实含义；
- 是否看得出审核失败和通过之间的因果关系；
- 是否看得出资源推荐不是无脑生成；
- 是否看得出向量库/语义索引让旧知服务新学习；
- 是否适合后续 HTML 做字幕和动效；
- 是否整体节奏舒服，评委无需懂技术也能跟上。
