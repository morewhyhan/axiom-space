# AXIOM Space A3 黄金案例 V21｜PPT 演示前置设计稿

> 用途：在制作 PPT / HTML 演示与录制最终视频之前，先统一每一页到底要证明什么。  
> 本稿不是单纯的逐字演讲稿，也不是最终页面代码；它是后续视觉设计、产品录屏、字幕动效和配音的共同内容真源。  
> 每个呈现单元固定回答四个问题：**PPT 写什么、画面放什么、视频证明什么、现场说什么。**  
> 目标受众：第一次听说 AXIOM、不了解 Agent、卡片盒和知识图谱的评委。  
> 建议成片：7 分钟。当前内容完整展开约 7 分 07 秒，允许先把价值讲透，再在真实录制后压缩约 7—10 秒；优先压缩解释页，不能压缩到看不清学生输出、审核失败、修正与通过。

---

## 一、整场汇报只让评委记住一件事

> **通用 AI 更擅长把知识输入给学生；AXIOM 让学生亲自输出，用证据确认掌握，再让这份证据改变下一次学习。**

AXIOM Space 的清晰定位是：

> **给高校学生使用的 AI 掌握学习系统。**

“掌握学习”必须用白话解释：

> 学生要用自己的话讲清，给出例子或反例，说明适用边界，并在新情境中仍然会用；通过审核后，这项知识才能成为永久卡，代表它达到本次掌握标准。

整场汇报不是介绍一组互不相干的功能，而是证明一条完整因果链：

```text
系统先认识学生
  -> 课程资料变成知识节点
  -> 学生打开一处尚未掌握的灵感卡
  -> Agent A 苏格拉底式追问
  -> 学生完成费曼式输出
  -> Agent B 保留原话与证据
  -> 审核失败则继续打磨
  -> 审核通过才升级为永久卡
  -> 画像、资源、图谱和路径随证据改变
  -> 旧知识成为下一次学习的起点
```

---

## 二、不能改变的真实操作顺序

中段产品实录必须严格按下面的顺序发生，不得为了页面好看而打乱：

```text
进入仓库
  -> 初次画像对话式构建
  -> 查看画像变化
  -> 导入学习资料
  -> 在仪表盘查看资料转换成的知识节点
  -> 解释知识图谱与三色卡片盒
  -> 在图谱上点击灵感卡，在工作台打开
  -> Agent A 苏格拉底式提问
  -> 学生用自己的话完成费曼输出
  -> Agent B 后台保留原话、更新画像与灵感卡
  -> 从永久卡讲到审核机制
  -> 第一次审核未通过
  -> 继续对话和打磨
  -> 再次审核通过，升级为永久卡
  -> 展示主动请求与系统建议两种资源方式
  -> 展示六类资源
  -> 展示知识图谱的几种模式
```

第 05—14 页应尽量来自**同一条连续录屏母视频**。PPT / HTML 只是随着视频进度改变标题、字幕、局部放大和布局，不把连续操作剪成互不相关的功能短片。

## 三、整套 PPT 的页面分工

| 阶段 | 页码 | 评委正在回答的问题 | 主要媒介 | 建议时间 |
|---|---:|---|---|---:|
| 建立问题 | 01—02 | 为什么现在还需要 AXIOM？ | 强标题＋真实界面背景 | 32 秒 |
| 说清方案 | 03—04 | AXIOM 是什么，系统如何工作？ | 定位页＋架构与路线图 | 45 秒 |
| 真实演示 | 05—14 | 一个学生究竟怎样从资料走到掌握？ | 同一条连续产品实录＋同步字幕 | 4 分 20 秒 |

全稿共 17 个呈现单元，但纯静态解释页只有开头、架构、创新、价值与结尾。第 05—14 页是同一段产品实录中的 10 个叙事状态，不是 10 次生硬切页。

### 决策记录怎样进入本稿（内部制作索引，不直接上屏）

| 决策记录 | 真正值得进入汇报的内容 | 最佳落点 | 取舍 |
|---|---|---|---|
| `01-技术选型` | 学习对象长期持久化；业务事实、语义索引与 Agent 操作分层 | 04、08、15 | 不展示技术栈清单，只保留系统边界 |
| `02-自研垂直领域 Agent` | Harness 的对象绑定、上下文装配、工具治理、证据路由与状态写回 | 04、10、11、15 | 不把五个子 Agent 画成五个聊天机器人 |
| `03-桌面优先还是 Web 优先` | 一个工作台承接跨页面、跨对象的连续闭环 | 05—14 | Web 是产品形态，不包装成创新 |
| `04-知识图谱创新` | 图谱模式回答不同认知问题；图谱继续反哺学习下一步 | 09、14、15 | 不以模式数量和 3D 动画作为价值 |
| `05-RAG 增强体验` | 旧知可召回、引用可回源、索引可重建；召回不等于掌握 | 08、11、15 | 不把向量相似度包装成学习结论 |
| `06-清晰、准确、必要` | 知识进入永久卡前的三道准入门 | 12 | 作为真实失败原因，不另起理论页 |

| `08-资源推送机制` | 多路信号找缺口，AI 形成候选，学生确认后才执行 | 13、14、15 | 不强调查找器数量，强调治理边界 |
| `09-根目录结构规则` | 工程治理与主线边界 | 答辩备用 | 不占七分钟主叙事 |
| `10-学习效果评估` | 费曼输出、卡片审核、后续反馈构成递进证据 | 11、12、16 | 未完成的集中健康度面板不当作已交付能力 |
| `11-两分钟语义索引与后台图谱增强` | 前台快速召回、后台深关系增强、业务事实回源 | 08、15、16 | 实测数据只作为运行证据，不冒充学习效果实验 |

**总取舍**：把机制放到它真正发生作用的页面；第 15 页只负责把前面已经看见的机制重新命名，不再第一次引入一串新概念。

# 四、逐页 PPT / 视频 / 口播设计

## 01｜开篇：AI 给出答案，不等于学生真正学会

**本页唯一主旨**：第一眼点明教育问题，同时让评委知道产品服务谁、是什么。

**页面形态**：强标题开场。背景使用真实课程仓库首帧，轻度虚化；不要使用抽象 AI 大脑、机器人或素材图库。

**PPT 上屏文字**

产品主语，大字显示：

> 给高校学生使用的 AI 掌握学习系统

主标题：

> AI 给出答案，  
> **不等于学生真正学会**

署名：

> AXIOM Space

**画面放什么**

- 背景能辨认出课程仓库，而不是聊天框；
- 标题出现后，仓库界面从虚化恢复清晰；
- 不播放后面审核、资源或图谱的预告蒙太奇。

**对应口播**

AXIOM Space 是给高校学生使用的 AI 掌握学习系统。AI 已经很会给答案，但答案出现，不等于学生真的学会。AXIOM 要解决的，就是如何证明掌握。

**转场**

标题缩小到左上角，背景仓库保持不变，进入第 02 页的问题对标。

---

## 02｜行业痛点：现有工具解决了输入，掌握仍然不可见

**本页唯一主旨**：不是否定现有工具，而是指出它们普遍没有完成“掌握证据闭环”。

**页面形态**：两侧对比。左边用克制的三类工具标签，右边用一个醒目的“学习缺口”。

**PPT 上屏文字**

主标题：

> 现有工具让知识更容易获得，  
> **却很难证明学生是否掌握**

左侧只保留：

> 通用 AI：解释、问答、生成  
> 课程平台：组织内容与任务  
> 笔记工具：保存资料与笔记

右侧三个缺口：

> 输出没有形成可追溯的系统证据  
> 掌握缺少明确审核门槛  
> 下一步很少由掌握证据动态改写

底部结论：

> 学习系统最该回答的问题：**学生凭什么算学会？**

**画面放什么**

- 左侧不是三张复杂卡片，只用三个简洁图标与一句能力说明；
- 右侧三个缺口依次出现，最后收拢成底部问题；
- 背景仍可隐约看见 AXIOM 仓库，保持项目上下文。

**对应口播**

通用 AI 会解释和生成，课程平台组织内容，笔记工具保存资料。但在常见流程里，发现盲区、判断掌握和安排下一步，仍主要留给学生自己。真正缺少的不是更多内容，而是真实输出、掌握证据和下一步反馈。

**真实性边界**

不能说“通用 AI 不会提问”或“传统工具完全不能个性化”。真实差异是：AXIOM 把学生原话连续写入画像、卡片、审核、资源和路径。

---

## 03｜产品定位：AXIOM 组织的是一次可验证的学习

**本页唯一主旨**：让零基础评委用一句话理解产品，不把它误认成聊天机器人、笔记工具或知识图谱展示器。

**页面形态**：一句定位＋一条极短闭环。

**PPT 上屏文字**

主标题：

> AXIOM 不替学生完成学习，  
> **而是组织学生完成一次可验证的学习**

核心闭环：

> 学生输出 → 证据审核 → 确认掌握 → 改变下一步

产品定位：

> AXIOM Space｜高校学生的 AI 掌握学习系统

**画面放什么**

- 中间四个词沿一条路径依次点亮；
- 每个词最多配一个真实产品缩略图：回答、审核、永久卡、下一路径；
- 不在这一页展开功能细节。

**对应口播**

AXIOM 不是再做一个聊天机器人，而是组织一次可验证的学习：学生亲自输出，系统保留证据，审核确认掌握，再让结果改变下一步。

## 04｜整体架构：一次学习怎样被系统接住

**本页唯一主旨**：用一组递进架构图说清 Harness 为什么必须存在：它把模型、工具、证据、卡片、图谱和路径牵引到同一条学习闭环里。

**页面形态**：两段递进式架构动画。先看完整学习闭环，再放大 Harness 与数据边界；信息分时出现，绝不能把两张复杂架构图同时铺满。

**PPT 上屏文字**

主标题：

> AXIOM 的系统架构：  
> **把一次学习变成可追踪的掌握闭环**

第一状态｜评委先看懂产品怎样运转：

```text
学生 + 课程资料
        ↓
一个长期学习工作台
  仓库｜画像｜Learn｜Forge｜Galaxy｜资源
        ↓
AXIOM Agent Harness｜统一学习编排
  绑定学习对象 → 装配学习现场 → 治理工具 → 路由证据 → 写回状态
        ↓
掌握学习核心
  苏格拉底追问 → 学生费曼输出 → 审核 → 反馈
        ↓
长期学习状态
  卡片｜画像｜评估｜图谱｜路径｜资源
```

第二状态｜镜头放大 Harness，屏幕只保留五个动作：

> 对象绑定｜上下文装配｜工具治理｜证据路由｜状态写回

四个判断只由口播解释，不再作为常驻文字：

> 当前在学什么？AI 能做什么？什么算学生证据？证据要写回哪里？

**画面放什么**

- 第一状态从上到下只有五层，不画代码模块和数据库表；
- “学生输出”必须位于视觉中心，不能让 AI 或技术栈抢主位；
- `AXIOM Agent Harness` 必须画成中间的编排层，而不是一个普通 Agent 头像；
- Harness 周围用五个小标签依次点亮：`对象绑定`、`上下文装配`、`工具治理`、`证据路由`、`状态写回`；
- “未通过”箭头返回工作台，“已通过”箭头进入永久卡并继续影响下一步；
- 可参考 `01-tech-architecture.png` 与 `02-agent-harness-capability-map.png` 的逻辑，但必须按当前浅色、金色、紫色品牌体系重画，不能原样粘贴内部决策图；
- 右下角可用小字标注：`同一知识点贯穿图谱、工作台、审核、资源与路径`。

架构图下方增加五阶段演示路线，但只作为细进度条，不与架构主体争夺视觉主位：

> 先认识学生 → 资料变成知识 → 学生亲自讲清 → 审核确认掌握 → 证据改变下一步

**对应口播**

AXIOM 的底层是一个长期学习工作台。资料、卡片、画像、图谱和路径都不是聊天里的临时文字，而是可以持续更新的学习对象。真正把它们接成系统的，是我们自研的 AXIOM Agent Harness。它不是一个更大的模型，而是包在通用 Agent 外面的学习运行框架。它每轮都要回答四件事：当前在学什么，AI 能做什么，什么学生输出可以成为证据，证据应该写回哪里。于是系统先绑定当前仓库、会话、卡片和路径，再装配画像、旧知和图谱上下文；随后按本轮意图只开放必要工具，高影响动作必须确认并留下审计；最后根据证据类型，选择性触发画像、审核、资源、图谱或路径更新，不会每轮重写全部状态。

**设计限制**

本页不展示 Next.js、Hono、Qdrant、LightRAG 等技术名词。这里只让评委先理解 Harness 怎样把一次回答接成学习闭环；数据边界留到第 08 页随导入动作出现。

## 05｜进入仓库：学习不是一次临时对话

**本页唯一主旨**：AXIOM 以课程仓库承接长期学习状态，而不是每次从空白聊天开始。

**页面形态**：连续母视频正式起点。

**PPT 上屏文字**

> 每门课程，  
> **都有一个持续生长的学习仓库**

小字幕：

> 资料｜画像｜卡片｜图谱｜路径

对象连续性字幕，仅在页面入口依次点亮时出现：

> 同一门课程、同一份学习状态，在不同页面之间不断裂

**视频必须拍到**

- 学生进入自己的课程仓库；
- 仓库内能看见资料、画像、卡片、图谱或路径入口；
- 后续 Learn、Forge、Galaxy 与 Cognition 始终显示同一个仓库身份；
- 登录注册若必须出现，只保留必要动作，不在表单上停留。

**对应口播**

小林先进入自己的课程仓库。这里保存的不是一次聊天记录，而是一门课程持续生长的资料、画像、卡片、图谱和学习路径。后面无论进入工作台、认知洞察还是知识图谱，处理的始终是同一份学习状态。

**转场**

鼠标直接点击“建立初始画像”，不中断母视频。

---

## 06｜初次画像：个性化从真实对话开始

**本页唯一主旨**：一段真实对话，形成六项等待后续证据验证的教学假设。

**页面形态**：完整连续对话实录。标题只出现 3 秒，随后移到安全区或退出。

**素材状态**：**待补录并核验**。必须录成“进入仓库 → 至少两轮画像对话 → 画像已更新 → 打开认知洞察”的连续片段，不能用固定问卷截图代替。

**PPT 上屏文字**

主标题：

> 一段真实对话，  
> **形成六项待验证的教学假设**

六个教学问题，随对话分两组出现：

> 学什么｜会什么｜怎么讲  
> 哪里卡｜讲多少｜怎么算会

过程字幕：

> 上一次回答 → 改变下一次提问

结束字幕：

> 这只是待验证的初始画像

**视频必须拍到**

- 至少两轮真实问答；
- 第二个问题明显使用了学生上一轮回答；
- 对话至少覆盖学习目标、当前基础与一种验收偏好；
- 对话结束后出现画像已建立或已更新的真实反馈；
- 直接进入认知洞察入口。

**对应口播**

第一次进入仓库，系统不会先让小林勾选几组固定标签，而是通过连续对话形成六个教学判断：他学什么、会什么、怎样讲更容易懂、哪里容易卡住、一次适合讲多少、怎样才算学会。它们不是人格标签，而是下一轮教学的控制参数。每次回答都会改变下一问；对话结束后得到的也不是永久结论，而是一组等待后续学习表现验证的初始假设。

---

## 07｜查看画像变化：AI 对学生的判断也要有证据

**本页唯一主旨**：学生不仅能看见和纠正画像，还能知道这项判断将怎样改变下一轮教学。

**页面形态**：认知洞察实录＋三处局部放大。

**PPT 上屏文字**

主标题：

> 每一条画像都有出处，  
> **也能被学生纠正**

四步证据链，依次出现：

> 学生原话 → 当前判断 → 教学干预 → 后续验证

底部短句：

> 来源可见｜依据强弱可见｜学生可以纠正

**视频必须拍到**

- 展开一条具体画像判断；
- 该判断能够回到一条学生原话或真实学习证据；
- 同时显示依据强弱或置信度、前后变化记录与学生纠正入口；
- 明确显示它将改变下一轮的起点、例子、节奏或验收方式之一；
- 不只展示一张没有来源的雷达图。

**对应口播**

在认知洞察中，小林可以看到系统观察到了什么、为什么这样理解、下一轮准备怎样帮助他。这项判断能回到原话，能看到证据强弱，也可以被小林纠正。新的学习信号会触发相关维度的增量分析，系统还会周期复核；只有证据支持的变化才写回。后续表现可以增强、削弱甚至推翻旧判断。个性化不是 AI 悄悄贴标签，而是双方共同维护一份可检查、可修正、会真实改变教学的假设。

**转场**

回到同一仓库，直接进入资料导入。

## 08｜导入资料：文件变成可以继续学习的知识对象

**本页唯一主旨**：AXIOM 先保存可信原文，再让资料进入前台可召回、后台可增强的知识网络。

**页面形态**：导入、解析、返回仪表盘的连续实录。

**PPT 上屏文字**

主标题：

> 一份课程资料，  
> **先成为可信对象，再进入知识网络**

过程字幕：

> 原文先保存 → 提取知识点 → 前台可搜索 → 后台补关系

视频缩到右侧时，左侧只出现一次双速语义简图：

> 业务事实源：原文、卡片、路径不丢失  
> 快速语义层：相关旧知先可被召回  
> 后台图谱层：实体与深层关系继续增强

收束句：

> 资料先可用，关系再增强

**视频必须拍到**

- 导入一份真实课程资料；
- 原始资料先保存成功，AI 抽取失败也不能导致原文丢失；
- 解析状态真实推进；
- 若页面已有状态，拍到“可搜索”与“图谱增强中 / 完成”的区别；若 UI 未呈现，不得伪造，只用 PPT 简图解释后台链路；
- 解析完成后直接返回仪表盘；
- 下一页出现的知识节点确实来自这份资料。

**对应口播**

接下来，小林导入真实课程资料。AXIOM 先把原文和来源保存成业务事实，AI 抽取失败也不会让资料丢失；随后提取知识节点，让快速语义层优先变得可搜索，再由 LightRAG 在后台继续补充实体与关系。这样资料不是导入即结束，而是开始参与后续对话、图谱、路径和资源。简单说，就是资料先可用，关系再增强。

## 09｜仪表盘与三色卡片：一眼分清从哪来、哪里没会、哪里已掌握

**本页唯一主旨**：知识图谱呈现课程结构，三色卡片盒呈现知识所处的学习状态。

**页面形态**：仪表盘全景＋局部放大。先让评委看整体，再解释左侧统计和三类卡片。

**PPT 上屏文字**

主标题：

> 在同一张图上，  
> **看见课程结构，也看见掌握状态**

三张大卡：

> 文献卡：资料从哪里来  
> 灵感卡：尚未掌握  
> 永久卡：已经掌握

三张卡解释完后换成图谱三问，不与卡片说明同时堆放：

> 卡片回答：知识处于什么状态  
> 连线回答：知识之间是什么关系  
> 星团回答：知识属于哪个主题

底部小句：

> 借鉴卢曼卡片盒的原子化、连接与长期沉淀思想

**视频必须拍到**

- 资料解析后，仪表盘出现新增节点或关系；
- 左侧卡片盒统计与图谱状态对应；
- 三类卡片颜色清晰可辨；
- 至少点亮一条真实关系和一个真实星团，让评委知道图谱不是随机排点；
- 镜头最终停在一张真实灵感卡上。

**对应口播**

回到仪表盘，刚才的资料已经变成知识节点，并与原有内容形成关系。左侧是三色卡片盒：文献卡保存资料来源；灵感卡代表尚未掌握；永久卡代表已经通过学生输出和审核，确认掌握。三种颜色不是装饰，而是知识状态；连线说明知识关系，星团说明主题归属。卡片、连线和星团都是真实学习对象，后续可以被 Agent、画像、路径和资源共同使用。

**真实性边界**

可以说系统借鉴卢曼卡片盒的原子化、连接与长期沉淀思想；不能说“文献、灵感、永久三种颜色和掌握含义就是卢曼原始体系”。

## 10｜打开灵感卡：进入一处尚未掌握的知识

**本页唯一主旨**：图谱上的知识点可以无缝进入学习工作台，同一个对象不会在页面之间断裂。

**页面形态**：8 秒左右的过渡动作，不单独做静态解释页。

**PPT 上屏文字**

> 点开一张灵感卡，  
> **就是进入一处尚未掌握的知识**

小字幕：

> 同一知识点：图谱 → 工作台 → 审核 → 再回到图谱

当工作台完成打开时出现 Harness 动作字幕：

> Current Focus 已绑定：当前仓库｜当前路径｜当前卡片

**视频必须拍到**

- 在仪表盘图谱上点击刚才停留的灵感卡；
- 工作台打开同一知识点；
- 标题或 Current Focus 能证明对象没有变化；
- 不能进入空白聊天页后再靠口播补背景，必须让评委看到绑定对象。

**对应口播**

现在，小林在图谱上打开一张灵感卡。Harness 先把当前仓库、路径和卡片绑定到 Current Focus，接下来所有提问、回答、记录和审核都围绕同一个尚未掌握的知识点发生。学生不需要每次重新向 AI 解释自己正在学什么。

## 11｜双 Agent 工作台：通用 AI 帮你输入，AXIOM 要求你输出

**本页唯一主旨**：Agent A 负责追问，学生负责表达，Agent B 负责保留证据；AI 不能替学生制造“我的理解”。

**页面形态**：核心对话实录。视频占主画面，三角色说明贴着真实界面出现，不另做角色介绍页。

**素材状态**：**待补录或连续性复验**。第 10—12 页必须属于同一知识点、同一工作台状态；旧知召回要发生在 Agent A 的提问过程中，不能插成一个打断操作链的独立步骤。

**PPT 上屏文字**

主标题：

> AI 可以解释，  
> **真正的理解必须由学生说出来**

角色标签：

> Agent A：苏格拉底式追问  
> 学生：费曼式输出  
> Agent B：保留原话与证据

第 10 页已经证明对象绑定，本页只保留另外三个 Harness 动作；出现下一条时上一条退出：

> 画像注入：按当前基础、卡点与验收方式追问  
> 旧知召回：从真实旧卡找到可迁移的桥梁  
> 证据写回：只保存学生真正说过的话

过程问题标签：

> 预测｜为什么｜例子｜反例｜边界

当旧永久卡被真实召回时，再出现一条短字幕：

> 旧永久卡 → 语义召回 → 为当前问题搭桥

**视频必须拍到**

- Agent A 不是先给完整标准答案，而是先让学生预测或解释；
- 追问开始时短暂显示本轮教学策略，例如“先例子后定义 / 一次一个机制 / 用迁移验证”，证明画像真的进入了教学；
- Agent A 提问过程中召回一张真实旧永久卡，打开其正文与来源，并把它用于形成类比或追问；
- 当前卡仍明确显示为待审核的灵感卡，证明旧知召回不能替当前知识通过审核；
- 学生至少完成一段有内容的原创回答；
- Agent A 根据该回答继续追问反例、边界或新情境；
- Agent B 在后台保存学生原话、更新灵感卡或画像；
- 不能出现 Agent B 替学生生成第一人称“我的理解”后直接写入。

**对应口播**

进入工作台后，Agent A 不急着把标准答案讲完，而是先读取当前卡片和已经形成的教学策略，让小林预测和解释。系统再从当前仓库的语义索引中召回一张过去已经掌握的相关永久卡，并回到真实来源，用旧知识为当前问题搭桥。随后 Agent A 继续追问为什么、例子、反例和边界，倒逼小林完成费曼式输出。新的学习信号会触发 Agent B 对相关画像维度做增量分析，系统还会周期复核；只有证据支持的变化才写回。学生负责讲，AI 负责问、找桥梁和记录；AI 可以整理表达，但不能替学生制造“我的理解”。旧知识只能帮助理解，不能替当前知识通过审核。

## 12｜掌握审核：没讲清，就不能算掌握

**本页唯一主旨**：永久卡代表已经掌握，因此必须经历真实审核；失败不是瑕疵，而是系统价值最强的证据。

**页面形态**：本片最重要的一段连续实录。第一次失败、继续追问、补充回答、第二次通过、青色变紫色必须属于同一知识点。

**素材状态**：**待补录或连续性复验**。必须和第 10—11 页连成一条母视频，并核对节点名、学生原话、卡片内容、失败原因和第二次通过依据始终一致。

**PPT 上屏文字**

主标题：

> **没讲清，就不能算掌握**

审核标准分成两个层次，不能画成四个同级标签：

> **知识质量**：清晰｜准确｜必要  
> **掌握证据**：学生解释｜反例与边界｜陌生情境迁移

白话收束：

> 这份知识值得留下，学生换个情境仍然会用，才算通过

过程字幕逐条替换，禁止同时堆在画面上：

> 第一次审核：缺少边界与反例 → 未通过  
> 继续学习：只追问这个缺口  
> 换个情境：仍能独立讲清  
> 第二次审核：通过  
> 永久卡＝已经掌握

**视频必须拍到**

- Agent B 只能提交审核，不能直接修改为永久卡；
- 第一次审核显示具体失败原因，卡片仍为灵感卡；
- 失败原因必须明确对应“清晰 / 准确 / 必要 / 迁移”中的一项或多项，不能只给一个总分；
- 失败原因直接改变下一轮问题；
- 学生补充边界、反例或新情境作答；
- 第二次审核能回到学生原话与审核依据；
- 卡片由灵感卡升级为永久卡，掌握状态同步变化。

**对应口播**

永久卡代表已经掌握，所以它不能由 AI 随手生成。审核也不是看文字是否流畅，而是依次问四件事：有没有讲清，是否有依据，这条知识是否值得独立留下，换个情境还能不能使用。第一次审核发现，小林虽然说出了基本机制，却缺少关键边界和反例，因此这张卡仍然是灵感卡。系统把失败原因送回工作台，Agent A 只针对这个缺口继续追问。小林补上反例，并在新的情境里重新解释。第二次审核能够回到他的原话、依据和新作答，因此通过；灵感卡升级为永久卡，代表这项知识达到本次掌握标准。失败不是流程瑕疵，而是系统知道下一步究竟该补什么的证据。

**真实性边界**

“永久卡＝已经掌握”是 AXIOM 主流程中的统一口径，含义是该知识已完成学生输出并达到本次审核标准；不延伸为“从此永远不会遗忘”或“AI 的判断绝对不会出错”。

## 13｜个性化资源：缺什么，补什么；偏好决定怎么补

**本页唯一主旨**：资源生成先判断是否真的需要，再根据当前缺口与学习偏好形成候选，并由学生决定是否执行。

**页面形态**：同一条连续资源实录，先展示主动请求，再展示系统建议，最后快速预览六类资源。

**素材状态**：主动请求与六类资源已有候选画面；**系统主动建议仍需补录并核验**，必须拍到当前缺口、画像偏好、推荐理由、证据强度、预期结果、接受 / 忽略，以及学生确认后才执行。

**PPT 上屏文字**

主标题：

> 知识缺口决定补什么，  
> **学习偏好决定怎么补**

页面最先出现一条四步主线，后续所有信息都围绕它替换：

> 发现缺口 → 形成候选 → 说明理由 → 学生确认

第一状态｜先判断要不要生成：

> 已掌握或已有等价资源 → 不重复生成  
> 旧知相关但不等价 → 只用于类比  
> 确有学习缺口 → 再选择资源与任务

两种方式：

> 学生主动请求：我知道自己想要什么  
> 系统提出建议：我还不知道该补什么

系统建议必须显示：

> 来源｜理由｜证据强度｜预期结果｜接受 / 忽略

六类资源只用动词解释：

> 文档＝讲解｜导图＝梳理｜题目＝验证  
> 代码＝实操｜图示＝说明｜视频＝演示

以上信息必须分阶段替换：先判断“要不要生成”，再出现“主动请求 / 系统建议”，建议卡出现后只保留推荐依据，最后进入六类资源时再换成六个用途；不能把全部文字同时堆在屏幕上。

工程上支持 10 种输出类型，但主片只按六种学习用途归类展示；格式数量放答辩备用，不能拿“后缀更多”冒充创新。

**视频必须拍到**

- 学生主动选择一种资源类型并打开真实结果；
- 系统根据当前缺口、画像偏好或本轮对话提出另一项建议；
- 系统建议必须先以候选状态出现，不能在学生确认前改变正式卡片、图谱或路径；
- 建议说明理由，学生可以接受或忽略，确认后才执行；
- 至少展示一次“已有等价内容，因此不重复生成”或“相关旧知只用于类比”的克制判断；
- 六类资源有真实内容缩略图，至少完整打开两类；
- 不能用同一段文字换六个文件后缀冒充六类资源。

**对应口播**

掌握一项知识以后，系统会面对新的缺口，但它不会立刻生成更多内容。它先检查学生是否已经掌握、是否已有等价资源、旧知识只是相关还是能够替代；只有确有缺口，才决定补什么。学生知道自己需要什么时，可以主动请求；不知道该补什么时，规则与学习证据先从画像、路径和图谱中发现候选，AI 只负责审核、过滤和整理理由，不能把薄证据抬成高置信事实。无论置信度多高，学生都只能选择接受或忽略，确认后才真正写入资源、任务或关系。价值不在生成了多少，而在是否补得必要、补得合适，同时不污染长期知识库。

**真实性边界**

“一次请求生成一个任务和一个预览”只是基础能力，不作为创新点。创新点是资源选择被真实学习证据约束，并进入下一步学习闭环。

## 14｜知识图谱：同一份证据，可以从不同问题来读

**本页唯一主旨**：图谱不是 3D 装饰；同一份真实学习对象可以投影成不同认知问题，并继续指出下一步。

**页面形态**：同一知识节点连续切换图谱视角。视频全屏，当前模式名与它回答的问题贴着控件出现。

**素材状态**：**待复验，必要时补录**。必须使用第 12 页审核完成后的同一仓库状态；证据模式需要真实打开原始资料或对话，地形与任务模式需要显示刚刚升级后的结果。

**PPT 上屏文字**

主标题：

> 同一份学习证据，  
> **可以从不同问题来读**

使用真实控件名称，只展示六个代表视角：

> 星系：我整体学了什么  
> 平面：哪些知识真实相关  
> 邻域：这个概念周围有什么  
> 任务：下一步应该怎么走  
> 地形：哪里尚未掌握、哪里已经掌握  
> 证据：这个结论凭什么可信

收束字幕：

> 图谱不只展示“已经有什么”，还暴露“下一步缺什么”

最后只保留三个结果词：

> 看结构｜查证据｜定下一步

**视频必须拍到**

- 始终围绕同一个真实知识节点切换模式；
- 控件必须真实点击 `星系 / 平面 / 邻域 / 任务 / 地形 / 证据`，字幕不能伪造不存在的模式名；
- 证据模式突出来源节点和证据关系；只有当前界面真实支持继续点击来源时，才拍到回跳原始资料；
- 地形模式能看见刚才灵感卡升级为永久卡的结果；
- 任务模式显示已完成状态，并突出下一个可用节点；
- 视角在变，但节点身份和底层证据没有变化。

**对应口播**

最后回到同一张知识图谱。星系看整体知识域，平面看真实关系，邻域围绕当前概念展开；任务看路径队列与下一个可用节点，地形看掌握状态，证据突出来源节点与证据关系。刚刚通过审核的节点显示为已完成，新的缺口成为下一学习节点。图谱不是换一种皮肤，而是把同一份学习证据投影成六个明确问题，并让结构继续反过来影响路径和资源。

## 15｜技术创新：创新不在接入大模型，而在重做学习判断

**本页唯一主旨**：把前面已经看见的动作归纳成七项关键机制，讲清自研垂直 Agent Harness 与普通 Agent 的根本区别。

**页面形态**：中间是 `AXIOM Agent Harness`，自身作为第一项创新；外圈六个机制块只保留关键词，并各自回看一个真实画面，不引入新功能。

**PPT 上屏文字**

主标题：

> 创新不在“接入大模型”，  
> **而在让学习判断可追溯、可验证、可复用**

七项关键机制：

三组传播结构：

> **可验证**：学生输出必须经过审核  
> **可解释**：画像、推荐与关系都有来源  
> **可继续**：证据会改变下一次学习

> **1｜垂直 Agent Harness**：绑定对象、装配上下文、治理工具、路由证据、写回状态  
> **2｜掌握授权**：输出证据先过审核  
> **3｜证据画像**：画像真正改变教学  
> **4｜前台教学＋后台分析**：教学与记录不互相打断  
> **5｜克制型资源与推送**：先判断需要，再由学生确认  
> **6｜业务真源＋双速语义**：事实可回源，旧知快召回，关系后增强  
> **7｜认知图谱与行动反馈**：看结构、查来源、定下一步

中央收束句：

> 普通 Agent 主要解决“怎样回答”；AXIOM Harness 还决定“在哪学、能做什么、留下什么证据、下一步怎样改变”

**画面放什么**

- 中央 Harness 先单独出现，并用 `对象绑定 / 上下文装配 / 工具治理 / 证据路由 / 状态写回` 五个动词完成定义；
- 外圈依次回看审核失败、画像策略、双 Agent 对话、资源候选、旧卡召回与双速语义、图谱切换六个短镜头；
- 七项内容不能同时以同等密度常驻。先出现中央 Harness，再分两轮出现外圈机制，最后只保留关键词形成全图；
- `Agent Harness` 不是另一个聊天机器人，也不是底层大模型；它是控制学习对象、工具边界、证据流向和状态写回的运行框架。

**对应口播**

回看刚才的过程，AXIOM 的创新不是简单接入大模型。第一，我们在通用 Agent 外自研了掌握学习 Harness，绑定对象、装配上下文、治理工具，并按证据类型选择性写回长期状态。第二，灵感卡必须经过学生输出与掌握审核才能成为永久卡。第三，六维画像不是标签，而会改变下一轮的起点、讲法、节奏和验收。第四，前台 Agent 保持苏格拉底式教学；新信号触发后台增量分析，系统再周期复核，不打断学生。第五，资源先判断是否必要，再形成带来源的候选，由学生确认。第六，业务库保存事实，Qdrant 快速召回旧知，LightRAG 在后台补深层关系。第七，知识图谱不只展示结果，还把同一份证据投影成结构、来源、状态与下一步。普通 Agent 主要解决怎样回答；AXIOM Harness 还决定这次学习发生在哪里、AI 能做什么、学生留下什么证据、下一轮怎样改变。

**真实性边界**

不能说系统“自研了底层大模型”或“彻底消除幻觉”。底层模型、`pi-agent-core / pi-ai`、Qdrant 与 LightRAG 是外部基础能力；AXIOM 自研的是垂直学习编排、对象约束、证据协议、工具治理和状态闭环。可以说系统通过事实回源、学生原话、审核门槛和用户确认，降低未经验证内容直接进入长期知识库的风险。

## 16｜落地价值：把学习过程变成三方都能使用的证据

**本页唯一主旨**：用行为证据与运行数据说明，AXIOM 沉淀的学习状态可以被下一次学习、教师干预和课程建设继续使用。

**页面形态**：先出现三方价值，再把三栏缩成顶部标签，底部出现“行为证明＋性能证明”；不单独做测试数据页。

**PPT 上屏文字**

主标题：

> AXIOM 保存的不是聊天记录，  
> **而是可以继续工作的学习状态**

三方价值：

> 学生：知道哪里没掌握，也知道为什么  
> 教师干预：可获得原始误解、追问过程和通过依据  
> 课程：保留可复用的知识、资源与学习证据

行为证明：

> 同一知识点真实驳回 → 针对缺口修正 → 再次审核通过  
> 旧知识跨会话召回并回到原卡｜资源与路径随证据改变

规模背景，不作为效果指标：

> 长期黄金仓库：312 张卡片

性能测试，使用其中 96 张真实卡片工作集：

> 快速索引及验证：81.25 秒｜热模型语义查询：0.25 秒

上屏角落小字：

> 本机 bge-m3 单并发历史实测；最终录制前复测

三方价值、行为证明与运行数据必须分三次出现。运行数据只证明工程链路可用和反馈及时，不直接等于学习效果提升。

**画面放什么**

- 学生栏配审核失败原因与下一步；
- 教师干预栏配学生原话、追问和审核依据，不暗示已经交付独立教师端；
- 课程栏配长期卡片、图谱关系和资源沉淀；
- 行为证明必须回看刚才同一条母视频，不能使用三段互不相关的截图；
- 性能数据只作为右下角的小型工程备证，标注测试对象和条件；不用“卡片越多越成功”的逻辑，也不抢三方价值的视觉主位；
- 最终录制前重新运行对应验证脚本并保存日志；若结果变化，以新实测替换，不沿用旧数字。

**对应口播**

对学生，系统说明哪里没掌握、为什么没通过、下一步补什么；对教师干预，它保留学生原话、误解位置和审核依据；对课程建设，它沉淀可复用的知识、资源和学习证据。刚才同一个知识点经历了真实驳回、针对性修正和再次通过，旧知识也能跨会话回到当前问题。工程实测中，96 张真实卡片完成快速索引和验证用时 81.25 秒，热模型语义查询约 0.25 秒，证明前台召回可以及时，而深层关系继续在后台增强。这些数据证明系统链路真实可运行，不把它夸大成已经完成学习效果对照实验。

**数据使用边界（内部制作说明）**

性能数字来自 `11-两分钟语义索引与后台图谱增强.md` 的本机 `bge-m3` 单并发历史实测记录；312 张卡片来自同一长期黄金仓库口径。这不是通用 SLA。最终录制前必须重新执行性能与黄金案例验证，保留日志或截图作为答辩备证；若复测失败或环境改变，应删除旧值，不能把历史快照说成现场结果。

## 六、PPT 与视频的融合规则

### 1. 静态 PPT 负责“建立判断”

第 01—04、15—17 页负责回答：为什么、是什么、为什么有效、有什么价值。标题必须是一个可以独立成立的判断，不能写“功能介绍”“系统架构”“创新点展示”这种目录标题。

### 2. 产品视频负责“提供证据”

第 05—14 页默认使用真实产品实录。每个动作必须同时出现：

```text
用户做了什么
  -> 页面发生什么变化
  -> 这项变化意味着什么
```

如果只拍到点击，没有结果，不算证明；如果只有配音声称能力存在，页面没有对应变化，也不算证明。

### 3. 文字必须贴着对应画面出现

- “画像有来源”只在原话来源被打开时出现；
- “画像改变教学”只在本轮策略或下一轮行为变化可见时出现；
- “对象绑定”只在 Current Focus 显示当前仓库、路径或卡片时出现；
- “资料先可用、关系再增强”必须贴着真实导入状态或独立机制简图出现，不能伪装成产品 UI；
- “Agent B 记录证据”只在后台记录或卡片更新发生时出现；
- “第一次审核未通过”只在失败结果真实出现时出现；
- “清晰 / 准确 / 必要”只在审核面板显示对应依据或失败原因时出现；
- “永久卡＝已经掌握”只在卡片升级发生时出现；
- “旧知迁移”只在真实旧卡被召回时出现；
- “学生可以忽略”只在接受 / 忽略入口同时可见时出现；
- “系统不会重复生成”只有在真实抑制结果或明确判断被展示时出现；
- “下一步改变”只在已完成状态与下一个可用节点同时可见时出现。

### 4. 视频可以移动，但不能失去上下文

- 解释产品动作时，视频保持全屏或占画面 70% 以上；
- 需要补充机制时，可把视频缩到右侧 55%—65%，左侧显示一句结论和一张简图；
- 机制解释结束后，视频立即回到主画面；
- 不要长期用小视频配大段文字；
- 不要在同一条连续操作中黑场或突然切到无关页面。

### 5. 一页只允许一个视觉主语

如果主标题已经是视觉中心，解释文字就必须弱化；如果真实视频正在发生关键变化，标题就要缩小退出，把视觉主位让给证据。主标题、视频、说明卡片不能同时争夺第一注意力。

### 6. 文字密度必须有硬上限

- 静态页主标题最多两行，辅助信息最多三组；
- 视频页同一时刻最多保留一个结论标题和一条动作字幕；
- 架构、资源和图谱页可以包含更多信息，但必须随讲解逐项出现、逐项退出；
- 任何正文需要评委停下来阅读超过 3 秒，就不应直接常驻在 PPT 上，应移入口播或导演备注。

## 七、三个媒介必须分别通过的理解测试

### 只看 PPT 上的文字

评委应该能够回答：

1. AXIOM 是给谁使用的什么软件；
2. 它解决了“输入很多但掌握不可见”的问题；
3. 学生必须亲自输出，永久卡必须经过清晰、准确、必要与迁移审核；
4. 画像不是标签，而是会改变下一轮教学的控制参数；
5. 资源和关系先作为带依据的候选，学生确认后才执行；
6. 旧知识能够参与下一次学习，但检索不能代替掌握审核；
7. Harness 负责把对象、工具、证据与下一步接成闭环。

### 只听配音

评委应该能够复述完整因果链，而不是只听到一串功能名。所有技术词第一次出现后，必须立即翻译成学生收益或页面变化。

### 静音看产品视频

评委应该能够辨认：

1. 学生先进入仓库并建立画像；
2. 画像判断能回到学生原话，并说明将怎样改变教学；
3. 资料导入后变成知识节点，并进入可召回状态；
4. 同一张灵感卡绑定 Current Focus 后进入工作台；
5. 学生回答，Agent B 记录真实证据；
6. 第一次审核失败，继续打磨，第二次通过；
7. 灵感卡变成永久卡；
8. 资源有主动与建议两种触发，建议可接受或忽略；
9. 图谱以真实控件切换不同认知视角，并明确突出下一个可用节点。

任何一项无法仅凭画面辨认，都要通过录制动作、局部放大或短字幕补强，而不是用更长配音掩盖。

## 八、制作前真实性门禁

- [ ] 产品定位始终是“给高校学生使用的 AI 掌握学习系统”。

- [ ] 永久卡升级来自学生原始输出和审核，不来自 AI 自动代写。
- [ ] “永久卡＝已经掌握”仅指达到本次审核标准，不延伸为永不遗忘或系统绝对正确。
- [ ] Agent A 是苏格拉底式提问，不写成维特根斯坦式提问。
- [ ] 学生完成费曼式输出，Agent B 只记录和整理证据。
- [ ] 新学习信号触发相关画像维度的增量分析，系统还会周期复核；不宣称每轮重写全部画像。
- [ ] 画像至少体现“学什么、会什么、怎么讲、哪里卡、讲多少、怎么算会”，并能改变教学动作。
- [ ] 第一次审核失败与第二次通过属于同一学生、同一仓库、同一知识点。
- [ ] 审核失败给出清晰、准确、必要或迁移中的具体原因，而不是只给总分。
- [ ] 向量召回只用于搭桥、去重或改变追问，不能直接证明当前知识已掌握。
- [ ] 业务事实源、Qdrant 快速语义层和 LightRAG 后台图谱层的职责不能混写。
- [ ] 系统资源建议显示理由和用户选择权，确认后才生成。
- [ ] 已掌握、已有等价资源或只有弱相关时，系统不会把“多生成”包装成个性化。
- [ ] 六类资源有不同学习用途，不是同一文本更换格式。
- [ ] 图谱演示使用真实控件名 `星系 / 平面 / 邻域 / 任务 / 地形 / 证据`，并始终基于同一份底层证据。
- [ ] Harness 被准确描述为自研学习运行框架，不宣称自研底层大模型或第三方基础组件。
- [ ] 不把单次资源生成任务包装成技术创新。
- [ ] 不宣称彻底消除大模型幻觉。
- [ ] 不把卡片总数直接等同于掌握数量。
- [ ] 0.25 秒与 81.25 秒性能数据在最终录制前重新实测，并保留日志或截图。
- [ ] 最终页是愿景，不展示完成 / 未完成清单，不黑屏。

## 九、一句话制作原则

> **每一页只说清一个判断；每一段视频只证明一个变化；每一句配音只解释这项变化为什么有价值。**
