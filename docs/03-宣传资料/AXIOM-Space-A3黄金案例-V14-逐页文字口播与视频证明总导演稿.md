## 一、全片只讲一个核心机制

**传播钩子**

> 当答案变得无限便宜，真正稀缺的是：问出那个能够暴露真实缺口的问题。

**产品机制**

> 通用 Agent 擅长生成答案；AXIOM 先用好问题暴露学生自己还不知道的盲区，再让每一次判断都受证据约束，让教学、资源和路径随着真实表现改变，直到换一道题仍然会。

评委看完后，应该能够用一句话复述：

> 通用 Agent 让答案变便宜；AXIOM 用好问题暴露盲区，用证据改写下一步，直到真正学会可以被证明。

全片隐藏在故事背后的产品原则是：

> **AI 可以提出判断，证据决定判断能走多远；AI 可以建议下一步，学生确认和评估结果决定系统能不能真的改变。**

全片只保留三个一级记忆锚点：

1. 好问题暴露盲区。
2. 证据改变下一步。
3. 换一道题仍然会，才算掌握。

15 页只分成四个叙事阶段：

| 阶段 | 页码 | 只回答一个问题 |
|---|---|---|
| 一、发现盲区 | 01A—02B | 小林已经学过，系统怎样发现他其实错在哪里？ |
| 二、让证据改变学习 | 03A—05B | 找到缺口以后，证据怎样决定跳过、复用、沉淀和推送？ |
| 三、证明真正掌握 | 06A—06B | 怎样证明小林不是看懂了，而是换一道题仍然会？ |
| 四、沉淀可复用证据 | 07A—08A | 一次掌握怎样继续服务学生，并成为教学者与课程可以复用的证据？ |

完整因果链：

```text
小林已经学过并发、能够复述定义，却在真实库存任务中判断错误
  -> 进入 Dashboard，看见他已经积累的知识与待验证能力
  -> 系统提出一个能够暴露缺口的预测问题
  -> 预测暴露隐藏误解
  -> 原话形成一个可以被推翻的学习假设
  -> 下一轮只改变一种教法，再用表现验证
  -> 已掌握内容跳过，相关旧知参与迁移，真实缺口继续追问
  -> 只有学生自己的表达进入“我的理解”，再沉淀为可审查知识
  -> 图谱与画像共同决定连接推送和必要资源
  -> 陌生迁移确认掌握
  -> 评估改写路径
  -> 学习状态跨时间延续
  -> 每个学生的误解、干预、迁移结果和有效资源按同一结构保存
  -> 零散聊天第一次具备汇聚为课程学习资产的条件
```

## 二、A 页、B 页和音频的职责

### A 页：让评委听懂

- 每页只回答一个问题。
- PPT 文字先给结论，再给最短机制。
- 口播负责把上一页自然带到这一页，并说清楚“为什么”。
- A 页不播放产品视频，只用静态信息建立理解框架。

### B 页：让评委相信

- 每段视频只证明一个状态变化，不承担完整教学。
- 页面必须固定显示“本段只证明什么”。
- A 页已经讲过的原理，B 页不再复述；B 页口播只点名画面中的输入、状态变化和结果。
- 口播由 HTML 独立播放，不写进视频文件；视频保持原始产品声音静音。
- 口播说到哪一步，视频就展示哪一步，禁止口播谈画像、画面却停在登录页。
- 除 01B 可保留极短的首次进入工作台外，02B 以后全部从已准备好的业务状态直接开始，不再登录、不再重复选课。

### 最终视频数量

| 观点页 | 是否需要下一张视频页 | 视频只证明什么 |
|---|---:|---|
| 01A 产品定义、Dashboard 与卡片盒 | 需要 01B | 一个预测问题能够暴露正确答案下面的隐藏误解 |
| 02A 可证伪画像与教学改变 | 需要 02B | 学生原话形成可纠正的假设，并真正改变下一问 |
| 03A 语义复用门与 Galaxy 多视图 | 需要 03B | 已会内容被抑制、旧知用于迁移，同一知识可按不同问题切换视图 |
| 04A 三类卡片与 Galaxy | 需要 04B | 内容先被驳回，补足证据后才进入长期知识与图谱 |
| 05A 图谱、画像与精准推送 | 需要 05B | 两类建议有依据，用户确认后才生成真实资源或关系 |
| 06A 陌生迁移与动态路径 | 需要 06B | 掌握证据真正改写了能力状态和学习路径 |
| 07A 长期状态与教师追溯 | 需要 07B | 学习可以继续，系统判断可以返回原始证据 |
| 08A 课程级价值收束 | 不需要 | 个体证据如何从零散聊天变成可汇聚、可复用的课程资产 |

> 系统没有继续讲答案，而是提出一个能够检验因果模型的预测问题，并由此暴露小林自己还不知道的缺口。

### PPT 固定上屏文字

**证明标签**

> 本段只证明：一个好问题，比再讲一遍答案更早暴露缺口

**主标题**

> 小林预测：B 会读到 0  
> **真实结果：A、B 都先读到 1**

**三段状态条**

```text
先预测  ->  运行交错过程  ->  记录具体误解
```

**结果标签**

> 隐藏缺口：把“读取、判断、写回”误认为不可打断的一步

### 本页完整口播

点击继续，预测现场恢复。小林先留下“B 会读到 0”的预测，运行结果却显示 A、B 都先读到 1。系统没有只判错，而是把“读取、判断、写回被想成完整一步”连同原话和时间记录下来。预测、冲突和具体缺口同时出现，才证明问题问到了盲区。

### 视频中需要点击

## 02A｜学生说错一句，下一问立即改变

### 本页目的

解释 AXIOM 怎样持续问出“刚好击中当前缺口”的问题：系统不急着给小林贴标签，而是依据他的原话提出一个可以被纠正的判断；下一轮只改变一种教法，再用新的表现检验这个判断。

### 评委看完本页只需记住

> 个性化不是给学生贴得更准的标签，而是让下一步因真实证据而改变。

### PPT 固定上屏文字

**主标题**

> 学生说错一句  
> **下一问立即改变**

**副标题**

> 好问题不是更难，而是刚好能够检验当前判断是否成立。

**核心因果链**

```text
“B 会读到 0”
  -> 系统提出假设：小林把并发动作想成顺序执行
  -> 下一轮只改变一种教法：逐步排列时间线
  -> 新表现验证或推翻这个假设
```

**画像不再是标签，而是一份可检验的约定**

> 依据哪句话｜当前怎样判断｜这轮只改什么｜怎样算有效｜失败后怎样换方法

**三条控制原则**

> 证据不足，只追问，不下结论  
> 小林可以纠正，旧判断随即停止使用  
> 一轮只改一种教法，才知道改变是否有效

**页脚结论**

> 画像不是结论，而是一个必须接受下一次表现检验的假设。

### 本页完整口播

小林说“B 会读到 0”，系统没有立刻给他贴上“并发基础薄弱”的标签，而是提出一个可以被推翻的判断：他可能把读取、判断和写回想成了顺序执行。下一轮只改变一种教法，让他亲手排列时间线；如果仍然无效，就换一种方法。画像不再是一份描述，而是一次有证据、能纠正、会改变下一步的教学尝试。

### 视频决策

需要。下一页 02B 必须连续展示“原话形成判断”和“下一问已经改变”，不能只拍一张静态画像图。

### 与下一页的连接

口播结束在“下一问改变”。视频开场直接从小林刚才的原话开始，沿证据链走到新的问题。

# 第二阶段：让证据改变学习

## 03A｜只学不会的，用会的帮助不会的

### 本页目的

## 五、音频与视频统一制作合同

### 音频

- 每页口播单独生成一条音频，由 HTML 根据页码自动播放。
- 视频文件不内嵌配音，避免后续改稿时重新压制视频。
- A 页口播可以完整解释；B 页口播必须严格描述当下正在发生的画面。
- B 页不重复 A 页已经讲过的原理，只点名输入、前后状态、证据来源和最终结果。
- 语气采用自然讲述，不使用“这一页我们看到”“接下来给大家展示”等制作腔。
- 同一页音频结束后再允许自动翻页；用户手动翻页时立即停止上一页音频。

### 视频

- 建议源录制分辨率为 `2560x1440`，最低交付为 `1920x1080`。
- 鼠标光标保持可见，点击应有轻量视觉反馈。
- 每段只保留能证明状态变化的 20—36 秒。
- 关键输入、状态变化和最终结果都必须在可读尺寸下出现。
- 02B 以后不再出现登录页；页面之间沿用同一小林账号和同一黄金案例数据。
- 每段结尾停留 2 秒，让评委看清最终状态后再翻页。
- 02B 必须拍到来源、可信度、证据状态和用户纠正入口。
- 05B 固定分成“为什么推”和“确认后交付”两个章节，章节标签始终可见。

### HTML 页面

- A 页不放视频占位框。
- B 页的视频区域保持 16:9；证明标签、主标题和结果字幕在视频外侧固定显示，不能压住产品关键内容。
- 页面右下角统一保留上一页、播放/暂停音频、下一页和音量控件。
- 音频、视频加载失败时显示明确状态，但不改变页面尺寸。
- 08A 是真正结束页，不创建 08B，也不自动循环。

## 七、最终复述测试

评委看完以后，应当能够不依赖项目术语复述以下五件事：

1. 小林学过并发、能够复述，却仍在真实任务中判断错误；AXIOM 解决的是“看过被误判成学会”。
2. 通用 Agent 可以继续给答案，AXIOM 用好问题暴露学生自己还不知道的盲区，再让原话证据改变下一问。
3. 语义记忆只负责想起相关内容，学习证据才有资格决定小林会不会；建议的确定程度也不能超过证据本身。
4. 只有学生自己的表达能进入“我的理解”；卡片合格不等于能力掌握，换一道题仍然会，评估才有资格改变能力和路径。
5. 同一证据链让学生继续学习，也让教学者看见为什么这样教，并让个体学习过程具备汇聚为课程资产的条件。

如果评委只能记住“画像、知识图谱、六类资源”，却说不出“好问题怎样暴露盲区、AI 的判断为什么必须受证据约束、为什么换一道题才算掌握”，说明演示仍然讲成了功能目录。只有当他能沿着小林从判断错误到独立迁移的变化，继续讲到证据怎样被复用，传播目标才真正完成。
