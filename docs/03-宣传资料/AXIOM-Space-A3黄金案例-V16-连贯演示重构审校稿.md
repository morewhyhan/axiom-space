# AXIOM Space A3 黄金案例 V16｜按真实操作顺序填充版

> 当前状态：只确定 MD 文稿、视频顺序和转场，不修改 HTML。  
> 唯一主线：严格沿着用户实际使用 AXIOM 的顺序向前。  
> 统一口径：灵感卡＝尚未掌握；永久卡＝已经掌握。  
> 提问方法：苏格拉底式提问，倒逼学生进行费曼输出。

---

## 一、唯一正确的叙事骨架

这版不再从“并发预测错误”开场，也不先讲功能和理论。

全片严格按下面的真实操作顺序展开：

```text
进入仓库
  -> 初次画像对话式构建
  -> 查看画像变化
  -> 导入学习资料
  -> 在仪表盘查看资料转换成的知识节点
  -> 在仪表盘图谱中点击一张灵感卡
  -> 在工作台打开这张灵感卡
  -> Agent A 进行苏格拉底式提问
  -> 学生回答并进行费曼输出
  -> Agent B 在后台记录原话、观察和卡片变化
  -> 从永久卡解释为什么需要审核
  -> 审核刚才的灵感卡，第一次没有通过
  -> 继续对话、追问和打磨
  -> 再次审核并通过
  -> 灵感卡升级为永久卡，代表已经掌握
  -> 展示资源推送的两种方式
  -> 展示知识图谱的几种模式
```

后续所有内容只能填进这条主线，不能改变这条主线的先后顺序。

---

## 二、为什么这条顺序是连贯的

它不是按照“画像、图谱、卡片、资源”的功能目录讲，而是每一步都由上一步产生：

```text
进入仓库
  -> 系统还不了解用户，所以先建立初始画像

画像建立完成
  -> 用户需要查看系统怎样理解自己

画像可见
  -> 接下来放入真实课程资料，开始学习

资料导入完成
  -> Dashboard 出现新的节点和卡片

Dashboard 出现灵感卡
  -> 用户点击尚未掌握的内容进入工作台

进入工作台
  -> Agent A 通过提问帮助学生打磨理解

学生回答
  -> Agent B 在后台记录学习证据

形成灵感卡
  -> 因为灵感卡代表尚未掌握，所以必须审核

审核未通过
  -> 返回对话继续追问、解释和修改

审核通过
  -> 灵感卡升级为永久卡，代表已经掌握

掌握机制讲清
  -> 再展示系统如何主动或按需提供学习资源

资源能力讲清
  -> 最后回到知识图谱，展示整个知识库可以怎样被观察
```

这才是“界面动作产生下一步”，而不是靠口播强行连接不同功能。

### 01｜进入自己的课程仓库

**页面类型**：视频页。  
**使用素材**：现有进入 Vault 实录。  
**本页主旨**：AXIOM 是给高校学生使用的长期 AI 学习软件。

**画面动作**

1. 小林处于已登录状态；
2. 选择“Java Web 课程仓库”；
3. 点击进入；
4. 首次画像引导面板出现。

**固定上屏文字**

> 给高校学生使用的 AI 学习软件

> 进入自己的课程仓库，  
> **资料、画像、卡片和学习记录都会在这里持续积累**

**完整口播**

AXIOM Space 是给高校学生使用的 AI 学习软件。小林首先进入自己的 Java Web 课程仓库。这里长期保存他的课程资料、学习画像、知识卡片、对话记录和学习路径。系统不会从一个空白聊天框开始，而是围绕这个课程仓库持续认识他、帮助他学习。

**转场**

> 第一次进入时，系统还不了解小林，所以第一步是通过对话建立初始画像。

### 02—03｜初始画像通过完整对话构建

**页面类型**：一条完整连续视频，可以根据长度分成两页播放，但不能换素材或打断时间线。  
**本页主旨**：画像来自连续对话，不来自静态问卷或 AI 猜测。

**连续源视频必须包含**

1. 点击“开始初始画像”；
2. Agent 提出第一问；
3. 小林回答学习目标；
4. Agent 根据上一回答继续下一问；
5. 小林回答当前基础；
6. 小林回答偏好的学习资源形式；
7. 小林回答主要卡点和反馈习惯；
8. 对话进度持续变化；
9. 出现“初始画像构建完成”；
10. 出现“查看认知洞察”入口。

**固定上屏文字**

> AI 每轮只问一个必要问题，  
> **通过真实对话建立可以继续变化的初始画像**

**完整口播**

Agent 每轮只问一个必要问题，并根据小林上一轮的回答继续追问。它了解学习目标、当前基础、信息偏好、主要阻塞和反馈习惯。这些不是固定标签，而是一份等待以后真实学习表现继续修正的初始画像。

**视频要求**

- 必须录制一条从第一问到完成提示的连续源视频；
- 源视频允许 60—120 秒；
- 成片可以加速、删掉等待时间，但不能打乱问答顺序；
- 第一组问答和最后一组问答必须完整保留；
- 最后必须看到“查看认知洞察”入口。

**转场**

> 对话完成以后，小林直接进入认知洞察，查看系统究竟怎样理解自己。

**页面类型**：视频页。  
**使用素材**：认知洞察实录。  
**本页主旨**：画像不是后台黑箱，用户能查看、纠正并看到变化。

**画面动作**

1. 从上一页“查看认知洞察”直接进入；
2. 查看初始画像；

4. 查看来源、置信度和状态；

**固定上屏文字**

> 画像不是后台标签，  
> **每一条判断都能看到依据，也会被以后表现继续修改**

**转场**

> 系统已经初步认识小林，接下来把真正要学的课程资料放进仓库。

**转场**

**页面类型**：Dashboard 停帧或极简文字覆盖，不单独扩展理论。  
**本页主旨**：建立后续审核所需的统一口径。

**固定上屏文字**

> 文献卡：资料来源  
> 灵感卡：尚未掌握

**口播**

粉色文献卡说明资料从哪里来；青色灵感卡记录正在形成、但尚未掌握的理解；紫色永久卡代表已经掌握。接下来我们就从一张灵感卡开始，看它怎样被真正学会。

## 第三阶段：从仪表盘点击灵感卡，进入工作台学习

### 08｜点击仪表盘上的灵感卡，在工作台打开

**页面类型**：必须连续的视频页。  
**本页主旨**：Dashboard 上的知识节点和工作台中的学习对象是同一个东西。

**连续画面**

```text
Dashboard 图谱
  -> 点击青色灵感卡“并发库存超卖”
  -> 打开卡片详情
  -> 点击进入学习
  -> Forge 工作台打开同一张卡片
```

**固定上屏文字**

> 从仪表盘点击一张尚未掌握的灵感卡，  
> **在工作台继续打磨同一个知识节点**

**转场**

> 进入工作台以后，Agent A 不先给答案，而是先要求小林说出自己的判断。

---

### 09｜Agent A 进行苏格拉底式提问

**页面类型**：与第 08 页连续的对话视频。  
**本页主旨**：通过追问让学生自己的理解显形。

**必须展示**

1. Agent A 提出可以被验证的问题；
2. 小林先做预测；
3. Agent A 追问“为什么”；
4. 小林解释；
5. Agent A 继续要求例子、反例或适用边界。

**固定上屏文字**

> 不急着把答案塞给学生，  
> **先用苏格拉底式提问让他暴露自己的理解**

**口播**

通用 Agent 主要帮助学生输入知识。AXIOM 在工作台中先进行苏格拉底式提问，让小林预测、解释、举例并说明边界。它不是为了把学生问住，而是为了让“我好像懂了”变成可以检查的具体表达。

---

### 10｜学生回答，Agent B 在后台记录

**页面类型**：继续使用同一条对话视频。  
**本页主旨**：Agent A 负责教学互动，Agent B 负责把学生输出变成长期学习证据。

**必须展示的联动**

```text
小林回答
  -> Agent A 继续追问
  -> Agent B 记录小林原话
  -> Agent B 更新画像观察
  -> Agent B 把当前理解写入灵感卡
```

**固定上屏文字**

> Agent A 在前台提问，  
> **Agent B 在后台记录原话、更新画像并写回灵感卡**

**口播**

小林用自己的话讲解读取、判断和写回，这就是费曼输出。Agent A 继续追问，Agent B 不抢占对话，而是在后台保留原话、记录观察、更新画像，并把这次输出写回当前灵感卡。此时它仍然表示尚未掌握。

**向量记忆在这里怎样填充**

如果需要展示向量数据库，不新增一段功能介绍，而是在本次对话中自然出现：

> Agent 根据当前灵感卡，从向量数据库找回小林过去已经掌握的相似永久卡，用旧理解帮助追问当前问题。

它只能帮助迁移旧理解，不能跳过当前卡片的审核。

**转场**

> Agent B 已经记录了这次输出，但灵感卡仍然代表尚未掌握。要升级成永久卡，必须通过审核。

## 第四阶段：从永久卡讲到审核能力

### 11｜永久卡为什么必须审核

**页面类型**：极简文字页。  
**本页主旨**：永久卡既然代表已经掌握，就不能由 AI 自动包装出来。

**固定上屏文字**

> 灵感卡可以记录正在形成的理解，  
> **永久卡必须经过审核，因为它代表已经掌握**

**口播**

永久卡不是“值得长期收藏”的标签，而是已经掌握的状态。因此，Agent B 可以记录灵感卡，却不能直接把它改成永久卡。刚才小林形成的理解必须接受审核。

**转场**

> 现在审核刚才那张灵感卡，第一次并没有通过。

---

### 12｜第一次审核没有通过

**页面类型**：视频页。  
**本页主旨**：没讲清原因、例子或边界，就仍然是灵感卡。

**画面动作**

1. 打开刚才的灵感卡；
2. 点击审核；
3. 系统显示未通过；
4. 明确显示缺少的内容；
5. 卡片仍保持青色灵感卡状态；
6. 点击“继续打磨”返回对话。

**固定上屏文字**

> 第一次审核没有通过，  
> **卡片仍然是灵感卡，也仍然代表尚未掌握**

**转场**

> 审核不是结束，而是告诉小林接下来应该把哪里讲清楚。

---

### 13｜继续对话、追问和打磨

**页面类型**：与审核失败连续的视频页。  
**本页主旨**：审核结果重新进入对话，推动学生补足理解。

**画面动作**

1. 审核指出缺少适用边界；
2. Agent A 针对这个缺口继续追问；
3. 小林重新解释；
4. 小林补充例子、反例或陌生场景；
5. Agent B 更新灵感卡内容；
6. 再次提交审核。

**固定上屏文字**

> 没通过，就回到对话继续打磨，  
> **直到学生能够用自己的话讲清原因、例子和边界**

---

### 14｜再次审核通过，升级为永久卡

**页面类型**：继续使用同一条审核视频。  
**本页主旨**：升级成功就是已经掌握。

**画面动作**

1. 再次提交审核；
2. 显示审核通过；
3. 卡片由青色变为紫色；
4. 状态从“灵感卡”变为“永久卡”；
5. 明确显示“已经掌握”。

**固定上屏文字**

> 审核通过，灵感卡升级为永久卡  
> **永久卡＝已经掌握**

**口播**

补足解释并再次审核后，这张卡片从青色灵感卡升级为紫色永久卡。在 AXIOM 中，这个变化只有一个含义：小林已经掌握了这部分知识。永久卡随后可以改变学习路径，也能在未来的新问题中被向量数据库再次调用。

**转场**

> 除了通过对话打磨知识，AXIOM 还用两种方式为学生提供需要的学习资源。

## 第五阶段：资源推送的两种方式

### 15｜方式一：用户主动请求资源

**页面类型**：视频页。  
**本页主旨**：学生明确想要什么资源，就可以主动生成什么资源。

**画面动作**

1. 小林在工作台提出资源请求；
2. 选择视频、导图、题目、代码等一种形式；
3. 系统创建生成任务；
4. 任务完成；
5. 打开真实预览。

**固定上屏文字**

> 方式一：用户主动请求  
> **想要什么资源，就生成什么资源**

**说明**

“一次请求生成一个任务和一个预览”只是基础能力，不作为本页核心优势。

---

### 16｜方式二：系统根据缺口和偏好主动推送

**页面类型**：视频页。  
**本页主旨**：系统根据当前学习证据，个性化推荐更合适的资源。

**必须展示的依据**

```text
知识图谱
  -> 判断当前缺什么

学习画像
  -> 判断学生偏好什么形式

当前卡片与对话
  -> 判断具体卡在哪里

系统主动推送
  -> 给出此刻更合适的资源
```

**固定上屏文字**

> 方式二：系统主动推送  
> **知识缺口决定补什么，学习偏好决定用什么形式补**

**口播**

第二种方式由系统主动发起。知识图谱判断学生缺什么，学习画像判断他更适合视频、导图、题目还是代码，当前对话说明具体卡在哪里。AXIOM 据此个性化推送最适合此刻的学习资源，而不是给每个人发同一份材料。

---

### 17｜能够生成哪些资源

**页面类型**：视频页。  
**使用素材**：六类资源预览实录。  
**本页主旨**：展示资源范围，但不一次把所有资源塞给用户。

**固定上屏文字**

> 文档、导图、题目、代码、图示与视频，  
> **需要什么，就生成什么**

**口播**

AXIOM 可以生成文档、导图、题目、代码、图示和视频。六类资源分别有真实内容和预览入口，但不会每次全部推给学生。用户主动请求，或者系统根据缺口与偏好主动推送，最终都只服务当前学习任务。

**转场**

> 最后回到知识图谱，看同一份知识库可以从哪些角度被理解。

## 第六阶段：知识图谱的几种模式

### 18｜用不同模式查看同一份知识网络

**页面类型**：视频页，也是整套演示的最后一段。  
**使用素材**：Galaxy 多模式实录。  
**本页主旨**：模式切换不是视觉特效，而是在回答不同问题。

**必须按顺序展示并解释**

1. **星团模式**：这门课程有哪些知识领域；
2. **关系模式**：当前知识点与哪些概念相连；
3. **卡片模式**：哪些来自资料，哪些尚未掌握，哪些已经掌握；
4. **来源模式**：每个知识节点来自哪份资料或哪次对话；
5. **掌握模式**：灵感卡与永久卡在整个知识网络中的分布；
6. **路径模式**：下一步应该进入哪个知识节点。

**固定上屏文字**

> 同一份知识网络，可以回答六个不同问题  
> **从“我有什么”，走向“我会什么、还要学什么”**

**完整口播**

同一份知识网络可以切换不同视角：星团模式看课程领域，关系模式看知识连接，卡片模式看来源与掌握状态，来源模式追溯证据，掌握模式区分灵感卡和永久卡，路径模式决定下一步。AXIOM 最终不是替学生保存更多内容，而是让他看见自己拥有什么、真正掌握了什么、接下来还需要学什么。

**最终停留文字**

> AXIOM Space  
> **让每一次真实输出，都成为可以继续生长的知识**

## 六、哪些内容只能填充，不能改变顺序

### 双 Agent

填在“点击灵感卡进入工作台”之后：

- Agent A 前台提问；
- Agent B 后台记录。

不能在进入工作台以前先讲架构。

### 费曼学习法

填在“学生回答”中：

- 学生用自己的话讲；
- 解释原因、例子和边界；
- 不是复述模型答案。

### 向量数据库

填在“工作台打磨灵感卡”与“永久卡未来复用”中：

- 找回过去相关的永久卡；
- 用旧理解帮助当前提问；
- 永久卡形成后可以在未来再次被调用。

不能单独插入一页数据库技术说明打断操作链。

### 个性化资源

填在“资源推送的两种方式”中：

- 图谱决定缺什么；
- 画像决定偏好什么；
- 当前对话决定具体卡点；
- 最终生成或推送相应资源。

### 长期价值

不单独打断主线，用三处真实动作体现：

1. 画像持续变化；
2. 永久卡可以被向量数据库再次调用；
3. 知识图谱持续显示掌握状态与下一步。

## 七、最终连贯性检查

一个完全不了解 AXIOM 的评委看完后，应该能按顺序复述：

1. 小林进入自己的课程仓库；
2. 通过连续对话建立初始画像；
3. 在认知洞察中查看画像变化；
4. 导入课程资料；
5. 在 Dashboard 看见资料变成的节点；
6. 点击一张灵感卡进入工作台；
7. Agent A 进行苏格拉底式提问；
8. 小林回答，Agent B 记录；
9. 灵感卡第一次审核失败；
10. 小林继续对话打磨；
11. 审核通过，升级为永久卡；
12. 资源可以由用户主动请求，也可以由系统主动推送；
13. 最后在知识图谱中切换不同模式查看整个知识库。

如果评委能复述这十三步，整套演示才真正连贯。
