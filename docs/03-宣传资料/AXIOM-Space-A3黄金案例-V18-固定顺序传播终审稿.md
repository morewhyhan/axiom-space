## 一、全片只能有一个核心

> **AXIOM 不以给出答案为终点，而是让学生输出，用证据确认掌握，再让这份证据改变下一次学习。**

全片的所有功能只能证明这一个核心，不能与它并列争夺记忆：

```text
学生输出
  -> Agent B 保留原话与学习证据
  -> 审核判断是否达到掌握标准
  -> 灵感卡升级为永久卡
  -> 画像、资源和学习路径随证据改变
  -> 旧理解在下一次学习中被重新调用
```

### 面向评委的真实对标

| 现有工具 | 擅长什么 | 通常仍由学生承担什么 | AXIOM 的差异 |
|---|---|---|---|
| 通用 AI | 解释、生成、问答 | 发现自己的盲区，判断是否真正掌握 | 要求学生输出，并把输出变成可追溯证据 |
| 课程平台 | 组织课程与固定练习 | 让内容适应个人缺口 | 画像与真实表现改变提问、资源和路径 |
| 笔记工具 | 保存资料与笔记 | 主动教学、审核理解、安排下一步 | 卡片状态与审核进入同一学习闭环 |

公开表达不能说“通用 Agent 完全不能提问或帮助输出”。真正差异不是单点能力，而是：

> **一份学生原话会连续改变画像、卡片、审核、资源和路径。**

## 二、不能改变的操作顺序

```text
进入仓库
  -> 初次画像对话式构建
  -> 查看画像变化
  -> 导入学习资料
  -> 在仪表盘查看资料转换成的知识节点
  -> 解释卢曼卡片盒与三色卡片
  -> 在仪表盘图谱上点击灵感卡，在工作台打开
  -> Agent A 苏格拉底式提问
  -> 学生用自己的话完成费曼输出
  -> Agent B 后台保留原话、更新画像与灵感卡
  -> 从永久卡讲到审核能力
  -> 第一次审核未通过
  -> 继续对话和打磨
  -> 再次审核通过，升级为永久卡
  -> 展示资源推送的两种方式
  -> 展示六类资源的真实用途与预览
  -> 展示知识图谱的几种模式
  -> 价值收束
```

V18 只把必须连续发生的动作合成连续实录，不改变其中任何动作的先后关系。

## 三、统一口径与真实性合同

### 1. 产品定位

> **AXIOM Space 是给高校学生使用的 AI 掌握学习系统。**

“掌握学习”必须立刻翻译成白话：学生要用自己的话讲清、接受审核，并让掌握结果改变后续学习。

### 2. 三类卡片

> 文献卡：资料从哪里来。  
> 灵感卡：尚未掌握。  
> 永久卡：已经掌握。

本片可以说“借鉴卢曼卡片盒的原子化、连接与长期沉淀思想”，不能说三种颜色与掌握含义来自卢曼原始体系。

### 3. 本片所称“已经掌握”

在 AXIOM 的学习主流程中，一张灵感卡必须同时满足以下条件，才能升级为永久卡：

1. 内容来自学生自己的原始输出；
2. 学生能用自己的话讲清机制；
3. 学生能处理例子、反例或适用边界；
4. 学生能在一个新情境中应用；
5. 审核依据能够回到原话、来源和本次作答。

因此，本片的安全表达是：

> **这张灵感卡在学生完成输出、处理边界与新情境，并通过审核后升级为永久卡；此时它代表这项知识已经掌握。**

这不表示学生从此永远不会出错，也不表示 AI 的判断是不可纠正的绝对结论。

### 4. 苏格拉底式提问与费曼输出

> Agent A 通过预测、为什么、例子、反例和边界追问；学生用自己的话把机制讲给别人听。

不能只显示术语或让学生复述标准答案。

### 5. 双 Agent

> 学生始终面对一个连续的学习工作台；Agent A 在前台提问，Agent B 在后台保留证据、更新画像和写回灵感卡。

Agent B 可以整理学生表达，但不能替学生生成“我的理解”。

### 6. 旧知迁移

过去已经掌握的永久卡可以被语义检索找回，用来给当前问题搭桥；内容相似不能直接让当前问题通过审核。

### 7. 系统主动推送

系统只提出建议，并显示来源、理由与预期结果；学生可以接受或忽略，确认后才执行。不能表达成系统擅自生成并写入知识库。

### 上屏文字硬度规则

1. 主标题必须是判断，不使用“功能介绍”“能力展示”式标题；
2. 主标题最多两行，评委 3 秒内必须能够读完；
3. 每页只保留一个主判断，辅助句只写画面中可以看见的证据；
4. 卢曼、苏格拉底、费曼、向量检索与图谱模式只做小标签，不与中文主结论争夺视觉主位；
5. 连续过程使用逐步出现的字幕，同一时刻不把所有步骤堆在屏幕上；
6. 第 01 与第 15 页必须把“给高校学生使用的 AI 学习软件”作为清晰产品主语，不得缩成难以看见的角标。
7. 最终定格最多保留“产品定位＋主标题＋愿景”三层，其余结论必须在此前逐条出现并退出。

### PPT 文字、视频与配音的融合规则

本稿所称“一张 PPT”，指一个完整的叙事单元，不等于一张纯静态页面。除最后定格外，默认使用真实产品实录作为主画面，PPT 文字作为覆盖在关键动作上的解释层。

```text
真实产品动作（证明产品做了什么）
  + 同步上屏文字（告诉评委现在看什么、这意味着什么）
  + 配音（解释为什么这样设计、它解决了什么问题）
```

每个单元必须遵守：

1. 主标题在本单元前 2—3 秒出现，随后缩小、移到安全区或退出，不能长期遮挡产品界面；
2. 操作进行时，只保留“当前动作”与“可见结果”两类短字幕；
3. 解释必须贴着它所解释的真实控件、状态变化或视频动作出现；
4. 静态流程图不能代替真实产品变化；需要解释时，优先使用真实界面停帧、局部放大、箭头、高亮框和阶段字幕；
5. 同一条连续操作中不插入另一张解释型 PPT，不黑场，不重新建立上下文；
6. 只有第 15 单元最后 8—10 秒允许成为纯静态定格；第 08 单元只能停在真实审核界面上叠加规则，不能另做原理页；
7. 如果没有对应实录，必须明确写“待补录”；如果只有概念动画，不能在验收时算作产品证明。

## 六、三个媒介必须分别通过的理解测试

### 只听口播

听众应当能够回答：

1. AXIOM 是给谁使用的？
2. 传统工具分别解决了什么，还留下什么问题？
3. 为什么系统要先建立可修正画像？
4. Agent A、学生和 Agent B 分别做什么？
5. 灵感卡怎样升级为永久卡？
6. 两种资源方式有什么区别？
7. 为什么一次学习能够改变下一次学习？

### 只看 PPT 文字

15 个主标题必须能独立组成下面这条逻辑：

```text
AI 给出答案，学生仍可能没有学会
  -> 画像从真实对话中形成，每次回答都会改变下一问
  -> 每条画像判断都能回到原话，也能被学生纠正
  -> PDF 变成可以继续学习的知识节点
  -> 看过、正在理解与已经掌握必须分开
  -> 点开哪里不会，就从哪里继续学
  -> AI 不替学生回答，学生必须自己讲明白
  -> AI 写得再漂亮，也不能直接算学生掌握
  -> 没讲清就继续学，通过审核才成为永久卡
  -> 学生可以主动要资源，系统建议也必须说明理由并由学生确认
  -> 六类资源分别服务六种学习动作
  -> 图谱追溯来源，也显示掌握结果和下一步
  -> 不看 AI 生成了多少，只看学生是否真的学会
```

### 只看静音视频

每个视频必须常驻或按阶段显示三种字幕：

1. 当前正在做什么；
2. 界面发生了什么变化；
3. 这次变化对学生意味着什么。

仅仅展示鼠标点击、加载动画、六个资源名称或图谱旋转，都不能算证明。

| 能直接证明 V18 全部主张的当前成片 | 0 / 1 | **不能宣称已经完成** |

进入 HTML 制作前必须先解决以下阻断项：

1. **第 02 单元缺实录**：必须补录“第一问→学生回答→下一问改变→画像已更新”的完整画像对话；
2. **第 06 单元起点待核**：现有 `scene-04-course-to-workbench.mp4` 如果从课程路径而不是 Dashboard 青色灵感卡进入，就不能使用，必须补录；
3. **第 07—09 单元缺连续证据**：现有素材分散，必须证明是同一仓库、同一节点、同一张卡和同一次学习；
4. **第 08 单元权限边界待证**：真实 UI 必须看见 Agent B 只能提交审核、不能直接晋级；
5. **第 10 单元上下文待核**：现有生成视频只有在当前对象确实是“锁方案选择与失效边界”时才可使用；
6. **第 11 单元缺实录**：必须补录有来源、理由、证据强度、预期结果以及接受 / 忽略入口的系统建议；
7. **第 13 单元来源待证**：来源模式必须真正打开原始资料或对话；
8. **第 14 单元必须重回审核后的图谱**：不能使用发生在学习和审核之前的旧 Galaxy 状态来证明“刚掌握的节点被跳过”；
9. **永久卡口径仍是发布门禁**：实现、需求规格、产品设计和测试文档没有完成同步前，不进入最终配音。

因此，V18 现在已经可以用来指导补录和剪辑，但不能用来宣称当前 HTML 与现有视频已经完成了全部证明。
