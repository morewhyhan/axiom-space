# AXIOM Space A3 黄金案例 V15｜真实用户使用顺序总导演稿

> 版本：V15  
> 用途：最终提交给评委的产品展示视频、HTML 演示页、配音与录屏统一依据  
> 基础版本：V14 总导演稿与当前 `local-tests/a3-golden-video-card` HTML  
> 本版核心变化：不推翻当前页面的视觉与有效文案，只把叙事主轴改成真实用户从首次进入 Vault 到形成长期学习证据的使用顺序。

---

## 一、V15 的唯一叙事决定

全片不再按照“画像、图谱、卡片、资源、评估”的功能目录讲，也不完全照搬 V14 的机制论证顺序。

全片只跟随一个人、一个 Vault、一份课程资料和一个知识节点：

> 小林进入自己的课程知识库，先通过对话建立可修正的初始画像；随后导入课程资料，系统把资料变成卡片、关系和学习路径。小林选择“并发库存超卖”节点学习，Agent A 在前台通过提问暴露误解，Agent B 在后台记录证据、更新画像并写回卡片。小林用自己的话解释，知识经过审核，再用陌生题验证是否真正掌握。最后，证据改变路径、推荐与下一次学习。

评委看完必须能够复述下面这条用户链路：

```text
进入 Vault
  -> 初始画像对话
  -> 认知洞察显示可修正画像
  -> 导入课程资料
  -> 资料变成卡片、关系、星团和路径
  -> Dashboard 解释知识库当前状态
  -> 三类卡片与永久知识审核
  -> Galaxy 用不同视角查看同一知识网络
  -> 选择一个节点进入学习
  -> Agent A 提问，Agent B 后台记录
  -> 画像改变下一问，费曼解释写回卡片
  -> 图谱判断缺什么，画像判断怎么补
  -> 生成和推送恰当资源
  -> 陌生迁移验证掌握
  -> 证据改变路径，并支持下次继续
```

## 二、为什么不能走两个极端

### 不能从注册页开始平铺

注册、登录、选择 Vault 只能证明产品有入口，不能证明产品有价值。正片最多用 2—3 秒带过登录状态，不能把评委的前 30 秒消耗在表单上。

### 不能继续完全沿用当前 HTML 的机制顺序

当前 HTML 的第一幕很强，但随后较快进入画像、旧卡、图谱和资源。对于第一次看 AXIOM 的评委，这些对象尚未被建立，容易产生三个问题：

1. 不知道画像最初从哪里来；
2. 不知道课程资料怎样变成知识节点；
3. 不知道 Dashboard、卡片盒、Galaxy 和工作台之间是什么关系。

### V15 的中和方式

- 保留当前第一幕的问题冲突，先回答“为什么需要 AXIOM”。
- 从第二阶段开始严格贴合首次使用顺序，回答“用户实际上怎么用”。
- 当前已经好看的文字页继续使用，只换位置或改一句转场。
- 新增页面只用于补齐真实流程中当前完全缺失的环节。
- 重复的行业痛点、创新总结和最终复述压缩合并，不让总页数失控。

## 三、全片结构：30 页，六个阶段

| 阶段 | 页码 | 用户此刻在做什么 | 评委此刻应当明白什么 |
|---|---:|---|---|
| 0. 问题钩子 | 01—02 | 看到一个“答案正确但仍未学会”的案例 | AXIOM 不是另一个答案生成器 |
| 1. 进入与认识用户 | 03—05 | 进入 Vault、完成初始画像、查看认知洞察 | 个性化从对话证据开始，不从标签开始 |
| 2. 建立课程知识库 | 06—11 | 导入资料、看 Dashboard、理解卡片盒和审核 | 资料会变成可学习、可追溯、可审核的知识对象 |
| 3. 看见并选择知识 | 12—14 | 切换 Galaxy 视角、选择节点进入工作台 | 图谱不是装饰，而是学习入口和决策依据 |
| 4. 完成一次学习闭环 | 15—24 | 提问、更新画像、写卡、生成资源、迁移评估 | 双 Agent、费曼学习和证据门共同把“看过”变成“掌握” |
| 5. 持续、验证与收束 | 25—30 | 路径改变、恢复学习、追溯依据、查看工程验证 | 一次学习会成为下一次学习的起点 |

建议配音净时长控制在 6 分钟左右。视频可以略长于口播，后期通过变速、跳切和局部停帧压缩。

## 四、逐页演讲稿与视频导演说明

## 第一阶段：先让评委知道为什么需要 AXIOM

### 01｜通用 Agent 已经给出解法，小林却仍可能没有学会

**页面类型**：文字页，沿用当前 `01-content` 视觉与核心文案。  
**本页主旨**：答案出现，不等于学习发生。

**固定上屏文字**

> 通用 Agent 已经给出解法，  
> **小林却仍可能没有学会**

> 如果他仍认为第二个请求一定会读到 0，那么答案虽然正确，脑中的并发模型仍然是错的。

**完整口播**

小林学过并发，也看过标准答案，却仍可能判断错真实执行顺序。普通 AI 往往在答案出现时结束，AXIOM 从这里才开始：它要找到误解、让学生重新表达，再用新的表现确认他是否真正掌握。

**转场**

> 先看小林究竟错在哪里。

---

### 02｜同一道题暴露了正确答案下面的错误理解

**页面类型**：视频页，沿用当前 `scene-01-video`；把当前 `scene-14-video` 的“待验证”含义并入口播，不再单独占一页。  
**本页主旨**：系统先让学生预测，再让现实结果制造认知冲突。

**固定上屏文字**

> 小林以为 B 会读到 0，  
> **结果 A、B 都先读到 1**

**完整口播**

标准答案已经出现，但能力仍保持待验证。AXIOM 先让小林预测；他认为 B 会读到零，真实结果却是两边都先读到一。这个冲突暴露了具体误解：他把读取、判断和写回想成了连续动作。

**转场**

> 现在从一个新用户第一次进入 AXIOM 开始，看这套系统怎样建立长期学习闭环。

## 第二阶段：进入 Vault，先建立可修正的初始画像

### 03｜进入自己的课程 Vault，系统先认识学习者

**页面类型**：新增视频页。  
**本页主旨**：Vault 是用户自己的长期课程空间；首次进入先建立画像，而不是先堆功能。

**画面动作**

1. 已登录状态下选择“小林｜Java Web 课程仓库”；
2. 进入工作区；
3. 首次画像引导面板自动出现；
4. 点击“开始初始画像”。

**固定上屏文字**

> 第一次进入课程仓库，  
> **先建立一份可以被以后表现修正的初始画像**

**完整口播**

登录后，小林进入自己的课程 Vault。这里保存课程资料、卡片、对话、图谱、画像和学习路径。首次进入时，系统不会假装已经了解他，而是邀请他先完成一段低成本的初始画像对话。

**拍摄限制**

- 不完整展示注册流程；
- 不在多个 Vault 之间来回切换；
- 画面必须明确“这是同一个小林账号和同一个课程仓库”。

---

### 04｜初始画像来自对话，不来自一张静态问卷

**页面类型**：新增视频页。  
**本页主旨**：初始画像记录目标、基础、偏好和卡点，但只是一组待验证假设。

**画面动作**

1. Agent 每轮只问一个问题；
2. 小林回答学习目标、当前基础、偏好的解释方式和主要卡点；
3. 完成六个必要维度；
4. 出现“初始画像已经完成”和“查看认知洞察”入口。

**固定上屏文字**

> AI 每轮只问一个必要问题，  
> **把自述变成待验证的初始假设**

**完整口播**

Agent 每轮只问一个必要问题，了解小林的目标、当前状态、信息编码方式、主要阻塞、执行负荷和反馈习惯。这里记录的不是性格测量，也不是能力结论，而是一组后续必须接受真实学习表现检验的初始假设。

**转场**

> 对话完成以后，小林可以直接检查系统究竟怎样理解自己。

---

### 05｜认知洞察让画像变化可见、可纠正

**页面类型**：新增视频页。  
**本页主旨**：用户拥有对画像的知情权与纠正权。

**画面动作**

1. 点击“查看认知洞察”；
2. 展示当前画像和初始画像时间点；
3. 展示一条画像观察的来源、置信度和状态；
4. 展示“后续教学会怎样调整”；
5. 不停留在装饰性雷达图上。

**固定上屏文字**

> 画像不是后台黑箱，  
> **每条判断都能查看、修正并等待后续验证**

**完整口播**

在认知洞察中，小林能看到当前画像、初始版本、观察依据和下一步建议。证据不足的判断只能保持低置信；用户可以纠正，后续真实表现也会继续改写它。个性化因此不是一个标签，而是一份双方都能检查的约定。

**转场**

> 系统初步认识了小林，接下来建立这门课程的知识环境。

---

## 第三阶段：导入资料，建立课程知识库

### 06｜课程讲义导入以后，不只是被保存

**页面类型**：视频页，沿用当前 `scene-19-course-import.mp4`。  
**本页主旨**：导入的资料成为后续学习对象的共同来源。

**固定上屏文字**

> 导入一份课程讲义，  
> **系统开始建立课程结构，而不是只做文件收藏**

**完整口播**

小林导入一份 Java Web 课程讲义，并标明所属主题。AXIOM 保存原始来源和内容指纹，再把资料交给解析流程。它不是把文件扔进收藏夹，而是准备生成文献卡、理解草稿、关系边和可以执行的学习路径。

---

### 07｜解析完成后，资料变成卡片、关系、星团和路径

**页面类型**：视频页，优先复用当前 `scene-04-course-to-workbench.mp4` 的前半段，并补录导入完成提示。  
**本页主旨**：文件变成可学习对象，且仍能回到来源。

**画面动作**

1. 导入任务完成；
2. 显示新增文献卡、灵感草稿、概念节点、关系边和路径步骤；
3. 打开其中一个节点，能回到原始资料；
4. 返回 Dashboard。

**固定上屏文字**

> 一份资料被拆成可学习的对象，  
> **每个节点仍然保留来源**

**完整口播**

解析完成后，一份资料变成了文献卡、待打磨的理解卡、概念节点、关系边和学习步骤。每个对象都属于同一个 Vault，也都能回到课程来源。接下来回到仪表盘，看这座知识库现在拥有什么。

---

### 08｜Dashboard 不是积分面板，而是课程知识库的状态总览

**页面类型**：新增视频页。  
**本页主旨**：第一次向评委解释 Dashboard 上各区域和数字的含义。

**画面动作**

1. 左侧依次高亮 NODES、EDGES、PERMANENT、FLEETING、LITERATURE；
2. 中央展示当前课程星团；
3. 右侧展示最近导入、画像更新和学习活动；
4. 点击“继续学习”或课程路径入口，但暂不进入。

**固定上屏文字**

> 节点、连接、三类卡片和最近变化，  
> **共同描述这个课程仓库的真实状态**

**完整口播**

Dashboard 展示的不是游戏积分。NODES 是知识对象，EDGES 是它们之间的关系；紫色永久卡保存已经通过审核的长期知识，青色灵感卡保存形成中的理解，粉色文献卡保存外部来源。中央星团展示课程结构，右侧记录导入、画像更新和学习进展。

**转场**

> 这三个颜色来自卢曼卡片盒，但 AXIOM 为学习场景增加了两道必须分开的门。

---

### 09｜三类卡片保存三种不同的知识状态

**页面类型**：文字页，沿用当前 `04-content` 的视觉和三色设计。  
**本页主旨**：来源、形成中的理解和长期知识不能混在一起。

**固定上屏文字**

> 粉色保存来源，青色保存形成中的理解，  
> **紫色只保存通过审核的长期知识**

**完整口播**

卢曼卡片盒把知识拆成可以持续连接和生长的卡片。AXIOM 用粉色文献卡保存来源，用青色灵感卡保存学生正在形成的理解，用紫色永久卡保存已经通过质量审核、值得长期复用的知识。颜色进入 Galaxy 后仍然保持一致。

---

### 10｜只有学生自己的表达进入“我的理解”

**页面类型**：视频页，沿用当前 `scene-06-dialogue-to-card.mp4`。  
**本页主旨**：AI 可以整理，但不能替学生制造理解证据。

**固定上屏文字**

> 小林自己的解释，  
> **从对话进入青色灵感卡**

**完整口播**

当小林用自己的话解释“两个请求依据同一份旧库存分别写回”时，Agent B 把这段表达写回当前灵感卡，并保留它来自哪次对话。AI 可以帮助结构化，但不能把自己的成熟答案冒充成学生的理解。

---

### 11｜永久卡不是换一种颜色，而是一次真实审核

**页面类型**：视频页，沿用当前 `scene-04-card-review.mp4`。  
**本页主旨**：知识质量审核会真实阻止不合格内容进入长期知识。

**固定上屏文字**

> 适用边界和证据不足，  
> **这张灵感卡不能成为永久知识**

**完整口播**

第一次提交时，这张卡缺少适用边界和来源依据，系统拒绝升级。小林补足条件并关联课程资料后，第二次才通过，卡片由青色变成紫色。请注意：永久卡只证明这条知识值得保存，还不能证明小林已经掌握。

**转场**

> 当卡片、来源和关系越来越多，用户需要从不同角度看同一座知识库。

---

## 第四阶段：从 Galaxy 选择一个知识节点进入学习

### 12｜Galaxy 用不同视角回答不同问题

**页面类型**：视频页，沿用当前 `scene-21-galaxy-views.mp4`。  
**本页主旨**：布局切换不是视觉皮肤，而是不同知识问题的观察工具。

**固定上屏文字**

> 星团、关系、类型与来源视角切换，  
> **背后仍是同一批知识对象**

**完整口播**

在 Galaxy 中，用户可以按课程星团看整体结构，按关系看知识连接，按卡片类型区分来源与理解，也可以聚焦节点的邻域和证据。视角改变，节点身份、卡片颜色和知识来路都不会丢。

---

### 13｜从图谱或路径选择一个节点，进入同一个学习对象

**页面类型**：视频页，沿用当前 `scene-04-course-to-workbench.mp4` 的后半段。  
**本页主旨**：Dashboard、路径、Galaxy 与 Forge 指向同一个知识节点。

**固定上屏文字**

> 从课程路径进入工作台，  
> **当前知识节点没有断开**

**完整口播**

小林从“并发库存超卖”节点进入课程路径，再打开当前学习步骤。Forge 中的对话、卡片、资料和任务始终绑定同一个学习对象；用户不是在几个互不相干的页面之间跳转。

---

### 14｜双 Agent 不扮演两个角色，而是分开教学与记录

**页面类型**：文字页，复用当前 `10-content` 的 Harness 视觉，但将位置后移。  
**本页主旨**：Agent A 面向用户教学，Agent B 在后台维护证据与长期状态。

**固定上屏文字**

> Agent A 在前台提问和教学，  
> **Agent B 在后台记录、更新画像并反馈路径**

**完整口播**

AXIOM 不是一个模型包办所有判断。Agent A 面向小林，负责提问、解释和工具协调；Agent B 不抢占对话，而是在每轮之后提取证据、更新画像、写回卡片并反馈路径。两条通道共同修改同一个持久学习对象，但各自权限不同。

**转场**

> 现在看这套双通道怎样处理小林刚才那句错误预测。

---

## 第五阶段：完成一次真实学习闭环

### 15｜维特根斯坦式提问：不急着解释，先把模糊理解变成可检验判断

**页面类型**：文字页，改写当前 `02-content`。  
**本页主旨**：用预测、边界、反例和必要条件让学生的理解显形。

**固定上屏文字**

> 不先重讲定义，  
> **先让小林做一个可以被现实推翻的判断**

**完整口播**

这里所谓维特根斯坦式提问，不是一个装饰性按钮，而是一条教学原则：把“我好像懂了”改写成可以公开检查的判断。Agent A 追问执行顺序、适用边界和反例，一次只改变一个变量，让小林的语言和实际机制正面相遇。

---

### 16｜一句错误原话形成画像假设，下一问真的改变

**页面类型**：视频页，沿用当前 `scene-02-profile-changes-teaching.mp4`。  
**本页主旨**：学习画像必须改变教学行为，否则只是标签。

**固定上屏文字**

> “B 会读到 0”形成可纠正的假设，  
> **下一问改成亲手排列时间线**

**完整口播**

系统没有给小林贴上“并发基础薄弱”的标签，而是提出一个可以被推翻的判断：他可能把读取、判断和写回想成顺序执行。返回学习现场后，下一问改成亲手排列时间线。原话、画像假设和新的教学动作连续出现。

---

### 17｜初始画像不是终点，真实学习继续改写认知洞察

**页面类型**：视频页，可复用第 05 页认知洞察录屏并补录更新事件。  
**本页主旨**：首次画像与动态画像是同一条历史链的两个时点。

**画面动作**

1. 从当前对话打开画像证据；
2. 显示“学习画像已更新”；
3. 对比初始画像与当前画像；
4. 展示新增观察来自“小林认为 B 会读到 0”；
5. 用户可纠正该观察。

**固定上屏文字**

> 初始画像只是起点，  
> **真实回答不断修正系统对小林的理解**

**完整口播**

初始画像只来自小林的自述，现在第一次真实学习为它增加了新证据。认知洞察保留初始版本和当前版本，也标明这次变化来自哪条回答。用户可以纠正，下一次表现还会继续检验它。

### 18｜旧知识只负责搭桥，不能冒充当前掌握

**页面类型**：文字页，沿用当前 `03-content`。  
**本页主旨**：语义相关、知识复用和能力掌握必须分开。

**固定上屏文字**

> 系统找回小林自己的旧卡，  
> **用旧问题解释眼前的新问题**

**完整口播**

系统找回小林过去的在线表格卡片。已有掌握证据的内容可以跳过，相关旧知只拿来搭桥，没有证据的当前缺口继续学习。语义记忆负责“想起什么”，正式评估才有资格回答“是否会了”。

### 19｜旧卡、当前节点和来源保持可追溯

**页面类型**：视频页，沿用当前 `scene-03-semantic-reuse.mp4`。  
**本页主旨**：被调用的是真实旧卡，不是 AI 临时编造的类比。

**固定上屏文字**

> 在线表格只负责搭桥，  
> **库存超卖仍要用新的表现验证**

**完整口播**

系统打开的是真实旧卡，正文、来源和时间都能核对。它帮助小林理解共享旧状态，却不会因为内容相似，就把库存超卖提前标记为掌握。

### 20｜费曼学习不是复述答案，而是让学生自己讲清机制

**页面类型**：视频页，复用当前 `scene-06-dialogue-to-card.mp4` 的费曼解释片段；如现有片段不足，补录一次完整解释。  
**本页主旨**：学生输出是学习证据，Agent B 只提炼并保留来源。

**固定上屏文字**

> 小林用自己的话讲清因果过程，  
> **Agent B 才把解释写回当前卡片**

**完整口播**

费曼学习法要求小林用自己的话讲清读取、判断和写回，而不是复述术语。Agent A 继续追问例子、边界和反例；当解释达到可检验标准后，Agent B 才把它提炼进“我的理解”，并保留原话证据。

**转场**

> 现在系统已经知道他缺什么，也知道怎样的解释方式对他更有效。

### 21｜图谱告诉系统缺什么，画像告诉系统怎么补

**页面类型**：文字页，沿用当前 `05-content`。  
**本页主旨**：推荐必须同时有知识缺口和学习方式依据。

**固定上屏文字**

> **两份证据共同支持一个资源建议**

**完整口播**

图谱显示小林已经理解共享旧状态，却缺少锁方案的适用边界；画像显示他更适合分步、可视化的过程解释。系统因此建议一段可拖动的并发动画。建议必须说明依据，证据有多强，系统最多就能有多确定。

### 23｜不同缺口对应不同资源，不用附件数量冒充能力

**固定上屏文字**

**完整口播**

### 24｜卡片合格不等于能力掌握，必须换一道题

**页面类型**：文字页，沿用当前 `06-content`。

**完整口播**

看过解释、写出卡片、看完动画，都不能自动等于掌握。AXIOM 把库存问题换成陌生的优惠券场景；只有小林仍能独立重建读取、判断和写回过程，能力状态才有资格改变。

**页面类型**：视频页，沿用当前 `scene-09-transfer-and-pass.mp4`。

### 26｜评估不是一个分数，而是下一步真的改变

**页面类型**：视频页，沿用当前 `scene-06-assessment-changes-path.mp4`。  
**本页主旨**：通过或未通过必须改变后续学习安排。

**固定上屏文字**

**完整口播**

迁移通过后，已有掌握证据的并发基础被自动跳过；仍然薄弱的锁方案边界进入下一步。评估不是结束页上的一个分数，而是会改变路径、推荐和下一次提问的业务事件。

## 七、双 Agent、费曼与维特根斯坦的真实表达边界

### 双 Agent

可以说：

> Agent A 前台教学与协调工具，Agent B 后台提取证据、更新画像、写回卡片并反馈路径。

不能说：

> 两个 Agent 会像两个老师一样自由讨论后自动替用户完成学习。

### 费曼学习法

可以说：

> 系统要求学生用自己的话讲清因果过程、例子和边界；通过的解释可以作为卡片与评估证据。

不能说：

> 只要学生写了一段话，就已经掌握。

### 维特根斯坦式提问

本片将它表达为教学原则，而不是伪装成一个不存在的产品按钮：

> 把“我懂了”变成公开、具体、可以被反例推翻的判断；追问词语的适用边界、必要条件和实际后果。

  -> 下一问改变

  -> 写卡与审核

## 九、最终复述测试

让一个完全不了解项目的人只看一次。如果他能回答以下问题，V15 才算合格：

1. AXIOM 是给谁使用的？  
   面向高校学生的 AI 掌握学习系统。

2. 用户第一次进入自己的课程仓库后先做什么？  
   通过 AI 对话建立一份可修正的初始画像，并在认知洞察中查看。

3. 导入资料后发生了什么？  
   资料成为文献卡、理解草稿、知识节点、关系边、星团和学习路径。

4. Dashboard 左边的数字是什么？  
   知识节点、连接数量，以及永久、灵感、文献三类卡片的状态。

5. 永久卡为什么不能直接生成？  
   它必须通过知识质量审核；AI 不能把未经验证的内容包装成长期知识。

6. Galaxy 的不同模式有什么用？  
   从课程星团、关系、类型、来源和节点邻域等不同问题观察同一知识网络。

7. 双 Agent 分别做什么？  
   Agent A 前台教学，Agent B 后台记录证据、更新画像、写卡并反馈路径。

8. 费曼与维特根斯坦式提问怎样进入产品？  
   系统先把模糊理解变成可检验判断，再要求学生用自己的话讲清机制、例子与边界。

9. 为什么看完资源仍不算掌握？  
   必须在陌生问题中仍能独立应用，能力状态才会改变。

10. AXIOM 的核心价值是什么？  
    找到没学会的地方，保存学生自己的理解，用新的表现验证掌握，再让证据改变下一步。

如果评委只能记住“画像、知识图谱、六类资源”，却不能沿着小林的真实操作讲出这条因果链，说明视频仍然是功能目录，而不是一段合格的产品演示。
