## 1. 黄金案例的工作目标

本片不是功能导览，也不是技术答辩。

它要让评委仅凭视频完成四个判断：

1. 学生确实存在一个普通 AI 没有完整解决的学习问题。
2. AXIOM 的主要功能围绕同一个学生、同一个知识点连续工作。
3. 画像、Harness、向量数据库、图谱、资源与审核不是装饰，它们分别改变了学习结果。
4. 页面中的真实记录足以证明这些能力已经实现，而不是产品设想。

统一证据结构：

```text
具体痛点
  → 用户操作
  → 页面变化
  → 技术为什么存在
  → 使用前后差异
  → 对应评分点
```

## 2. 建议中心案例

### 小林能照着写 Visitor，却说不清为什么少一步就会选错方法

小林已经能照着 UML 写出 Visitor，也知道 `accept` 和 `visit` 两个名称。但当代码中变量写成 `Document element = new PdfNode()` 时，他误以为直接执行 `visitor.visit(element)` 会自动选择 PDF 对应的方法。

真正缺口不是“没背过 Visitor”，而是他把 Java 的编译期重载和运行时重写混在了一起。

这段案例对非技术评委只解释到以下程度：

> 小林会抄标准代码，却不知道为什么中间那一步不能删。删掉以后，程序看到的只是“一个普通文档”，无法选择 PDF 的专用处理。AXIOM 没有重新讲一遍定义，而是找回他以前学过的 Java 类型知识，让他亲自预测、运行、解释和迁移。

正片不承担完整 Java 教学。代码输出、卡片原话、失败审核和陌生 AST 迁移只作为“这个学生真的完成过判断”的页面证据。

选择这个案例的原因：

- 当前黄金数据已经围绕 Visitor 建立了完整的短期与长期记录。
- 它能自然展示“知道名词不等于掌握机制”。
- 它能同时承载画像干预、语义召回、对话辅导、卡片写入、独立审核、资源生成、图谱关系、路径调整和长期复测。
- 它存在明确失败结果，不需要把演示账号包装成全会。

## 4. 35% 创新与实用性：必须用案例证明的四个判断

### 4.1 创新一：系统重新定义了“完成”

普通 AI 的完成条件：回复已经生成。

AXIOM 的完成条件：用户已经输出解释，能够处理反例和陌生迁移，并通过独立审核。

最短证据画面：

```text
小林说“我懂了”
  → 第一次永久卡审核失败
  → 页面列出缺少反例与迁移
  → 小林完成陌生 AST
  → 第二次通过
```

### 4.2 创新二：过去学过的知识会参与当前判断

普通 AI 的记忆通常停留在聊天上下文或资料检索。

AXIOM 把永久卡、掌握证据和语义索引结合起来：

- 近义重复用于去重，避免重新生成已经掌握的概念。
- 语义相关但不等价的知识用于类比和迁移。
- 已经掌握的前置内容可以从路径中跳过。

最短证据画面：

```text
Visitor 当前问题
  → 召回“重载看静态类型”和“重写看运行时类型”
  → 显示真实旧卡引用
  → Agent 用旧知识追问当前调用轨迹
  → 路径不再重复 Java 基础定义
```

### 4.3 创新三：Harness 把发散生成牵引到学习结果

最短证据不是架构图，而是一条对象变化链：

```text
Learn 当前步骤
  → Forge Current Focus
  → Agent A 对话
  → Agent B 写入卡片与观察
  → Assess 审核
  → Learn / Cognition / Galaxy 同步变化
```

架构图只在这条链已经出现以后补充 3—5 秒，用于告诉评委：刚才的跨页面变化来自统一 Harness，而不是手工拼接的几个页面。

### 4.4 创新四：知识图谱是认知投影，不是 3D 装饰

最短证据画面：同一 Visitor 节点连续切换三种视图。

- 邻域：它与哪些旧知识有关。
- 任务流：下一步应该做什么。
- 证据：哪些资料、对话和评估支撑这张永久卡。

评委只需看到“同一份数据回答三个不同问题”，不需要观看所有布局动画。

## 5. 商业与应用价值的最短证明

### 学生侧

切肤之痛不是“没有学习资料”，而是：

> 我看过很多解释，却不知道自己的缺口在哪里；我以为懂了，也没人要求我证明；下一次再遇到相似问题，我仍然从零开始。

案例结果：小林从错误预测到陌生迁移，再到隔日复测；系统保存的不是一份替他写好的答案，而是他为什么能够被判定为掌握的证据。

### 教师侧

普通平台通常只给教师分数和完成率。

AXIOM 能展示：学生原始说法、误解发生在哪一步、采用了什么干预、第一次为什么失败、后来凭什么通过。教师看到的是过程证据，而不只是一行 86 分。

### 高校与课程侧

同一门课程的资料、资源、路径、量规、常见误解和有效干预可以持续沉淀。这里可以展示长期档案，但不能把 312 张卡片直接等同于 312 个已掌握概念。

## 7. 当前真实缺口与录制门禁

### 7.1 黄金数据自检已经通过


当前黄金档案已经通过普通会话、卡片线程、画像来源、正式评估、资源、推送和快速语义索引等契约检查。录制前仍必须再次执行同一检查，避免数据库被后续调试数据污染。

### 7.2 系统级学习健康度不能作为主画面

设计文档明确记录：系统层评估已有后端数据，但缺少集中展示界面。因此正片优先展示已经可见的费曼评估、卡片审核、正式测验和路径反馈，不宣称已有完整“学习健康度”总面板。

### 7.3 防幻觉必须保持准确表述

可以说：

> 系统通过来源引用、结构校验、独立审核、工具确认和审计记录降低错误进入长期知识的风险。

不能说：

> 系统已经彻底消除大模型幻觉。

### 7.4 PPT、DOCX、PDF 的类型口径

PPTX、DOCX、PDF 是讲解材料节点的文件附件，不应为了凑数量被说成独立知识资源类型。赛题“至少五类资源”的主证据使用文档、导图、题目、代码、图示和视频六类；文件附件用于证明真实交付能力。
