# AXIOM Space — 端到端测试计划

## 产品定位

基于**卢曼卡片盒笔记法**和**费曼学习法**的 AI 个性化知识构建系统。以"概念卡片"为基本学习单位，通过三盒体系（闪念盒→文献盒→永久盒）完成知识从输入、内化到系统化输出的全流程。

## 核心学习闭环

```
┌─────────────────────────────────────────────────────────┐
│                    用户学习闭环                           │
│                                                         │
│  ① 选择领域 → 创建巨匠Agent → 对话初识                    │
│     ↓ (AI 探测知识边界，构建 6 维画像)                    │
│  ② 文献盒 — AI 生成个性化学习资料(5种)                    │
│     ↓ (用户研读资料)                                     │
│  ③ 闪念盒 — 用户记录零散想法、感悟、疑问                   │
│     ↓ (AI 引导打磨)                                      │
│  ④ 永久盒 — 标准化概念卡片(定义+举例+关联+应用)           │
│     ↓ (双向链接 → 知识图谱)                              │
│  ⑤ AI 迭代 — 更新画像，优化推荐，加速学习                 │
│     ↓                                                     │
│  ← ← ← ← ← ← 循环，越学越快 → → → → → →                │
└─────────────────────────────────────────────────────────┘
```

## 四个 Agent 角色

| Agent | 角色 | 职责 |
|-------|------|------|
| **Oracle (巨匠)** | 学科奠基人 | 苏格拉底式对话，探测知识边界 |
| **Literature (文献)** | 资料生成 | 基于画像生成 5 种学习资源 |
| **Forge (锻造)** | 卡片构建 | 引导用户打磨闪念，形成标准卡片 |
| **Guide (向导)** | 迭代优化 | 分析永久盒，更新画像，优化路径 |

## 用户界面五模式

| 模式 | 对应功能 | 用户操作 |
|------|---------|---------|
| **星系 (Galaxy)** | 知识图谱 | 浏览卡片之间的双向链接关系 |
| **锻造 (Forge)** | 对话+编辑 | 与 Oracle 对话、编辑卡片 |
| **认知 (Cognition)** | 画像可视化 | 查看 6 维学习画像 |
| **学习 (Learn)** | 学习管理 | 查看学习进度、推荐列表 |
| **仪表盘 (Dashboard)** | 数据统计 | 查看节点数、连接数、学习统计 |

# 测试用例

## 🔴 第一级：完整学习闭环

### E2E-01 从注册到第一张永久卡片

**测试目标：** 验证用户从零到完成一张标准概念卡片的完整链路。

**前置条件：** 系统已启动，数据库已初始化。

| 步骤 | 操作 | 预期结果 |
|:----:|------|----------|
| 1 | 打开 `http://localhost:3000` | 页面渲染，无白屏 |
| 2 | 点击登录 → 注册账号 | 注册成功，自动登录 |
| 3 | 进入**锻造 (Forge)** 模式 | 左侧对话面板 + 右侧编辑面板正常渲染 |
| 4 | 在对话窗口发送第一轮消息（如"我想学习人工智能"） | Oracle Agent 回复，消息流式输出 |
| 5 | 对话 3-5 轮，Oracle Agent 探测知识边界 | 每轮都有回复，回复有上下文连贯性 |
| 6 | 在对话中 Agent 调用 `create_fleeing_card` 工具 | 数据库 card 表新增一条 fleeting 类型记录 |
| 7 | 在编辑器中打开该闪念卡片，补充内容 | 编辑器加载卡片 Markdown 内容 |
| 8 | 保存后切换到**永久盒**，AI 引导打磨 | 打磨后卡片类型变为 permanent |
| 9 | 检查永久卡片内容是否完整（定义+举例+关联+应用） | 四个部分均有内容 |
| 10 | 切换到**星系 (Galaxy)** 模式 | 知识图谱渲染，能看到刚才的卡片节点 |

**验证点：**
- [ ] 注册流程完整
- [ ] Oracle 对话有回复，非空
- [ ] fleeting 卡片写入数据库
- [ ] 编辑器能加载和保存卡片
- [ ] permanent 卡片包含四要素（定义+举例+关联+应用）
- [ ] 知识图谱有对应节点

### E2E-02 文献生成与学习闭环

**测试目标：** 验证 AI 能基于对话生成学习资料，用户能查阅资料后记录闪念，最终沉淀为永久卡片。

**前置条件：** E2E-01 已完成，用户已与 Oracle 对话并完成初始画像。

| 步骤 | 操作 | 预期结果 |
|:----:|------|----------|
| 1 | 在与 Oracle 对话足够多轮后，AI 调用资源生成工具 | 系统调用资源生成 API |
| 2 | 在**学习 (Learn)** 模式查看生成的文献列表 | 列表中显示 5 种资源（讲解文档、思维导图、练习题等） |
| 3 | 打开其中一篇文献阅读 | 文献内容正常渲染（Markdown） |
| 4 | 阅读后记录一条闪念笔记 | 闪念进入 fleeting 类型卡片 |
| 5 | 对闪念进行打磨，形成永久卡片 | 卡片类型变为 permanent |
| 6 | 卡片中正确使用了 `[[wikilink]]` 标注关联 | 知识图谱出现连线 |

**验证点：**
- [ ] 资源生成功能触发
- [ ] 5 种资源类型均可查看
- [ ] 闪念笔记能正确录入
- [ ] 打磨后生成永久卡片
- [ ] 双向链接在知识图谱中可见

## 测试执行顺序

```
E2E-01 (完整闭环)     ← 核心测试，必须先通过
  ↓
E2E-02 (文献+学习)    ← 依赖 E2E-01 的用户和数据
  ↓
E2E-03 (卡片 CRUD)   ← 可独立测试
  ↓
E2E-04 (Oracle 对话)  ← 依赖 E2E-01 的 Agent
  ↓
E2E-05 (多 Agent)    ← 依赖 E2E-02 的资源生成
  ↓
E2E-06 (打磨流程)     ← 依赖 E2E-03 的卡片
  ↓
E2E-07 (用户隔离)     ← 可独立测试
  ↓
E2E-08 (导出)        ← 依赖 E2E-03 的数据
  ↓
E2E-09/10 (UI)       ← 最后验证
```

## 通过标准

| 级别 | 测试 | 要求 |
|:----:|------|:----:|
| 🔴 | E2E-01~03 | **必须全部通过** |
| 🟡 | E2E-04~08 | **建议全部通过** |

```bash
# 1. 清理数据库
rm -f prisma/dev.db && npx prisma db push

# 2. 启动服务
pnpm dev

# 3. 验证服务
curl http://localhost:3000/api/health
# → { "status": "ok" }

```

## 失败处理

- 如果 E2E-01 失败，**不继续执行**后续测试
- 如果 E2E-03 失败，E2E-08（导出）预期失败
- 每轮测试前清理数据库，保证隔离
