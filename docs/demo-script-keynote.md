# AXIOM Space · Keynote 版演示脚本

> **框架**：Why → How → What + One More Thing
> **总时长**：6 分 50 秒 · 一镜到底
> **设计哲学**：黑屏与白字，留白与重音，每一句都该被记住

---

## §0 风格说明

这一版与上一版的根本区别——

| 维度 | 上一版 | Keynote 版 |
|------|--------|-----------|
| 旁白密度 | 1460 字 | **约 1100 字**（减 25%） |
| 句子长度 | 中等到长 | **短句为主**，三句以下成段 |
| 对比方式 | 大量 Before-After 卡片 | 仅 **3 张**关键时刻使用 |
| 节奏 | 持续推进 | **三次长留白**（每次 ≥ 3 秒静默） |
| 字幕 | 信息密集 | **一行一句**，每屏不超过 12 字 |
| 视觉 | UI 直接出现 | **黑屏开场 + 产品名延迟揭幕** |
| 结尾 | 总结回扣 | **One More Thing** 收束 |

> 苹果发布会从不解释每一个功能，它**让你想拥有它**。
> 这一版的目标，是把"想拥有它"这件事，放在一切之前。

## 附录 B · 三次关键留白

苹果发布会最强的工具是**沉默**。本稿设计了三次刻意留白，每一次都必须保留：

| 留白位置 | 时长 | 作用 |
|---------|------|------|
| 0:00 屏幕只有"2σ" | **3 秒** | 让评委产生疑问："这是什么？" |

| 5:00 卡片升级动画完成后 | **3 秒** | 让"它才是你的"这句话沉底 |

## 附录 D · 拍摄关键提示

1. **开场禁止任何音乐。** 只有键盘环境音和呼吸感。音乐从 AXIOM 字揭幕处开始，深邃、克制、低频。
2. **字幕字号大、字数少。** Bloom 段每屏最多 3 行，每行不超过 16 字。
3. **苹果留白原则**：当画面足够强时，停止说话。脚本中标注"留白"的位置，**演讲者必须真正停止**，不要用任何填补音。
4. **One More Thing 段落**：单独使用一种节奏——更慢、更轻。Markdown 文件夹截图必须**真实**，不能是模拟。
5. **结尾字"让一对一导师不再是少数人的特权"必须居中独占一屏。** 不要堆叠 logo 或团队信息——那些在留白 4 秒后再出现。
6. **Hermes 三层架构图**翻面那一下是镜头戏。提前录制好动画，**不要现场触发**，避免延迟。
7. **整片唯一的「我们」语态保留**：剩下全是「它」「你」。把产品说成人，把观众放进去。
