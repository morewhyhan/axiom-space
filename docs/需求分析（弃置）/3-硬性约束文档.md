# 硬性约束文档

***

## 一、必须做到的（赛题硬性要求）

### 1. 核心功能（赛题第22-34行）

| 功能            | 要求                                     | 我们的实现                              |
| ------------- | -------------------------------------- | ---------------------------------- |
| **对话式学习画像构建** | 摒弃表单，自然语言对话，≥6个维度，动态更新                 | 巨匠Agent对话式探测，实时更新                  |
| **多智能体协同**    | 须体现"多智能体"架构，不同角色智能体协作                  | 4个Agent：巨匠、文献生成、卡片构建、迭代优化          |
| **资源生成≥5种**   | 至少5种类型：讲解文档、思维导图、练习题目、拓展阅读、视频/动画、代码案例等 | 文献盒生成：讲解文档、思维导图、练习题目、拓展阅读、代码案例（5种） |
| **个性化学习路径**   | 规划科学、动态的学习路径，明确步骤和顺序                   | 基于三盒体系自然融入，AI推荐学习顺序                |
| **资源精准推送**    | 基于画像推送文档、视频、题库、实操案例等                   | AI根据画像生成文献盒内容，精准推送                 |

### 2. 非功能性需求（赛题第38-43行）

| 需求        | 要求                                     | 我们的实现                       |
| --------- | -------------------------------------- | --------------------------- |
| **界面交互**  | 美观简洁，符合现代AI产品规范（流式输出、Markdown渲染、卡片化展示） | Web端React+TS，流式对话，卡片化展示三盒内容 |
| **防幻觉机制** | 确保生成内容无事实性错误、无敏感违规信息                   | 大模型输出验证+敏感词过滤+事实核查          |
| **响应效率**  | 核心功能响应时间合理，多模态生成需进度追踪或流式呈现             | 流式对话+资源生成进度条                |
| **开源标注**  | 使用开源项目/工具需标注名称、来源、协议                   | 文档中明确标注所有依赖                 |

### 3. 技术约束（赛题第47-49行）

| 约束         | 要求                      | 我们的实现               |
| ---------- | ----------------------- | ------------------- |
| **AI工具选型** | 使用AI辅助工具需选用**科大讯飞相关工具** | 大模型使用讯飞星火API        |
| **多智能体框架** | 需明确"多智能体协同框架"           | 自研多Agent协作框架，文档详细说明 |

### 4. 测试数据（赛题第53-55行）

| 要求                         | 我们的实现                   |
| -------------------------- | ----------------------- |
| 自行构造至少一门完整高校专业课程的初始知识库/文档集 | 准备《人工智能导论》完整课程知识库作为系统输入 |

### 5. 文档要求（赛题第59-66行）

| 文档          | 要求                              |
| ----------- | ------------------------------- |
| **系统开发说明书** | 需求分析+智能体设计开发全流程+前沿AI技术融合思路+创新实践 |
| **测试说明书**   | 测试用例、测试结果、性能评估                  |
| **图表**      | 流程图、架构图、数据支撑                    |

### 6. 提交物（赛题第81-88行）

| 提交物             | 要求                         |
| --------------- | -------------------------- |
| **演示PPT**       | 展示应用价值、技术融合思路、创新价值、核心功能    |
| **完整运行文件**      | 源码、数据集、配置文件，可正常运行          |
| **演示视频**        | ≤7分钟，展示操作流程、核心功能、多模态资源生成效果 |
| **AI Coding说明** | 如使用AI编程工具，需说明              |

### 7. 加分项（赛题第32-34行）

| 加分项        | 我们的态度                             |
| ---------- | --------------------------------- |
| **智能辅导**   | **不做**（巨匠Agent对话已包含辅导功能，不单独列为加分项） |
| **学习效果评估** | **不做**（迭代优化Agent已包含评估功能，不单独列为加分项） |

## 二、明确不做的（边界界定）

### 1. 应用形态

| 不做              | 原因              |
| --------------- | --------------- |
| ❌ 移动端APP        | 赛题允许，但我们要聚焦Web端 |
| ❌ 小程序           | 同上              |
| ❌ 桌面端（Electron） | 已改为Web端，更符合赛题倾向 |

### 2. 核心理念边界

| 不做           | 原因               |
| ------------ | ---------------- |
| ❌ 表单式画像采集    | 违背核心设计理念，坚持对话式   |
| ❌ AI代劳生成知识卡片 | 违背费曼学习法，用户必须自己构建 |
| ❌ 纯本地文件存储    | 已改为数据库存储，Web端要求  |
| ❌ 社交功能       | 不在赛题范围内，不分散精力    |
| ❌ 游戏化机制      | 不在赛题范围内，保持严肃学习体验 |

### 3. 资源生成边界

| 不做           | 原因                              |
| ------------ | ------------------------------- |
| ❌ 多模态视频/动画生成 | 技术复杂度高，投入产出比低，以讲解文档、思维导图、代码案例为主 |
| ❌ PPT自动生成    | 不在核心5种资源类型中，不做                  |

### 4. 功能边界

| 不做            | 原因             |
| ------------- | -------------- |
| ❌ 用户系统（注册/登录） | 初赛阶段不要求，聚焦核心功能 |
| ❌ 多租户/多班级管理   | 不在赛题范围内        |
| ❌ 支付/商业化      | 不在赛题范围内        |
