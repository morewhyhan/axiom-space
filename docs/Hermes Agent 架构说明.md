# Hermes Agent — 架构说明

> 本文档用普通人能听懂的语言，说明 AXIOM Space 的多智能体系统是怎么设计的。
> 适合：新加入的开发者、参赛评委、感兴趣的技术爱好者。

---

## 一、从赛题要求说起

这个项目是"第十五届中国软件杯大赛"的参赛作品。赛题要求做一个**基于大模型的个性化学习多智能体系统**，具体要支持 5 个能力：

| # | 赛题要求 | 大白话 |
|:-:|---------|--------|
| 1 | 对话式学习画像构建 | 跟你聊聊天就能知道你的水平，不用填表格 |
| 2 | 多智能体协同资源生成 | 多个 AI 分工合作，帮你生成学习资料 |
| 3 | 个性化学习路径规划 | 根据你的情况，规划先学什么后学什么 |
| 4 | 智能辅导 | 学习中有问题可以随时问 |
| 5 | 学习效果评估 | 判断你到底学会没学会 |

为了实现这 5 个能力，系统设计了**三层 Agent 体系**。

**三层的关系：**

- **第一层（底座）**：双 Agent 并行，这是整个系统的运行基础
- **第二层（角色）**：5 种专业角色，Agent A 可以召唤它们帮忙
- **第三层（业务）**：对应赛题要求，由第一层和第二层协作实现

## 三、第一层：双 Agent 底座（系统的运行基础）

### Agent A — 前台教学 Agent

**代码位置：** `server/core/agent/agent.ts`

**职责：** 跟你对话，教你知识。它保持完整的对话上下文，不会被打断。

**它能做三件事：**

1. **跟你聊天** — 苏格拉底式的教学对话，你懂就深入，不懂就换角度
2. **调用工具** — 读书写文件、查知识库、搜索记忆（通过工具系统）
3. **召唤子 Agent** — 需要帮忙时，创建第二层的子 Agent 来协作

### Agent B — 后台监控 Agent

**实际是两个组件：**

**① BackgroundAnalyzer（每轮对话后触发）**

每次你跟 Agent A 聊完一轮，这个组件就会自动运行：

```
Agent B 读取刚刚的对话
    ↓
分析三个维度：
├── 用户画像 → 推断你的水平变化、偏好、薄弱点
├── 技能检测 → 你是不是展现出了某种能力？
└── 卡片建议 → 刚才聊的内容值得做成卡片吗？
    ↓
输出结构化 JSON → 更新数据库
```

**② BackgroundReview（每 N 轮对话触发）**

聊了几轮之后，系统 fork 一个独立的 AI 实例，从头到尾审查一遍对话历史：

```
独立 AI 实例启动（不干扰 Agent A）
    ├── 记忆审查 → 用户有没有透露出偏好/期望/行为模式？
    └── 技能审查 → 有没有学到可复用的方法或策略？
    ↓
如果有值得记住的 → 存入长期记忆
如果没有 → 什么都不做
```

### 双 Agent 的协作方式

```
时间线：

你发消息 → Agent A 回复（教学）
              ↓
         Agent B 启动（分析本轮对话）
              ↓
         Agent B 更新画像/检测技能/建议卡片
              ↓
你发下一条消息 → Agent A 回复（带着更新后的画像）
                   ↓
              Agent B 再次启动...（循环）

每 N 轮：BackgroundReview 额外审查一次历史对话
```

**关键：** Agent B 完全不干扰 Agent A。你在使用中感觉不到 Agent B 的存在，但每次对话后画像在更新，知识在累积。

## 四、第二层：子 Agent 角色系统（5 种专业角色）

Agent A 可以召唤子 Agent。每个子 Agent 有特定的专业能力：

| 角色 | 中文名 | 专业能力 | 什么时候召唤 |
|:----:|--------|---------|------------|
| **Oracle** | 巨匠导师 | 苏格拉底式教学对话 | 用户一开始学习时 |
| **Profile** | 画像分析师 | 从对话中提取用户特征 | 需要更新画像时 |
| **Forge** | 卡片锻造师 | 引导用户打磨知识卡片 | 用户有闪念想法时 |
| **Guide** | 学习向导 | 规划学习路径、推荐资源 | 需要决定先学什么时 |
| **Assess** | 评估专家 | 出题测试理解程度 | 需要检验学习效果时 |

**它们怎么联动？** 用一个学习场景举例：

```
你刚学完"涌现理论"
    ↓
Agent A 召唤 Assess 子 Agent
    ↓
Assess Agent 出题："请用你自己的话解释涌现"
    ↓
你回答
    ↓
Assess Agent 判断你的理解程度（掌握/模糊/不懂）
    ↓
把结果返回给 Agent A
    ↓
Agent A 根据结果安排下一步：
  → 掌握 → 进入下一个概念
  → 模糊 → 召唤 Forge Agent 引导你打磨卡片
  → 不懂 → 换一个角度重新讲解
```

## 五、第三层：对应赛题 5 个要求的具体实现

### 要求 1：对话式画像构建（≥6 个维度）

**由谁完成：** Agent A（对话）+ Agent B（提取）+ Profile Agent（分析）

```
Agent A 跟你聊天 → 记录对话
Agent B 分析对话 → 提取特征
Profile Agent 整理 → 形成结构化画像
```

画像包含 6+ 个维度：知识基础、认知风格、学习偏好、薄弱环节、兴趣方向、理解深度。

### 要求 2：多智能体协同资源生成（≥5 种资源）

**由谁完成：** Agent A（协调）+ 多个子 Agent（分工）

```
你提出需求："我想学习神经网络"
    ↓
Agent A 理解需求 → 拆解任务
    ↓
同时召唤多个子 Agent：
├── Literature Agent → 生成讲解文档
├── Guide Agent → 生成知识点思维导图
├── Assess Agent → 生成练习题
├── Oracle Agent → 推荐拓展阅读
└── Forge Agent → 准备代码实操案例
    ↓
汇总 5 种资源，存入文献盒
```

### 要求 3：个性化学习路径规划

**由谁完成：** Guide Agent（规划）+ Agent B（反馈）

```
Guide Agent 读取你的画像
    ↓
分析你已掌握的概念和薄弱环节
    ↓
规划学习顺序："先学A → 再学B → 然后学C"
    ↓
Agent A 按路径教学
Agent B 监控学习效果 → 反馈给 Guide Agent 调整路径
```

### 要求 4：智能辅导

**由谁完成：** Agent A（实时辅导）

就是 Agent A 的日常教学对话。你有问题随时问，它随时答。

### 要求 5：学习效果评估

**由谁完成：** Assess Agent（出题）+ Agent B（分析）

```
Assess Agent 定期出题测试
Agent B 分析你的回答质量
    ↓
判断掌握程度 → 更新画像 → 调整学习计划
```

## 七、实际操作流程

你作为一个用户，感受不到这三层架构。你看到的流程是这样的：

```
打开网页 → 选择"学习人工智能"
    ↓
① 系统创建"冯·诺依曼"导师（底层是第一层+第二层的 Oracle）
② 你开始跟他聊天（Agent A 在工作）
③ 每次对话完，画像在悄悄更新（Agent B 在工作）
④ 学了一段时间，系统生成学习资料（子 Agent 协作）
⑤ 你有想法随时记下来（Forge Agent 引导打磨）
⑥ 想检验学习效果？系统出题考你（Assess Agent）
⑦ 越学，系统越了解你，推荐越精准（Agent B 持续更新画像）
```

**全程你只需要做一件事：跟 Agent 聊天。** 其他所有 Agent 在后台协作，不需要你操作。
