## 核心原则

### 1. RPC 是 API 调用的唯一方式

使用 Hono 的 RPC 特性调用 API，像调用本地函数一样调用远程 API。

```typescript
// ✅ 正确：使用 RPC
import { client } from '@/lib/api-client'
const res = await client.api.tasks.$get()
const data = await res.json()

// ❌ 禁止：直接使用 fetch/axios
const res = await fetch('/api/tasks')
```

### 2. 类型推导是关键

使用 `InferRequestType` 和 `InferResponseType` 自动推导类型：

```typescript
import { InferRequestType, InferResponseType } from 'hono/client'

const $post = client.api.tasks.$post
type PostRequestType = InferRequestType<typeof $post>['json']
type PostResponseType = InferResponseType<typeof $post>['data']
```

### 3. 自定义 Hook 封装一切

在 `hooks/` 中封装 RPC + React Query：

```typescript
// hooks/use-tasks.ts
export function useTasks() {
  return useQuery({
    queryKey: ['tasks'],
    queryFn: async () => {
      const res = await client.api.tasks.$get()
      return (await res.json()).data
    },
  })
}
```

### 4. 状态管理三原则

- **全局状态** → Zustand（`stores/`）
- **服务端数据** → React Query（`hooks/` 中封装）
- **局部 UI 状态** → `useState` / `useReducer`

### 5. 错误处理统一

```typescript
// server/api/error.ts 中定义
throw new ApiError({ code: 404, message: '资源不存在' })

// app 层：app.onError(handleError) 统一捕获
```

## 必遵规范

### 项目结构

| 目录 | 用途 |
|------|------|
| `server/api/` | Hono API 路由 |
| `server/api/routes/` | 具体路由模块 |
| `server/api/error.ts` | 错误处理 |
| `server/api/validator.ts` | Zod 校验器 |
| `hooks/use-*.ts` | 自定义 Hook |
| `stores/*.ts` | Zustand Store |
| `components/` | React 组件 |
| `lib/` | 工具库 |

### React Query 规范

```typescript
// useQuery：必须处理 isLoading + error
const { data, isLoading, error } = useQuery({ ... })
if (isLoading) return <div>加载中...</div>
if (error) return <div>加载失败</div>

// useMutation：必须处理 onSuccess + onError + invalidateQueries
const mutation = useMutation({
  mutationFn: ...,
  onSuccess: () => {
    queryClient.invalidateQueries({ queryKey: ['tasks'] })
    toast.success('成功')
  },
  onError: (e) => toast.error('失败：' + e.message),
})
```

### 禁止事项

| 技术 | 替代方案 |
|------|---------|
| fetch / axios / ky | Hono RPC |
| useState + useEffect 管理数据 | React Query |
| Redux / SWR | React Query |
| CSS Modules / Styled Components | Tailwind CSS |
| Ant Design / Material-UI | shadcn/ui |

## 开发流程

添加新功能的标准 4 步：

```
1. prisma/schema.prisma    →  定义数据模型
2. server/api/routes/      →  编写后端 API
3. hooks/use-*.ts          →  封装 Hook（RPC + React Query）
4. app/<page>/page.tsx     →  编写页面
```
