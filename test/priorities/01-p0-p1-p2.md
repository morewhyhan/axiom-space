# P0 / P1 / P2 必测清单

对照 `08-测试计划.md` 第 8 章。

## P0 必测

| 用例 ID | 范围 | 测试方式 | 输入 | 预期输出 | 通过标准 | 如果错了，可能是什么错 |
|---|---|---|---|---|---|---|
| P0-001 | 数据边界 | API 集成 | userA 访问 userB 数据 | PermissionError | 返回 PermissionError，响应体不含 userB 的对象 ID 或内容 | 认证上下文错；API 没做 owner 校验；响应序列化前已查出越界数据 |
| P0-002 | Card 契约 | API + 数据库 | 重复 path、非法 type | Validation / Conflict | 重复 path 返回 ConflictError；非法 type 返回 ValidationError；都不写入 Card | 数据库唯一性缺失；领域枚举缺失；错误路径仍写入 |
| P0-003 | Path / Step | API + 领域单元 | path、step、card | consistent state | Step.pathId 指向 Path；同一 Path 下 Step.order 唯一且按升序返回；Step 绑定 Card 时 Path、Step、Card 三者 vaultId 相同；PathProgress.doneSteps 等于 completed/mastered Step 数 | Step 绑定跨 Vault Card；PathProgress 源数据错 |
| P0-004 | Session | API 集成 | archived session 写消息 | StateTransitionError | archived Session 写入失败，LearningMessage 数量不增加 | 状态机没阻止写入；消息先写后校验 |
| P0-005 | Graph | API 集成 | 跨 Vault Edge | BoundaryError | 跨 Vault Edge 请求返回 BoundaryError，Edge 表不新增记录 | Edge 服务没校验 source/target Vault |
| P0-006 | Assessment / Profile | 领域单元 | 无 evidence 结果 | rejected | evidence 为空时 Step.mastery、Capability、EducationProfile 均不变化 | 掌握度、能力、画像被无证据结果污染 |
| P0-007 | Agent 安全 | Agent 契约 | high risk tool | confirmation required | 高风险 ToolCall 返回 confirmation，未批准前不执行写操作 | ToolRisk 标注错；Agent 绕过确认流 |
| P0-008 | Sidecar | 异步 / Sidecar | RAG / Resource / Notification 失败 | main object still saved | 副作用 failed 时 Card / Path / Session 主对象仍可读取且状态不回滚 | 副作用和主事务耦合；失败回滚主对象 |

## P1 必测

| 用例 ID | 范围 | 测试方式 | 输入 | 预期输出 | 通过标准 | 如果错了，可能是什么错 |
|---|---|---|---|---|---|---|
| P1-001 | 重复导入 | API 集成 | same document twice | duplicate handled | 第二次导入返回 duplicate/skipped 或合并结果；不会创建重复 literature/permanent | contentHash 去重缺失；导入幂等失败 |
| P1-002 | WikiLink 同步 | API + 领域单元 | update content | edges changed | 新增/删除 WikiLink 后 Edge 表变化与 LinkSyncResult 一致 | LinkSync 只增不删；结果和数据库不一致 |
| P1-003 | PromotionAttempt | 领域单元 | low quality card | missing sections | 返回 missingSections/reason；Card.type 不变 | 低质量卡升级；失败原因不可见 |
| P1-004 | Search / Recommendation | API + 领域单元 | query、profile | result + reason | 每条结果/推荐都有 targetId 和 reason；targetId 查不到则不通过 | 搜索结果断链；推荐无依据 |
| P1-005 | BackgroundJob | 异步 | failed job | failed state | failed job 可查 status=failed 和 error；不会静默消失 | 后台任务吞错 |
| P1-006 | VaultExport | API 集成 | export vault | package | export manifest 中所有 card.vaultId 都等于请求 vaultId | 导出混入其他 Vault |

## P2 必测

| 用例 ID | 范围 | 测试方式 | 输入 | 预期输出 | 通过标准 | 如果错了，可能是什么错 |
|---|---|---|---|---|---|---|
| P2-001 | Dashboard 指标 | API + UI | source objects | stats | 相同源对象重复计算得到相同 stats；请求 Dashboard 不写源表 | 指标计算不稳定；读模型有写副作用 |
| P2-002 | RecentActivity 降级 | UI / E2E | deleted target | fallback display | target 删除后 activity 显示 deleted/fallback，不出现空白崩溃 | 活动记录不处理断链 |
| P2-003 | Subagent 复盘 | Agent 契约 | subagent failure | run record | 失败 run record 含 role、inputSummary、error、status=failed | 子 Agent 失败无审计 |
| P2-004 | ResourceManifest | 异步 | multi-file output | manifest | manifest.items 数量等于产物文件数，每项有 path/ref 和 status | 多文件资源漏登记 |
| P2-005 | MCP 工具 | Agent 契约 | external tool | ToolContract | MCPToolDefinition 未转 ToolContract 前不能执行；执行时遵守 risk/confirmation | 外部工具绕过本地安全层 |
| P2-006 | UI 布局 | UI / E2E | layout changes | UI only | layout 改变后 Card/Path/Session/Edge 数据不变化 | UI 布局状态污染领域数据 |
