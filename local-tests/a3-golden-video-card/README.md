# AXIOM Space A3 黄金案例 · V18 融合演示版

这是评委展示视频使用的 16:9 HTML 演示稿。当前版本以
`docs/03-宣传资料/AXIOM-Space-A3黄金案例-V18-固定顺序传播终审稿.md`
为内容真源，把原先分开的讲解页与视频页重构为 **15 个融合演示单元**。

## 不可改变的叙事顺序

1. 进入自己的课程仓库。
2. 用连续对话建立初始画像。
3. 查看画像判断、依据与变化。
4. 导入学习资料。
5. 在 Dashboard 查看知识节点与三色卡片。
6. 从图谱点击青色灵感卡，在工作台打开同一节点。
7. Agent A 苏格拉底式追问，学生完成费曼输出，Agent B 保存证据。
8. 解释永久卡的审核边界。
9. 审核未通过后继续打磨，复审通过后灵感卡升级为永久卡。
10. 学生主动点名请求资源。
11. 系统基于缺口与偏好提出有依据、可拒绝的资源建议。
12. 展示六类资源对应的六种学习动作。
13. 查看图谱的结构、关系、来源三种视角。
14. 查看图谱的卡片、掌握、路径三种视角。
15. 用浅色静态页收束长期价值。

## 设计合同

- 保留米白、青绿、粉/青/紫三色卡与 Axiom Sans 视觉语言。
- 每页只有一个主判断；主汉字必须先于解释被看见。
- 产品视频是页面主体，文字直接贴着真实动作与结果出现。
- 同一时刻最多保留一个主字幕和一个必要标注。
- 粉色文献卡＝资料来源；青色灵感卡＝尚未掌握；紫色永久卡＝已经掌握。
- Agent A 负责追问，学生负责亲自输出，Agent B 负责保存原话与学习证据。
- 候选、碎片或未核验素材必须显示导演状态，不能冒充最终证据。
- 缺素材页一律使用浅色补录板，禁止黑屏和空白页。
- 第 15 页不创建视频元素，只保留产品定位、价值主张与愿景。

## 文件结构

- `index.html`：演示外壳、控制栏与口播面板。
- `v18-story.js`：15 页内容、完整口播、素材状态、时间 beat 与页面视觉数据。
- `v18-deck.js`：动态渲染、导航、视频/口播同步、导演视图与录制视图。
- `v18-deck.css`：当前浅色视觉体系与 15 页布局。
- `story.js`、`deck.js`、`deck.css`：旧版保留文件，不再由 `index.html` 加载。

## 当前素材真实性状态

已有候选素材不等于已经通过最终验收。重点缺口仍是：

- 02：完整的初始画像连续对话。
- 06：Dashboard 青色灵感卡到同名工作台的准确路径。
- 07：Agent A → 学生 → Agent B → 旧知召回的连续母带。
- 09：审核失败、针对性追问、新情境作答、复审通过与路径改变的连续母带。
- 11：有依据、可拒绝、确认后才生成的系统资源建议。
- 14：审核后同一仓库状态下的卡片、掌握与路径模式。

导演视图会如实显示这些状态；录制最终成片前必须逐项补齐或验收。

## 操作

- `←` / `PageUp`：上一页。
- `→` / `PageDown` / `Space` / `Enter`：下一页。
- `P`：播放或暂停当前实录与时间字幕。
- `N`：打开或关闭本页完整口播与画面 beat。
- `D`：切换导演信息。
- `R`：切换录制视图。
- `F`：进入或退出全屏。
- `Home` / `End`：首尾页。
- `#unit-01` 至 `#unit-15`：直接打开指定单元。

## 发布门禁

最终录制前至少检查：

- 正好 15 页，且顺序与 V18 一致。
- 每页 3 秒内能说出唯一主旨。
- 静音观看仍能看懂“正在做什么”和“发生了什么结果”。
- 关键动作与配音误差不超过约 120ms。
- 主标题、动作字幕和产品定位在 1920×1080、1366×768、1280×720 均无裁切。
- 视频与页面之间没有黑场；第 15 页保持浅色定格。
- 所有候选、碎片、未核验和缺失素材在发布版本中已经清零。
