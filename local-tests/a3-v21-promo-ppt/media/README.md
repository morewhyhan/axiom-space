# A3 宣传片实录证据标准

这组视频不是装饰素材，而是页面主张的可见证据。每一幕只有同时满足下面四项，才算通过：

1. 画面里能看见具体操作或明确对象。
2. 操作前后能看见状态发生变化。
3. 结果能在界面中直接读取，不依赖配音补充。
4. 结果能自然交给下一幕继续使用。

如果画面只能证明“打开了某个页面”，页面文字就不能写成“已经证明跨页面连续、个性化或掌握”。这类情况必须补录、补数据，或把文案收回到画面实际证明的范围。

## 当前数据基线

- 长期档案仓库（产品可见口径）：311 张卡片、357 条关系、102 张永久卡、98 张灵感卡、111 张文献卡、10 个星团。数据库内还有 1 张系统内部仓库根卡，不对用户展示，不计入 PPT 口径。
- 学习证据：133 个会话、12 次评估、45 条能力记录、5 项技能状态、96 条语义索引。
- 资源证据：12 个资源任务、6 条推送建议。
- 导入专项仓库：导入前为 0 个节点；导入同一份设计模式原文后，界面显示 38 个节点、223 条关系、37 张灵感卡、1 张文献卡和 1 条学习路径。

## 导入片段的录制校验顺序

导入前后不能靠两套预置截图拼接。重录时必须使用同一个 `a3-cap-import-proof-vault-20260718` 仓库，并按下面顺序完成：

1. 运行 `scripts/video/prepare-a3-cap-import-proof.ts`，清空专项仓库。
2. 以 `A3_IMPORT_PROOF_EXPECT=before` 运行 `scripts/video/check-a3-cap-import-proof.ts`；只有产品可见卡片、关系、路径、文档均为 0 才能录制导入前画面。
3. 在产品界面提交同一份设计模式原文，等待实际解析与学习路径生成完成。
4. 以 `A3_IMPORT_PROOF_EXPECT=after` 再次运行校验；只有 38 张产品可见卡片、37 张灵感卡、1 张文献卡、223 条关系、1 条路径、1 份文档全部命中，才能录制最终仪表盘。

这样三段画面证明的是同一次真实导入，而不是“先准备一个空仓库，再切到另一个大仓库”。

## 替换视频时的失败条件

- 黑屏、加载页或静止空白超过 1 秒。
- 关键数字、标题或状态小到无法在 PPT 画中画尺寸下读取。
- 页面主张的对象和视频中的对象不是同一张卡、同一仓库或同一主题。
- 只有结果，没有前置动作；或只有动作，没有可见结果。
- 多段视频切换后，舞台字幕和右侧说明仍停留在上一段。
- 通过配音声称“个性化、连续、掌握、写回”，但界面没有出现对应证据。

所有片段保持 H.264 MP4、16:9、静音自动播放。一个页面需要多段连续证据时，在 `data-video-src` 中用 `|` 连接；对应字幕在 `data-video-captions` 中用 `||` 连接，条数必须一致。
